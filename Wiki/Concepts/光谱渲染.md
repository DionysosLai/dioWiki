# 光谱渲染 (Spectral Rendering)

## 定义

光谱渲染是一种使用波长采样的光谱分布（而非 RGB 三通道）进行光照计算的渲染方法。这是 pbrt 第4版的核心特性变化。

## 为什么弃用 RGB？

- RGB 是**显示设备**的颜色表示，不是物理量
- 光谱分布是**连续函数**，RGB 只有3个采样点
- 使用光谱可以准确模拟**色散**（如棱镜分光）等物理效果
- 色度介质（散射属性随波长变化）需要光谱渲染

## pbrt 第4版的实现

- 完全弃用 RGB 进行光照计算
- 使用 `SampledSpectrum` 类表示波长采样的光谱分布
- 场景中的颜色参数在渲染时转换为光谱
- 最终输出再从光谱转换为显示设备的颜色

### 光谱表示体系

| 类 | 用途 |
|----|------|
| `Spectrum` | 通用光谱接口（标签联合体/tagged union） |
| `ConstantSpectrum` | 所有波长返回相同值 |
| `DenselySampledSpectrum` | 每纳米一个采样值的密集表示 |
| `PiecewiseLinearSpectrum` | 分段线性插值（CIE 标准数据等） |
| `RGBAlbedoSpectrum` | 从 RGB 反照率构造的有界光谱 |
| `RGBIlluminantSpectrum` | 从 RGB 发射值构造的无界光谱 |
| `RGBUnboundedSpectrum` | 通用 RGB → 光谱转换 |
| `SampledSpectrum` | 运行时携带的波长-值对（NSpectrumSamples = 4） |

### 颜色空间

pbrt 中的颜色流水线：**场景光谱 → SampledSpectrum → XYZ → RGB**

- 所有光照计算在**光谱域**进行
- `PixelSensor` 模拟物理传感器的光谱响应
- 输出时经 XYZ → 目标色空间（sRGB、DCI-P3 等）转换

## 可见光波长范围

- 约 **380 nm - 780 nm**
- 低波长 (~400 nm) → 蓝色
- 中波长 (~550 nm) → 绿色
- 高波长 (~650 nm) → 红色

## 色散效应

光谱渲染能准确模拟**色散**——不同波长的光在介质中折射率不同：
- 棱镜将白光分解为彩虹
- 钻石的"火彩"
- 色散介质（[[体积散射]]中的散射属性随波长变化）

这是 RGB 渲染**无法**模拟的效果。
