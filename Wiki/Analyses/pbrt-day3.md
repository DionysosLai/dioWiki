**对应阅读材料**：`Raw/pbrt/1 Introduction/1.3 pbrt System Overview.md`
**预计时长**：1.5-2 小时
**定位**：第 1 周第 3 天，是首遍阅读中**建立系统地图的关键一天**

## 今天在 8 周计划中的位置

- Day 1：读完 Preface + Ch1 开头，知道 pbrt 想教什么
- Day 2：读完 1.1 文学编程 + 1.2 光线追踪 7 大组件

## 今天的核心目标

读完 1.3 后，你应该能说清楚这三件事：

1. pbrt 是靠哪 **14 个基类型（base types）** 搭起来的
2. pbrt 的执行分为哪 **3 个阶段**，谁负责什么
3. **Integrator（积分器）** 为什么是整本书的主角

只要能答这三题，Day 3 就算完成。**不求看懂每个接口的每个方法，也不求记住所有类名。**

## 分段阅读指引

### 第 1 段：14 个基类型（1.3 开头 + 表 1.1）

**抓什么**：

- pbrt **整个系统就是围绕这 14 个基类型的接口**展开的。后面 15 章基本都在讲其中某几个。
- 表 1.1 是**全书导航表**，以后你翻书卡壳时可以反复回来查。

**先混个脸熟就够，不要硬记**。建议把这 14 个按功能粗分成 4 组：

| 组 | 类型 | 对应的章节 | 干什么的 |
|----|------|-----------|---------|
| 物理量 | Spectrum | Ch4 | 表示光的颜色/能量 |
| 几何 | Shape, Primitive | Ch6, Ch7 | 场景里的几何形状 + 加速结构 |
| 图像形成 | Camera, Sampler, Filter | Ch5, Ch8 | 相机怎么发射线、怎么采样、怎么合像素 |
| 材质与光 | BxDF, Material, Texture, Medium, Light, LightSampler | Ch9-Ch12 | 光打到表面会发生什么 |
| 统筹者 | Integrator | Ch13-Ch15 | 把以上全部串起来，最终出图 |

**暗线**：注意 pbrt 第 4 版不再用传统的虚函数继承，而是用 **tag-based dispatch（标签分派）**。这是为了让同一份代码能同时在 CPU 和 GPU 上跑。首遍**知道有这件事**就行，不用钻研实现。

### 第 2 段：1.3.1 执行的三阶段

**一句话版**：

1. **解析阶段** — 读场景描述文件，生成一个 `BasicScene`（还不能直接渲染）
2. **对象构造阶段** — 把 `BasicScene` 里的描述变成真实的 Shape/Material/Light 对象
3. **渲染阶段** — 主渲染循环，调用 Integrator 出图

**关键点**：pbrt 故意把“**场景描述**”和“**可渲染对象**”分成两步，因为 GPU 渲染需要做一堆额外的内存布局与数据搬运。这一刀切得干净，是整个系统能同时跑 CPU/GPU 的基础。

### 第 3 段：1.3.2 main() 的 7 步

```
1. 参数字符串化
2. 声明变量
3. 处理命令行参数
4. InitPBRT(options)          ← 初始化系统
5. ParseFiles(&builder, ...)  ← 触发解析阶段
6. 渲染场景                    ← 触发对象构造 + 渲染阶段
7. CleanupPBRT()              ← 清理
```

**不要去读 cmd/pbrt.cpp 的真实代码**。首遍你只需要知道：整本书讲的所有东西，都塞在第 6 步“Render the scene”里面。

### 第 4 段：1.3.3 Integrator（今天的重点）

Integrator 是整本书的主角，从 Ch13 开始真正爆发，但 Ch1 这里就埋下了骨架。

读这一段时，**只追一条主线**：

```
Integrator::Render()
   └── ImageTileIntegrator::Render()          // 把图像切成 tile，并行处理
        └── EvaluatePixelSample()             // 对每个像素做一次采样
             └── RayIntegrator::EvaluatePixelSample()
                  └── Li(ray, ...)            // 真正计算这条射线带回的 radiance
```

**四个层级的职责**：

| 层级 | 干什么 |
|------|-------|
| `Integrator` | 抽象基类，定义 `Render()` 接口 |
| `ImageTileIntegrator` | 把图像分块并行化（**这是 CPU 并行化的骨架**） |
| `RayIntegrator` | 为每个像素发射一条 camera ray，把求 radiance 的活委托给 `Li()` |
| 各种具体 Integrator（如 `PathIntegrator`） | 实现 `Li()`，真正做光传输计算 |

**理解到“每条 ray 都对应一次 Li() 调用”就够了**。Li() 里是什么，留到 Ch13 再说。

## 常见的坑

1. **想一次看懂 tag-based dispatch** — 别陷进去，它是第 4 版 pbrt 的工程实现技巧，不影响你理解渲染算法。
2. **想记住所有源文件路径** — 表 1.1 里的 `base/*.h` 路径将来查书时再看即可。
3. **在 Integrator 这里就去想“PathIntegrator 到底怎么实现”** — 不要，那是 Ch13 的事。Day 3 **只抓骨架，不碰血肉**。
4. **在 CPU/GPU 差异上纠结** — 1.3 里对此只做了引子，Ch15 才是正文。现在知道“系统设计时考虑了两条路径”就够。
