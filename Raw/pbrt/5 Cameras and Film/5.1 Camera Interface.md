## 5.1 Camera Interface

The Camera class uses the usual TaggedPointer-based approach to dynamically dispatch interface method calls to the correct implementation based on the actual type of the camera. Camera is defined in the file base/camera.h.

> Camera 类使用通常的基于 TaggedPointer 的方法，根据相机的实际类型动态分派接口方法调用到正确的实现。Camera 定义在文件 base/camera.h 中。

The first method that cameras must implement is GenerateRay(), which computes the ray corresponding to a given image sample. It is important that the direction component of the returned ray be normalized—many other parts of the system will depend on this behavior. If for some reason there is no valid ray for the given CameraSample, then the pstd::optional return value should be unset. The SampledWavelengths for the ray are passed as a non-const reference so that cameras can model dispersion in their lenses, in which case only a single wavelength of light is tracked by the ray and the GenerateRay() method will call SampledWavelengths::TerminateSecondary().

> 相机必须实现的第一个方法是 GenerateRay()，它计算与给定图像样本对应的光线。返回光线的方向分量必须被归一化——系统的许多其他部分将依赖于这一行为。如果由于某种原因给定的 CameraSample 没有有效光线，则 pstd::optional 返回值应为空。光线的 SampledWavelengths 作为非 const 引用传递，这样相机可以对其透镜中的色散进行建模，在这种情况下光线只追踪单一波长的光，GenerateRay() 方法将调用 SampledWavelengths::TerminateSecondary()。

The CameraSample structure that is passed to GenerateRay() holds all the sample values needed to specify a camera ray. Its pFilm member gives the point on the film to which the generated ray should carry radiance. The point on the lens the ray passes through is in pLens (for cameras that include the notion of lenses), and time gives the time at which the ray should sample the scene. If the camera itself is in motion, the time value determines what camera position to use when generating the ray.

> 传递给 GenerateRay() 的 CameraSample 结构体包含指定相机光线所需的所有采样值。其 pFilm 成员给出生成的光线应将辐射传送到胶片上的点。pLens 表示光线通过透镜上的点（对于包含透镜概念的相机），time 给出光线应采样场景的时间。如果相机本身在运动中，time 值决定生成光线时使用哪个相机位置。

Finally, the filterWeight member variable is an additional scale factor that is applied when the ray's radiance is added to the image stored by the film; it accounts for the reconstruction filter used to filter image samples at each pixel. This topic is discussed in Sections 5.4.3 and 8.8.

> 最后，filterWeight 成员变量是一个额外的缩放因子，在光线的辐射被添加到胶片存储的图像中时应用；它考虑了在每个像素处过滤图像样本所使用的重建滤波器。该主题在第 5.4.3 节和第 8.8 节中讨论。

The CameraRay structure that is returned by GenerateRay() includes both a ray and a spectral weight associated with it. Simple camera models leave the weight at the default value of one, while more sophisticated ones like RealisticCamera return a weight that is used in modeling the radiometry of image formation.

> GenerateRay() 返回的 CameraRay 结构体包含光线和与之关联的光谱权重。简单的相机模型将权重保持默认值 1，而更复杂的模型（如 RealisticCamera）返回的权重用于建模成像的辐射度量。

Cameras must also provide an implementation of GenerateRayDifferential(), which computes a main ray like GenerateRay() but also computes the corresponding rays for pixels shifted one pixel in the x and y directions on the film plane. This information about how camera rays change as a function of position on the film helps give other parts of the system a notion of how much of the film area a particular camera ray's sample represents, which is useful for antialiasing texture lookups.

> 相机还必须提供 GenerateRayDifferential() 的实现，它像 GenerateRay() 一样计算主光线，但也计算胶片平面上向 x 和 y 方向各偏移一个像素的对应光线。关于相机光线如何随胶片上位置变化而变化的信息，帮助系统其他部分了解特定相机光线的样本代表了多大的胶片面积，这对纹理查找的反走样很有用。

Camera implementations must provide access to their Film, which allows other parts of the system to determine things such as the resolution of the output image.

> 相机实现必须提供对其 Film 的访问，这允许系统其他部分确定诸如输出图像分辨率等信息。

Just like real-world cameras, pbrt's camera models include the notion of a shutter that opens for a short period of time to expose the film to light. One result of this nonzero exposure time is *motion blur*: objects that are in motion relative to the camera during the exposure are blurred. Time is yet another thing that is amenable to point sampling and Monte Carlo integration: given an appropriate distribution of ray times between the shutter open time and the shutter close time, it is possible to compute images that exhibit motion blur.

> 与真实世界的相机一样，pbrt 的相机模型包含快门的概念——快门在短时间内打开以将胶片暴露于光线中。非零曝光时间的一个结果是*运动模糊*：在曝光期间相对于相机运动的物体会变模糊。时间也适合点采样和蒙特卡罗积分：给定在快门打开和关闭时间之间适当分布的光线时间，可以计算出呈现运动模糊的图像。

The SampleTime() interface method should therefore map a uniform random sample u in the range [0, 1) to a time when the camera's shutter is open. Normally, it is just used to linearly interpolate between the shutter open and close times.

> 因此 SampleTime() 接口方法应将 [0, 1) 范围内的均匀随机样本 u 映射到相机快门打开时的某个时刻。通常它只是用于在快门打开和关闭时间之间线性插值。

The last interface method allows camera implementations to set fields in the ImageMetadata class to specify transformation matrices related to the camera. If the output image format has support for storing this sort of auxiliary information, it will be included in the final image that is written to disk.

> 最后一个接口方法允许相机实现在 ImageMetadata 类中设置字段以指定与相机相关的变换矩阵。如果输出图像格式支持存储这类辅助信息，它将包含在写入磁盘的最终图像中。

### 5.1.1 Camera Coordinate Spaces

Before we start to describe the implementation of pbrt's camera models, we will define some of the coordinate spaces that they use. In addition to world space, which was introduced in Section 3.1, we will now introduce four additional coordinate spaces, *object space*, *camera space*, *camera-world space*, and *rendering space*. In sum, we have:

> 在开始描述 pbrt 相机模型的实现之前，我们将定义它们使用的一些坐标空间。除了第 3.1 节引入的世界空间外，我们现在引入四个额外的坐标空间：*物体空间*、*相机空间*、*相机-世界空间*和*渲染空间*。总结如下：

- *Object space:* This is the coordinate system in which geometric primitives are defined. For example, spheres in pbrt are defined to be centered at the origin of their object space.
- *World space:* While each primitive may have its own object space, all objects in the scene are placed in relation to a single world space.
- *Camera space:* A camera is placed in the scene at some world space point with a particular viewing direction and orientation. This camera defines a new coordinate system with its origin at the camera's location.
- *Camera-world space:* Like camera space, the origin of this coordinate system is the camera's position, but it maintains the orientation of world space.
- *Rendering space:* This is the coordinate system into which the scene is transformed for the purposes of rendering. In pbrt, it may be world space, camera space, or camera-world space.

> - *物体空间*：几何图元在其中定义的坐标系。例如，pbrt 中的球体被定义为以其物体空间原点为中心。
> - *世界空间*：虽然每个图元可能有自己的物体空间，但场景中的所有对象都相对于单一的世界空间放置。
> - *相机空间*：相机放置在场景中世界空间的某个点，具有特定的观察方向和朝向。该相机定义了一个新的坐标系，原点在相机位置。
> - *相机-世界空间*：与相机空间类似，该坐标系的原点是相机位置，但保持世界空间的朝向。
> - *渲染空间*：场景为渲染目的而变换到的坐标系。在 pbrt 中，它可能是世界空间、相机空间或相机-世界空间。

Renderers based on rasterization traditionally do most of their computations in camera space. Conversely, many ray tracers (including all versions of pbrt prior to this one) render in world space. A problem with that approach stems from the fact that floating-point numbers have more precision close to the origin than far away from it. If the camera is placed far from the origin, there may be insufficient precision to accurately represent the part of the scene that it is looking at.

> 基于光栅化的渲染器传统上在相机空间中执行大部分计算。相反，许多光线追踪器（包括 pbrt 之前的所有版本）在世界空间中渲染。这种方法的一个问题源于浮点数在靠近原点时比远离原点时具有更高的精度。如果相机放置在远离原点的位置，可能没有足够的精度来准确表示它正在观察的场景部分。

Figure 5.1 illustrates the precision problem with rendering in world space. In (a), the scene is rendered with the camera and objects as originally specified. In (b), both the camera and the scene have been translated 1,000,000 units in each dimension—much less precision is available, and the discretization of floating-point numbers is visible in the geometric model. In (c), if the translated scene is rendered in camera-world space, much more precision is available and the geometric detail is preserved.

> 图 5.1 说明了在世界空间中渲染的精度问题。在 (a) 中，场景按原始规格用相机和物体渲染。在 (b) 中，相机和场景都在每个维度上平移了 1,000,000 个单位——可用精度大大降低，浮点数的离散化在几何模型中可见。在 (c) 中，如果在相机-世界空间中渲染平移后的场景，则有更多精度可用，几何细节得以保留。

Rendering in camera space naturally provides the most floating-point precision for the objects closest to the camera. However, there is a problem with using camera space with ray tracing: axis-aligned bounding boxes of features aligned to coordinate axes become less effective if the camera is rotated with respect to the scene, which increases rendering time.

> 在相机空间中渲染自然为最靠近相机的物体提供最高的浮点精度。然而，将相机空间用于光线追踪存在一个问题：如果相机相对于场景旋转，与坐标轴对齐的特征的轴对齐包围盒效率会降低，从而增加渲染时间。

Rendering using camera-world space gives the best of both worlds: the camera is at the origin and the scene is translated accordingly. However, the rotation is not applied to the scene geometry, thus preserving good bounding boxes for the acceleration structures. The CameraTransform class abstracts the choice of which particular coordinate system is used for rendering by handling the details of transforming among the various spaces.

> 使用相机-世界空间进行渲染兼具两者之长：相机位于原点，场景相应地进行平移。然而，旋转不会应用于场景几何体，从而为加速结构保留了良好的包围盒。CameraTransform 类通过处理各种空间之间变换的细节，抽象了选择哪个特定坐标系用于渲染的过程。

CameraTransform maintains two transformations: one from camera space to the rendering space, and one from the rendering space to world space. In pbrt, the latter transformation cannot be animated; any animation in the camera transformation is kept in the first transformation. This ensures that a moving camera does not cause static geometry in the scene to become animated, which in turn would harm performance.

> CameraTransform 维护两个变换：一个从相机空间到渲染空间，另一个从渲染空间到世界空间。在 pbrt 中，后一个变换不能有动画；相机变换中的任何动画都保留在第一个变换中。这确保了移动的相机不会导致场景中的静态几何体变为动画的，因为那会损害性能。

The CameraTransform constructor takes the world-from-camera transformation as specified in the scene description and decomposes it into the two transformations described earlier. The default rendering space is camera-world, though this choice can be overridden using a command-line option.

> CameraTransform 构造函数接受场景描述中指定的世界-从-相机变换，并将其分解为前述的两个变换。默认渲染空间是相机-世界空间，但可以通过命令行选项覆盖此选择。

For camera-space rendering, the world-from-camera transformation should be used for worldFromRender. For camera-world space, the world-from-render transformation is given by translating to the camera's position at the midpoint of the frame. For world-space rendering, worldFromRender is the identity transformation.

> 对于相机空间渲染，world-from-camera 变换应用于 worldFromRender。对于相机-世界空间，world-from-render 变换由平移到帧中间点处相机位置来给出。对于世界空间渲染，worldFromRender 是单位变换。

Camera implementations must make their CameraTransform available to other parts of the system.

> 相机实现必须使其 CameraTransform 可供系统其他部分访问。

### 5.1.2 The CameraBase Class

All of the camera implementations in this chapter share some common functionality that we have factored into a single class, CameraBase, from which all of them inherit. CameraBase, as well as all the camera implementations, is defined in the files cameras.h and cameras.cpp.

> 本章中所有相机实现共享一些公共功能，我们将其提取到一个类 CameraBase 中，所有相机实现都继承自它。CameraBase 以及所有相机实现都定义在文件 cameras.h 和 cameras.cpp 中。

The CameraBase constructor takes a variety of parameters that are applicable to all of pbrt's cameras:

> CameraBase 构造函数接受适用于所有 pbrt 相机的各种参数：

- One of the most important is the transformation that places the camera in the scene, which is represented by a CameraTransform and is stored in the cameraTransform member variable.
- Next is a pair of floating-point values that give the times at which the camera's shutter opens and closes.
- A Film instance stores the final image and models the film sensor.
- Last is a Medium instance that represents the scattering medium that the camera lies in, if any.

> - 最重要的参数之一是将相机放置在场景中的变换，由 CameraTransform 表示并存储在 cameraTransform 成员变量中。
> - 接下来是一对浮点值，给出相机快门打开和关闭的时间。
> - 一个 Film 实例存储最终图像并对胶片传感器建模。
> - 最后是一个 Medium 实例，表示相机所在的散射介质（如果有的话）。

The SampleTime() method is implemented by linearly interpolating between the shutter open and close times using the sample u.

> SampleTime() 方法通过使用样本 u 在快门打开和关闭时间之间线性插值来实现。

CameraBase provides a GenerateRayDifferential() method that computes a ray differential via multiple calls to a camera's GenerateRay() method. Two attempts are made to find the ray differential: one using forward differencing and one using backward differencing by a fraction of a pixel. It is important to try both of these due to vignetting at the edges of images formed by realistic camera models—sometimes the main ray is valid but shifting in one direction moves past the image formed by the lens system. In that case, trying the other direction may successfully generate a ray.

> CameraBase 提供了一个 GenerateRayDifferential() 方法，通过多次调用相机的 GenerateRay() 方法来计算光线微分。对光线微分进行两次尝试：一次使用前向差分，一次使用后向差分，偏移量为像素的一个分数。由于写实相机模型形成的图像边缘处的暗角，两种尝试都很重要——有时主光线是有效的，但向一个方向偏移会超出透镜系统形成的图像。在这种情况下，尝试另一个方向可能成功生成光线。

If it was possible to generate the auxiliary ray, then the corresponding pixel-wide differential is initialized via differencing. If a valid ray was found for both x and y, we can set the hasDifferentials member variable to true. Otherwise, the main ray can still be traced, just without differentials available.

> 如果能够生成辅助光线，则通过差分初始化相应的像素宽度微分。如果 x 和 y 都找到了有效光线，我们可以将 hasDifferentials 成员变量设为 true。否则，主光线仍然可以追踪，只是没有可用的微分。

Finally, for the convenience of its subclasses, CameraBase provides various transformation methods that use the CameraTransform.

> 最后，为了其子类的便利，CameraBase 提供了使用 CameraTransform 的各种变换方法。
