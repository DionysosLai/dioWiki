## 8.4 Independent Sampler

The IndependentSampler is perhaps the simplest possible (correct) implementation of the Sampler interface. It returns independent uniform sample values for each sample request without making any further effort to ensure the quality of the distribution. The IndependentSampler should never be used for rendering if image quality is a concern, but it is useful for setting a baseline to compare against better samplers.

> IndependentSampler 可能是 Sampler 接口最简单的（正确）实现。它为每个采样请求返回独立均匀的采样值，不做任何进一步的努力来确保分布质量。如果关心图像质量，永远不应使用 IndependentSampler 进行渲染，但它作为与更好采样器比较的基线很有用。

Like many samplers, IndependentSampler takes a seed to initialize the pseudo-random number generator. An instance of the RNG class generates sample coordinate values. To ensure deterministic behavior for a given pixel and sample index, `StartPixelSample()` resets the RNG to a deterministic state by choosing a sequence based on pixel coordinates and seed, then advancing by `sampleIndex * 65536 + dimension`.

> 与许多采样器一样，IndependentSampler 接受一个种子来初始化伪随机数生成器。RNG 类的一个实例生成采样坐标值。为确保给定像素和采样索引的确定性行为，`StartPixelSample()` 通过根据像素坐标和种子选择序列，然后前进 `sampleIndex * 65536 + dimension` 来将 RNG 重置为确定性状态。

All the methods for analyzing sampling patterns from Section 8.2 are in agreement: the IndependentSampler is a terrible sampler. Independent uniform samples contain all frequencies equally (they are the definition of white noise), so they do not push aliasing out to higher frequencies. The discrepancy of uniform random samples is 1—the worst possible. This sampler's only saving grace comes when integrating a function with significant high-frequency energy (relative to the sampling rate), where it does about as well as any more sophisticated sampler.

> 8.2 节分析采样模式的所有方法都一致认为：IndependentSampler 是一个糟糕的采样器。独立均匀采样在所有频率上能量相等（它们是白噪声的定义），因此不会将走样推向高频。均匀随机采样的偏差为 1——最差的可能值。该采样器唯一的优点出现在积分具有显著高频能量（相对于采样率）的函数时，此时它与任何更复杂的采样器表现相当。
