## 8.8 Image Reconstruction

As discussed in Section 5.4.3, each pixel in the Film computes an estimate of the integral of a filter function with image samples. Sampling theory provides a mathematical foundation for this filtering: (1) reconstruct a continuous image function from samples, (2) prefilter to remove frequencies past the Nyquist limit, (3) sample at pixel locations. These two steps can be combined into a single filter function since we only need values at pixel locations.

> 如 5.4.3 节讨论的，Film 中每个像素计算滤波函数与图像采样的积分估计。采样理论为此滤波提供了数学基础：(1) 从采样重建连续图像函数，(2) 预滤波去除超过奈奎斯特限的频率，(3) 在像素位置采样。由于我们只需要像素位置的值，前两步可以合并为单一滤波函数。

The ideal sinc filter is not appropriate here: it is prone to ringing (Gibbs phenomenon) when frequencies exceed the Nyquist limit, and it has infinite support requiring all image samples for each output pixel. In practice, there is no single best filter—choosing requires a mixture of quantitative evaluation and qualitative judgment. pbrt therefore provides a variety of choices.

> 理想的 sinc 滤波器在此不适用：当频率超过奈奎斯特限时容易振铃（吉布斯现象），且其具有无限支撑，每个输出像素需要所有图像采样。实际上没有单一的最佳滤波器——选择需要混合定量评估和定性判断。因此 pbrt 提供了多种选择。

### 8.8.1 Filter Interface

The Filter class defines the interface for pixel reconstruction filters. All filters are 2D functions centered at the origin with a radius beyond which they have value 0 (the support is twice the radius). Key methods:

> Filter 类定义了像素重建滤波器的接口。所有滤波器都是以原点为中心的二维函数，超出半径的值为 0（支撑为半径的两倍）。关键方法：

- **`Radius()`**: Returns the filter's radius in x and y.
- **`Evaluate(p)`**: Evaluates the filter function at point p. May be called with points outside the radius (must return 0). Not required to integrate to 1 since the pixel estimator is self-normalizing.
- **`Integral()`**: Returns the filter's integral (most compute this in closed form).
- **`Sample(u)`**: Importance sampling method taking a random sample u in [0,1)². Returns a `FilterSample` with the sampled position p and a weight (ratio of filter value to PDF).

> - **`Radius()`**：返回滤波器的 x 和 y 半径。
> - **`Evaluate(p)`**：在点 p 处评估滤波函数。可能以半径外的点调用（必须返回 0）。不要求积分为 1，因为像素估计量是自归一化的。
> - **`Integral()`**：返回滤波器的积分（大多数以闭合形式计算）。
> - **`Sample(u)`**：重要性采样方法，接受 [0,1)² 中的随机样本 u。返回包含采样位置 p 和权重（滤波值与 PDF 之比）的 `FilterSample`。

The `GetCameraSample()` template function uses the filter's `Sample()` method to compute CameraSamples. It is templated on the sampler type for performance—if a concrete type (e.g., HaltonSampler) is passed, methods are inlined, which improves GPU rendering performance. The image sample position is found by adding the filter's offset to the pixel coordinate plus a 0.5 shift for discrete-to-continuous pixel mapping.

> `GetCameraSample()` 模板函数使用滤波器的 `Sample()` 方法计算 CameraSample。它对采样器类型进行模板化以提高性能——如果传递具体类型（如 HaltonSampler），方法会被内联，提升 GPU 渲染性能。图像采样位置通过将滤波器偏移加上像素坐标再加 0.5（用于离散到连续像素映射）来获得。

### 8.8.2 FilterSampler

Not all Filters can easily sample from their distributions, so the FilterSampler class wraps up sampling based on a tabularized representation. It evaluates the filter at 32 samples per unit extent in each dimension, creating a 2D table. A `PiecewiseConstant2D` distribution is used for importance sampling.

> 不是所有滤波器都能轻松从其分布中采样，因此 FilterSampler 类封装了基于表格化表示的采样。它在每个维度每单位范围内以 32 个样本评估滤波器，创建二维表。使用 `PiecewiseConstant2D` 分布进行重要性采样。

Two important implementation details: (1) The tabularized filter values (not `Filter::Evaluate()`) are used for the returned weight, ensuring constant weight for each cell and avoiding variance from the sampling distribution not being exactly proportional to the filter. (2) Integer coordinates from `PiecewiseConstant2D::Sample()` are used to index the table, avoiding floating-point round-off errors that would cause the ratio f/pdf to not be exactly 1.

> 两个重要的实现细节：(1) 返回权重使用表格化滤波值（而非 `Filter::Evaluate()`），确保每个单元格的权重恒定，避免采样分布与滤波器不完全成比例时产生的方差。(2) 使用 `PiecewiseConstant2D::Sample()` 的整数坐标索引表格，避免浮点舍入误差导致 f/pdf 比值不精确为 1。

### 8.8.3 Box Filter

The box filter equally weights all samples within a square region. It is the *de facto* result when filtering is not addressed explicitly. Although computationally efficient, it is about the worst filter possible—it allows high-frequency sample data to leak into reconstructed values (postaliasing). It does reasonably well with step functions but extremely poorly with smooth sinusoidal functions.

> Box 滤波器对正方形区域内所有采样赋予相等权重。当没有明确处理滤波时，它是事实上的默认结果。虽然计算效率高，但它几乎是最差的滤波器——允许高频采样数据泄漏到重建值中（后走样）。对阶跃函数表现尚可，但对光滑正弦函数表现极差。

Evaluation checks if the point is inside the box. Sampling uses linear interpolation of u within the filter extent, always returning weight 1. The integral equals the filter's area (2r_x × 2r_y).

> 评估检查点是否在盒子内。采样使用 u 在滤波器范围内的线性插值，权重始终为 1。积分等于滤波器面积 (2r_x × 2r_y)。

### 8.8.4 Triangle Filter

The triangle filter gives slightly better results than the box: weight falls off linearly from the center over the square extent. Evaluation is the product of two linear functions going to 0 at the filter width in both directions. Sampling uses the `SampleTent()` utility function (separable in each dimension), with weight always 1. The integral is r_x² × r_y².

> 三角滤波器比 box 略好：权重从中心向正方形范围线性衰减。评估是两个在各方向滤波器宽度处衰减到 0 的线性函数之积。采样使用 `SampleTent()` 工具函数（每个维度可分离），权重始终为 1。积分为 r_x² × r_y²。

### 8.8.5 Gaussian Filter

The Gaussian filter gives a reasonably good result in practice. It tends to cause slight blurring but helps mask remaining aliasing. It is pbrt's default filter.

> 高斯滤波器在实践中给出相当好的结果。它倾向于产生轻微模糊，但有助于掩盖剩余走样。它是 pbrt 的默认滤波器。

The filter function (Equation 8.26) subtracts the Gaussian value at the radius endpoint to ensure the filter reaches 0 at its boundary. The integral uses the error function via `GaussianIntegral()`. Larger σ causes slower falloff and more blurring. Because the offset term prevents exact sampling via the inverse error function, a FilterSampler is used.

> 滤波函数（公式 8.26）减去半径端点处的高斯值，确保滤波器在边界处达到 0。积分通过 `GaussianIntegral()` 使用误差函数。较大的 σ 导致较慢的衰减和更多模糊。由于偏移项阻止通过逆误差函数精确采样，使用 FilterSampler。

### 8.8.6 Mitchell Filter

Mitchell and Netravali (1988) developed a parameterized family of filters to systematically explore the filter design space. After analyzing test subjects' subjective responses, they developed a filter that trades off between *ringing* (phantom edges) and *blurring*—two common artifacts from poor reconstruction.

> Mitchell 和 Netravali (1988) 开发了一个参数化滤波器族，以系统地探索滤波器设计空间。分析测试对象的主观反应后，他们开发了一个在**振铃**（幽灵边缘）和**模糊**之间权衡的滤波器——两种常见的重建伪影。

The filter has *negative lobes* at its edges that improve edge sharpness (reduced blurring), but if too large cause ringing. Because pixel values can become negative, they eventually need clamping. The filter is parameterized by B and C (recommended to lie along B + 2C = 1). It is the product of separable 1D cubic polynomials with C⁰ and C¹ continuity at key points. Its integral (r_x × r_y / 4) is independent of B and C. A FilterSampler is used for sampling since inverting a cubic polynomial requires solving a quartic.

> 滤波器在边缘具有**负瓣**，改善边缘锐度（减少模糊），但如果太大会导致振铃。因为像素值可能变负，最终需要钳位。滤波器由 B 和 C 参数化（推荐沿 B + 2C = 1 线取值）。它是可分离的一维三次多项式之积，在关键点处具有 C⁰ 和 C¹ 连续性。其积分 (r_x × r_y / 4) 与 B 和 C 无关。由于反转三次多项式需要求解四次方程，使用 FilterSampler 进行采样。

### 8.8.7 Windowed Sinc Filter

The LanczosSincFilter implements a filter based on the sinc function multiplied by a windowing function that goes to 0 after τ cycles. The *Lanczos window* is the central lobe of the sinc scaled to cover the τ cycles (Equation 8.27).

> LanczosSincFilter 实现了一个基于 sinc 函数的滤波器，乘以在 τ 个周期后归零的窗口函数。**Lanczos 窗口**是 sinc 的中心瓣缩放到覆盖 τ 个周期（公式 8.27）。

The windowed sinc shows much less ringing than the infinite sinc for step functions, and does well with sinusoidal functions until prealiasing begins. The `WindowedSinc()` utility function uses `Sinc()` which is implemented via `SinXOverX()` for numerical robustness. A FilterSampler is used for sampling, and the integral is approximated by a Riemann sum (no closed form).

> 加窗 sinc 对阶跃函数的振铃比无限 sinc 少很多，对正弦函数表现良好直到预走样开始。`WindowedSinc()` 工具函数使用 `Sinc()`，后者通过 `SinXOverX()` 实现以保证数值稳健性。使用 FilterSampler 进行采样，积分通过黎曼和近似（没有闭合形式）。
