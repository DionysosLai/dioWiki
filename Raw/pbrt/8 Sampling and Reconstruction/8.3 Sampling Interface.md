## 8.3 Sampling Interface

pbrt's Sampler interface makes it possible to use a variety of sample generation algorithms for rendering. The sample points are used by Integrators in many ways: determining points on the image plane for camera rays, selecting which light source to trace a shadow ray to, etc. The benefits of carefully crafted sampling patterns are not just theoretical—they can substantially improve image quality with relatively small runtime expense.

> pbrt 的 Sampler 接口使得可以在渲染中使用多种采样生成算法。采样点被 Integrator 以多种方式使用：确定图像平面上产生相机光线的点、选择向哪个光源追踪阴影光线等。精心设计的采样模式的好处不仅是理论上的——它们能以相对较小的运行时开销大幅提升图像质量。

The task of a Sampler is to generate uniform d-dimensional sample points, where each coordinate's value is in [0, 1). The total number of dimensions is not set ahead of time; Samplers must generate additional dimensions on demand, depending on the calculations performed by the light transport algorithms.

> Sampler 的任务是生成均匀的 d 维采样点，每个坐标值在 [0, 1) 范围内。维度总数不是预先设定的；Sampler 必须按需生成额外维度，具体取决于光传输算法执行的计算。

Key interface methods:

> 关键接口方法：

- **`SamplesPerPixel()`**: Returns the number of samples to be taken in each pixel.
- **`StartPixelSample(p, sampleIndex, dimension)`**: Called when an Integrator starts work on a given pixel sample. Serves two purposes: (1) some Samplers use pixel knowledge to improve sample distribution across adjacent pixels, (2) puts the sampler in a deterministic state for debugging reproducibility.
- **`Get1D()` / `Get2D()`**: Request 1 or 2 dimensions of the sample point. Some samplers generate better distributions when they know two dimensions will be used together. The interface does not support 3D+ requests.
- **`GetPixel2D()`**: Retrieves the 2D sample for the film plane position. Some samplers handle these dimensions differently from other 2D samples.
- **`Clone(alloc)`**: Returns a copy of the Sampler, since Samplers store per-sample state and are not thread-safe.

> - **`SamplesPerPixel()`**：返回每个像素要取的样本数。
> - **`StartPixelSample(p, sampleIndex, dimension)`**：当 Integrator 开始处理给定像素样本时调用。有两个目的：(1) 某些 Sampler 利用像素信息改善相邻像素间的采样分布，(2) 将采样器置于确定性状态以保证调试可重现性。
> - **`Get1D()` / `Get2D()`**：请求采样点的 1 或 2 个维度。某些采样器在知道两个维度将一起使用时可以生成更好的分布。接口不支持 3D+ 请求。
> - **`GetPixel2D()`**：获取胶片平面位置的 2D 采样。某些采样器对这些维度的处理方式与其他 2D 采样不同。
> - **`Clone(alloc)`**：返回 Sampler 的副本，因为 Sampler 存储逐样本状态，不是线程安全的。

The constant `OneMinusEpsilon` represents the largest representable floating-point value less than 1, used by Sampler implementations to clamp sample values.

> 常量 `OneMinusEpsilon` 表示小于 1 的最大可表示浮点值，Sampler 实现用它来钳位采样值。

A sharp edge of these interfaces: code that uses sample values must be carefully written to always request sample dimensions in the same order. If conditional code paths consume different numbers of dimensions, some function may receive different sample dimensions across different paths, thwarting the sampler's efforts to provide well-distributed points.

> 这些接口的一个棘手之处：使用采样值的代码必须仔细编写，始终以相同顺序请求采样维度。如果条件代码路径消耗不同数量的维度，某些函数可能在不同路径中接收不同的采样维度，破坏采样器提供良好分布点的努力。
