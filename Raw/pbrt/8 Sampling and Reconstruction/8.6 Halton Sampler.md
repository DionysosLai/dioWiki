## 8.6 Halton Sampler

This section introduces the HaltonSampler, based on algorithms that directly generate low-discrepancy sample points that are simultaneously well distributed over all dimensions—not just one or two at a time as the StratifiedSampler did.

> 本节介绍 HaltonSampler，它基于直接生成低偏差采样点的算法，这些点在所有维度上同时具有良好分布——而不是像 StratifiedSampler 那样一次只处理一两个维度。

### 8.6.1 Hammersley and Halton Points

Hammersley and Halton points are constructed using the *radical inverse*. A positive integer value a can be expressed in a base b with a sequence of digits (Equation 8.18). The radical inverse function Φ_b converts a nonnegative integer to a fractional value in [0,1) by reflecting these digits about the radix point (Equation 8.19).

> Hammersley 和 Halton 点使用**基数反转**构造。正整数值 a 可以用基数 b 的一系列数字表示（公式 8.18）。基数反转函数 Φ_b 通过将这些数字关于小数点反射，将非负整数转换为 [0,1) 中的小数值（公式 8.19）。

The simplest low-discrepancy sequence is the *van der Corput sequence*: Φ_2(0), Φ_2(1), Φ_2(2), ... It recursively splits the intervals of the 1D line in half, generating a sample point at the center of each interval. Its discrepancy is O(log N / N), which is optimal.

> 最简单的低偏差序列是 **van der Corput 序列**：Φ_2(0), Φ_2(1), Φ_2(2), ... 它递归地将一维线段的区间一分为二，在每个区间中心生成采样点。其偏差为 O(log N / N)，是最优的。

The d-dimensional *Halton sequence* uses the radical inverse with a different prime base for each dimension: x_a = (Φ_2(a), Φ_3(a), Φ_5(a), ..., Φ_{p_d}(a)). All prefixes are well distributed, so low discrepancy is maintained as samples are added. Its discrepancy is O((log N)^d / N), which is asymptotically optimal. If the total number is fixed, the *Hammersley point set* can be used for slightly lower discrepancy, where the first dimension is simply a/N.

> d 维 **Halton 序列**对每个维度使用不同素数基的基数反转：x_a = (Φ_2(a), Φ_3(a), Φ_5(a), ..., Φ_{p_d}(a))。所有前缀都具有良好分布，因此添加样本时保持低偏差。其偏差为 O((log N)^d / N)，渐近最优。如果总数固定，可使用 **Hammersley 点集**获得稍低的偏差，其第一维度简单地为 a/N。

The `RadicalInverse()` function computes the radical inverse by extracting digits via integer division, reversing them, and multiplying by b^{-n}. The `InverseRadicalInverse()` function computes the inverse mapping given the reversed digits and number of digits.

> `RadicalInverse()` 函数通过整数除法提取数字、反转它们并乘以 b^{-n} 来计算基数反转。`InverseRadicalInverse()` 函数在给定反转后的数字和数字位数时计算逆映射。

### 8.6.2 Randomization via Scrambling

One disadvantage of fully deterministic Hammersley/Halton sequences is that variance estimation is impossible. Furthermore, as the base increases, lower-dimensional projections exhibit regular patterns that can cause visible error. *Scrambling* addresses both issues by randomizing the digits of each sample coordinate with random permutations. Over all permutations, each coordinate value is uniformly distributed over [0,1).

> 完全确定性的 Hammersley/Halton 序列的一个缺点是无法估计方差。此外，随着基数增加，低维投影呈现规则模式，可能导致可见误差。**置乱**通过用随机排列随机化每个采样坐标的数字来解决这两个问题。在所有排列上，每个坐标值在 [0,1) 上均匀分布。

The *scrambled radical inverse* (Equation 8.20) applies a permutation π to each digit, where each digit has a distinct permutation. Important subtleties: (1) zero trailing digits must continue to be processed, (2) each digit must have its own permutation—otherwise scrambled values would not be uniformly distributed.

> **置乱基数反转**（公式 8.20）对每个数字应用排列 π，每个数字有不同的排列。重要的细节：(1) 尾随零数字必须继续处理，(2) 每个数字必须有自己的排列——否则置乱值不会均匀分布。

The `DigitPermutation` class manages permutations for a single base. It determines how many digits need handling (based on floating-point precision limits) and generates random permutations for each digit index using `PermutationElement()`. Permutations are stored in a flat array of uint16_t (max base 65,536; pbrt supports up to 1,000 dimensions → max prime 7,919).

> `DigitPermutation` 类管理单一基数的排列。它确定需要处理多少位数字（基于浮点精度限制），并使用 `PermutationElement()` 为每个数字索引生成随机排列。排列存储在 uint16_t 的平坦数组中（最大基数 65,536；pbrt 支持最多 1,000 维 → 最大素数 7,919）。

*Owen scrambling* defines digit permutations that depend not only on the current digit's index but also on the values of previous digits. Remarkably, for a class of smooth functions, the integration error decreases at rate O(N^{-3/2}(log N)^{(d-1)/2}), a substantial improvement over regular Monte Carlo. Owen scrambling breaks up shared structure among all samples (especially the trailing zero digits). The `OwenScrambledRadicalInverse()` function implements this by hashing the previously accumulated digits (`MixBits(hash ^ reversedDigits)`) to seed the permutation for each digit.

> **Owen 置乱**定义的数字排列不仅取决于当前数字的索引，还取决于前面数字的值。令人惊叹的是，对于一类光滑函数，积分误差以 O(N^{-3/2}(log N)^{(d-1)/2}) 的速率下降，比常规蒙特卡洛有实质性改进。Owen 置乱打破了所有样本间共享的结构（特别是尾随零数字）。`OwenScrambledRadicalInverse()` 函数通过对先前累积的数字进行哈希 (`MixBits(hash ^ reversedDigits)`) 来为每个数字的排列提供种子。

### 8.6.3 Halton Sampler Implementation

The HaltonSampler scales the domain of the first two Halton dimensions so it covers an integral number of pixels, ensuring adjacent pixels' samples are well distributed with respect to each other. It finds the smallest power-of-base (2 or 3) scale factor larger than the image resolution (capped at MaxHaltonResolution = 128). For larger images, a tile of Halton points repeats across the image.

> HaltonSampler 缩放前两个 Halton 维度的域使其覆盖整数个像素，确保相邻像素的样本彼此之间分布良好。它找到大于图像分辨率（上限为 MaxHaltonResolution = 128）的最小基数幂（2 或 3）缩放因子。对于更大的图像，Halton 点的贴片在图像上重复。

The scaling by b^j means the first j digits of the radical inverse determine which pixel a sample lands in, while remaining digits give the fractional offset within the pixel. `StartPixelSample()` computes the Halton index for the requested pixel by solving a system of equations (Equation 8.21) using the inverse radical inverse and multiplicative inverses (computed via the extended GCD algorithm). The sample stride is baseScales[0] × baseScales[1]—each pixel is visited once every this many samples.

> 通过 b^j 缩放意味着基数反转的前 j 个数字确定样本落在哪个像素，其余数字给出像素内的小数偏移。`StartPixelSample()` 通过使用逆基数反转和乘法逆元（通过扩展 GCD 算法计算）求解方程组（公式 8.21）来计算请求像素的 Halton 索引。样本步幅为 baseScales[0] × baseScales[1]——每隔这么多样本访问每个像素一次。

The `SampleDimension()` method calls the appropriate radical inverse function (plain, scrambled, or Owen-scrambled) based on the selected randomization strategy. `GetPixel2D()` always uses unscrambled `RadicalInverse()` (since haltonIndex computation doesn't account for scrambling) and strips the pixel-selecting digits by right-shifting (base 2) or dividing (base 3).

> `SampleDimension()` 方法根据选定的随机化策略调用适当的基数反转函数（普通、置乱或 Owen 置乱）。`GetPixel2D()` 始终使用未置乱的 `RadicalInverse()`（因为 haltonIndex 计算不考虑置乱），并通过右移（基数 2）或除法（基数 3）去除选择像素的数字。

### 8.6.4 Evaluation

Power spectra analysis reveals key characteristics: all three randomization strategies have low energy along the two axes, indicating good performance with functions that vary mostly in one dimension. This is because each dimension uses an independent radical inverse, so 1D projections of all N sample points are stratified (unlike jittered sampling where only √N are stratified along each axis).

> 功率谱分析揭示了关键特征：所有三种随机化策略沿两个轴的能量都很低，表明对主要在一个维度变化的函数性能良好。这是因为每个维度使用独立的基数反转，所以所有 N 个采样点的一维投影都是分层的（不像抖动采样只有 √N 个沿每个轴分层）。

However, the non-randomized Halton sampler has wide PSD variation at higher frequencies (some frequencies 100× more than others), causing unpleasant patterns in rendered images. Random digit permutations substantially improve the PSD but still have structure. Owen scrambling gives a remarkably uniform power spectrum at higher frequencies while maintaining low power along the axes.

> 然而，非随机化的 Halton 采样器在高频处的 PSD 变化很大（某些频率比其他频率高 100 倍），导致渲染图像中出现不悦目的模式。随机数字排列大幅改善了 PSD 但仍有结构。Owen 置乱在高频处给出了非常均匀的功率谱，同时保持沿轴的低功率。

For simple 2D test functions (Gaussian and rotated checkerboard), the Halton sampler achieves higher asymptotic convergence rate than both stratified and independent sampling, with Owen-scrambled Halton achieving nearly 2000× lower MSE than independent sampling at 4,096 samples. For a complex scene with tens of integration dimensions, the Halton sampler gives roughly 30% lower MSE than independent sampling at 4,096 spp.

> 对于简单二维测试函数（高斯和旋转棋盘），Halton 采样器比分层和独立采样都实现了更高的渐近收敛率，Owen 置乱的 Halton 在 4,096 个样本时比独立采样 MSE 低近 2000 倍。对于具有数十个积分维度的复杂场景，Halton 采样器在 4,096 spp 时比独立采样 MSE 低约 30%。
