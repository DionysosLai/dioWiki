## 7.1 Primitive Interface and Geometric Primitives

The Primitive class defines the Primitive interface. It and the Primitive implementations that are described in this section are defined in the files cpu/primitive.h and cpu/primitive.cpp.

> Primitive 类定义了 Primitive 接口。它和本节描述的 Primitive 实现都定义在 cpu/primitive.h 和 cpu/primitive.cpp 文件中。

Primitive is a TaggedPointer that can hold SimplePrimitive, GeometricPrimitive, TransformedPrimitive, AnimatedPrimitive, BVHAggregate, or KdTreeAggregate.

> Primitive 是一个 TaggedPointer，可以持有 SimplePrimitive、GeometricPrimitive、TransformedPrimitive、AnimatedPrimitive、BVHAggregate 或 KdTreeAggregate。

The Primitive interface is composed of only three methods, each of which corresponds to a Shape method. The first, `Bounds()`, returns a bounding box that encloses the primitive's geometry in rendering space. There are many uses for such a bound; one of the most important is to place the Primitive in the acceleration data structures. The other two methods provide the two types of ray intersection tests.

> Primitive 接口仅由三个方法组成，每个都对应一个 Shape 方法。第一个 `Bounds()` 返回一个在渲染空间中包围图元几何体的包围盒，其最重要的用途之一是将 Primitive 放入加速数据结构中。另外两个方法提供两种光线相交测试。

Upon finding an intersection, a Primitive's `Intersect()` method is also responsible for initializing a few member variables in the SurfaceInteraction in the ShapeIntersection that it returns. The first two are representations of the shape's material and its emissive properties, if it is itself an emitter. For convenience, SurfaceInteraction provides a method `SetIntersectionProperties()` to set these, which reduces the risk of inadvertently not setting all of them. The second two are related to medium scattering properties and will be described in Section 11.4.

> 找到交点后，Primitive 的 `Intersect()` 方法还负责初始化返回的 ShapeIntersection 中 SurfaceInteraction 的几个成员变量。前两个是形状的材质和发光属性（如果它本身是发射体）的表示。为方便起见，SurfaceInteraction 提供了 `SetIntersectionProperties()` 方法来设置这些，以减少遗漏的风险。后两个与介质散射属性相关，将在 11.4 节中描述。

### 7.1.1 Geometric Primitives

The GeometricPrimitive class provides a basic implementation of the Primitive interface that stores a variety of properties that may be associated with a shape.

> GeometricPrimitive 类提供了 Primitive 接口的基本实现，存储可能与形状关联的各种属性。

Each GeometricPrimitive holds a Shape with a description of its appearance properties, including its material, its emissive properties if it is a light source, the participating media on each side of its surface, and an optional *alpha texture*, which can be used to make some parts of a shape's surface disappear.

> 每个 GeometricPrimitive 持有一个 Shape 及其外观属性描述，包括材质、作为光源时的发光属性、表面两侧的参与介质，以及一个可选的 **alpha 纹理**——可用于使形状表面的某些部分消失。

GeometricPrimitive's `Intersect()` calls the `Intersect()` method of its Shape to do the actual intersection test and to initialize a ShapeIntersection to describe the intersection, if any. If an intersection is found, then additional processing specific to the GeometricPrimitive is performed.

> GeometricPrimitive 的 `Intersect()` 调用其 Shape 的 `Intersect()` 方法来执行实际的相交测试并初始化 ShapeIntersection。如果找到交点，则执行 GeometricPrimitive 特有的额外处理。

If an alpha texture is associated with the shape, then the intersection point is tested against the alpha texture before a successful intersection is reported. The alpha texture can be thought of as a scalar function over the shape's surface that indicates whether the surface is actually present at each point. An alpha value of 0 indicates that it is not, and 1 that it is. Alpha textures are useful for representing objects like leaves: a leaf might be modeled as a single triangle or bilinear patch, with an alpha texture cutting out the edges so that a detailed outline of a leaf remains.

> 如果形状关联了 alpha 纹理，则在报告成功交点之前，会对交点进行 alpha 纹理测试。alpha 纹理可视为形状表面上的标量函数，指示每个点处表面是否实际存在。alpha 值为 0 表示不存在，1 表示存在。alpha 纹理对于表示树叶等物体很有用：一片树叶可以建模为单个三角形或双线性面片，用 alpha 纹理切除边缘以留下详细的叶片轮廓。

If the alpha texture has a value of 0 or 1 at the intersection point, then it is easy to decide whether or not the intersection reported by the shape is valid. For intermediate alpha values, the correct answer is less clear. One possibility would be to use a fixed threshold; however, this approach leads to hard transitions at the resulting boundary. Another option would be to return the alpha from the intersection method and leave calling code to handle it; however, that would make the Primitive intersection interfaces more complex and place a new burden on integrators.

> 如果 alpha 纹理在交点处的值为 0 或 1，则容易判断形状报告的交点是否有效。对于中间的 alpha 值，正确答案不那么明确。一种方案是使用固定阈值，但这会导致边界处的硬过渡。另一种方案是从相交方法返回 alpha 值并交由调用代码处理，但这会使 Primitive 相交接口更复杂并给积分器增加新负担。

A *stochastic alpha test* addresses these issues. With it, intersections with the shape are randomly reported with probability proportional to the value of the alpha texture. This approach is easy to implement, gives the expected results for an alpha of 0 or 1, and with a sufficient number of samples gives a better result than using a fixed threshold.

> **随机 alpha 测试**解决了这些问题。通过它，与形状的交点以与 alpha 纹理值成比例的概率被随机报告。这种方法易于实现，在 alpha 为 0 或 1 时给出预期结果，且在足够多的样本下给出比固定阈值更好的结果。

One challenge in performing the stochastic alpha test is generating a uniform random number to apply it. For a given ray and shape, we would like this number to be the same across multiple runs of the system; doing so is a part of making the set of computations performed by pbrt be deterministic, which is a great help for debugging. However, it is important that different random numbers be used for different rays. The `HashFloat()` utility function provides a solution: here it is used to compute a random floating-point value between 0 and 1 for the alpha test; this value is determined by the ray's origin and direction.

> 执行随机 alpha 测试的一个挑战是生成用于应用它的均匀随机数。对于给定的光线和形状，我们希望这个数在系统多次运行中保持一致；这是使 pbrt 的计算具有确定性的一部分，对调试很有帮助。然而，不同光线使用不同随机数很重要。`HashFloat()` 工具函数提供了解决方案：这里它用于计算 alpha 测试的 0 到 1 之间的随机浮点值；该值由光线的原点和方向决定。

If the alpha test indicates that the intersection should be ignored, then another intersection test is performed with the current GeometricPrimitive, with a recursive call to `Intersect()`. This additional test is important for shapes like spheres, where we may reject the closest intersection but then intersect the shape again further along the ray. This recursive call requires adjustment of the tMax value to account for the distance along the ray to the initial alpha tested intersection point. Then, if it reports an intersection, the reported tHit value should account for that segment as well.

> 如果 alpha 测试指示应忽略该交点，则通过递归调用 `Intersect()` 对当前 GeometricPrimitive 执行另一次相交测试。对于球体等形状，这种额外测试很重要，因为我们可能拒绝最近的交点但在光线更远处再次与形状相交。这个递归调用需要调整 tMax 值以考虑到光线到初始 alpha 测试交点的距离。如果它报告了一个交点，报告的 tHit 值也应考虑该段距离。

The `IntersectP()` method must also handle the case of the GeometricPrimitive having an alpha texture associated with it. In that case, a full intersection test is performed. In the more common case of no alpha texture, `Shape::IntersectP()` can be called directly.

> `IntersectP()` 方法也必须处理 GeometricPrimitive 关联了 alpha 纹理的情况。在这种情况下，执行完整的相交测试。在更常见的没有 alpha 纹理的情况下，可以直接调用 `Shape::IntersectP()`。

Most objects in a scene are neither emissive nor have alpha textures. Further, only a few of them typically represent the boundary between two different types of participating media. It is wasteful to store nullptr values for the corresponding member variables of GeometricPrimitive in that common case. Therefore, pbrt also provides SimplePrimitive, which also implements the Primitive interface but does not store those values.

> 场景中的大多数物体既不发光也没有 alpha 纹理。此外，通常只有少数物体表示两种不同参与介质之间的边界。在常见情况下为 GeometricPrimitive 的相应成员变量存储 nullptr 值是浪费的。因此，pbrt 还提供了 SimplePrimitive，它同样实现 Primitive 接口但不存储这些值。

Because SimplePrimitive only stores a shape and a material, it saves 32 bytes of memory. For scenes with millions of primitives, the overall savings can be meaningful.

> 由于 SimplePrimitive 仅存储 shape 和 material，它节省了 32 字节内存。对于包含数百万图元的场景，总体节省可能非常可观。

### 7.1.2 Object Instancing and Primitives in Motion

Object instancing is a classic technique in rendering that reuses transformed copies of a single collection of geometry at multiple positions in a scene. For example, in a model of a concert hall with thousands of identical seats, the scene description can be compressed substantially if all the seats refer to a shared geometric representation of a single seat. The ecosystem scene in Figure 7.2 has 23,241 individual plants of various types, although only 31 unique plant models. Because each plant model is instanced multiple times with a different transformation for each instance, the complete scene has a total of 3.1 billion triangles. However, only 24 million triangles are stored in memory thanks to primitive reuse through object instancing. pbrt uses just over 4 GB of memory when rendering this scene with object instancing, but would need upward of 516 GB to render it without instancing.

> 物体实例化是渲染中的经典技术，在场景中的多个位置重用单个几何体集合的变换副本。例如，在拥有数千个相同座位的音乐厅模型中，如果所有座位都引用一个共享的单个座位几何表示，场景描述可以大幅压缩。图 7.2 的生态系统场景有 23,241 棵各类植物，但只有 31 个独特的植物模型。由于每个植物模型被多次实例化（每个实例有不同的变换），完整场景共有 31 亿个三角形。然而，由于通过物体实例化的图元重用，内存中只存储了 2400 万个三角形。pbrt 使用物体实例化渲染此场景时仅使用略超 4 GB 的内存，但不使用实例化时需要超过 516 GB。

The TransformedPrimitive implementation of the Primitive interface makes object instancing possible in pbrt. Rather than holding a shape, it stores a single Primitive as well as a Transform that is injected in between the underlying primitive and its representation in the scene. This extra transformation enables object instancing.

> TransformedPrimitive 对 Primitive 接口的实现使 pbrt 中的物体实例化成为可能。它不持有形状，而是存储一个 Primitive 和一个 Transform，该变换被注入到底层图元与其在场景中的表示之间。这个额外的变换启用了物体实例化。

Recall that the Shapes of Chapter 6 themselves had rendering from object space transformations applied to them to place them in the scene. If a shape is held by a TransformedPrimitive, then the shape's notion of rendering space is not the actual scene rendering space—only after the TransformedPrimitive's transformation is also applied is the shape actually in rendering space. For instanced primitives, letting Shapes know all the instance transforms is of limited utility: we would not want the TriangleMesh to make a copy of its vertex positions for each instance transformation and transform them all the way to rendering space, since this would negate the memory savings of object instancing.

> 回忆第 6 章的 Shape 本身已经应用了从物体空间到渲染空间的变换来将它们放置在场景中。如果一个形状被 TransformedPrimitive 持有，那么该形状的"渲染空间"概念并非实际的场景渲染空间——只有在 TransformedPrimitive 的变换也被应用之后，形状才真正处于渲染空间中。对于实例化图元，让 Shape 知道所有实例变换的用处有限：我们不希望 TriangleMesh 为每个实例变换复制其顶点位置并将它们全部变换到渲染空间，因为这将抵消物体实例化带来的内存节省。

The key task of TransformedPrimitive is to bridge between the Primitive interface that it implements and the Primitive that it holds, accounting for the effects of the rendering from primitive space transformation. The `Intersect()` method first transforms the given ray to the primitive's coordinate system via the inverse of `renderFromPrimitive`, passes the transformed ray to the held primitive's `Intersect()` routine. If an intersection is found, the SurfaceInteraction needs to be transformed back to rendering space. Note that any returned `ShapeIntersection::tHit` value from the primitive can be returned to the caller as is.

> TransformedPrimitive 的关键任务是在它实现的 Primitive 接口和它持有的 Primitive 之间进行桥接，考虑从图元空间到渲染空间变换的效果。`Intersect()` 方法首先通过 `renderFromPrimitive` 的逆变换将给定光线变换到图元的坐标系，然后将变换后的光线传递给持有的图元的 `Intersect()` 例程。如果找到交点，SurfaceInteraction 需要被变换回渲染空间。注意，图元返回的任何 `ShapeIntersection::tHit` 值可以原样返回给调用者。

The AnimatedPrimitive class uses an AnimatedTransform in place of the Transform stored by TransformedPrimitives. It thus enables rigid-body animation of primitives in the scene.

> AnimatedPrimitive 类使用 AnimatedTransform 替代 TransformedPrimitive 中存储的 Transform。因此它支持场景中图元的刚体动画。

The AnimatedTransform class uses substantially more memory than Transform. On the system used to develop pbrt, the former uses 696 bytes of memory, while the latter uses 128. Thus, just as was the case with GeometricPrimitive and SimplePrimitive, it is worthwhile to only use AnimatedPrimitive for shapes that actually are animated. Making this distinction is the task of the code that constructs the scene specification used for rendering.

> AnimatedTransform 类使用的内存比 Transform 多得多。在开发 pbrt 的系统上，前者使用 696 字节内存，后者使用 128 字节。因此，正如 GeometricPrimitive 和 SimplePrimitive 的情况一样，仅对实际有动画的形状使用 AnimatedPrimitive 是值得的。做这种区分是构建用于渲染的场景规范的代码的任务。

A bounding box of the primitive over the frame's time range is found via the `AnimatedTransform::MotionBounds()` method. The rest of the implementations of the AnimatedPrimitive intersection methods parallel those of TransformedPrimitive, just using an AnimatedTransform.

> 通过 `AnimatedTransform::MotionBounds()` 方法找到图元在帧时间范围内的包围盒。AnimatedPrimitive 相交方法的其余实现与 TransformedPrimitive 类似，只是使用了 AnimatedTransform。
