## 7.3 Bounding Volume Hierarchies

Bounding volume hierarchies (BVHs) are an approach for ray intersection acceleration based on primitive subdivision, where the primitives are partitioned into a hierarchy of disjoint sets. Primitives are stored in the leaves, and each node stores a bounding box of the primitives in the nodes beneath it. Thus, as a ray traverses through the tree, any time it does not intersect a node's bounds, the subtree beneath that node can be skipped.

> 包围体层次结构（BVH）是一种基于图元细分的光线相交加速方法，其中图元被划分为不相交集合的层次结构。图元存储在叶节点中，每个节点存储其下方节点中图元的包围盒。因此，当光线遍历树时，任何时候它不与节点的边界相交，该节点下方的子树就可以被跳过。

One property of primitive subdivision is that each primitive appears in the hierarchy only once. In contrast, a primitive may overlap multiple spatial regions with spatial subdivision and thus may be tested for intersection multiple times. Another implication is that the amount of memory needed to represent the primitive subdivision hierarchy is bounded. For a binary BVH that stores a single primitive in each leaf, the total number of nodes is 2n−1, where n is the number of primitives.

> 图元细分的一个性质是每个图元在层次结构中只出现一次。相比之下，在空间细分中一个图元可能与多个空间区域重叠，因而可能被多次测试相交。另一个推论是表示图元细分层次结构所需的内存量是有界的。对于在每个叶节点存储一个图元的二叉 BVH，节点总数为 2n−1，其中 n 是图元数量。

BVHs are more efficient to build than kd-trees, and are generally more numerically robust and less prone to missed intersections due to round-off errors. The BVH aggregate, BVHAggregate, is therefore the default acceleration structure in pbrt.

> BVH 的构建效率高于 kd-tree，且通常在数值上更鲁棒，更不容易因舍入误差而遗漏交点。因此 BVH 聚合体 BVHAggregate 是 pbrt 中默认的加速结构。

The BVHAggregate constructor takes an enumerator value that describes which of four algorithms to use when partitioning primitives to build the tree. The default, SAH, indicates that an algorithm based on the "surface area heuristic" should be used. An alternative, HLBVH, can be constructed more efficiently (and more easily parallelized), but it does not build trees that are as effective as SAH. The remaining two approaches, Middle and EqualCounts, use even less computation but create fairly low-quality trees.

> BVHAggregate 构造函数接受一个枚举值来描述构建树时用于划分图元的四种算法中的哪一种。默认的 SAH 指示使用基于"表面积启发式"的算法。替代方案 HLBVH 构建更高效（也更容易并行化），但构建的树不如 SAH 有效。其余两种方法 Middle 和 EqualCounts 使用更少的计算但生成质量相当低的树。

### 7.3.1 BVH Construction

There are three stages to BVH construction in the implementation here. First, bounding information about each primitive is computed and stored in an array that will be used during tree construction. Next, the tree is built using the algorithm choice encoded in splitMethod. The result is a binary tree where each interior node holds pointers to its children and each leaf node holds references to one or more primitives. Finally, this tree is converted to a more compact (and thus more efficient) pointerless representation for use during rendering.

> BVH 构建分三个阶段。首先，计算每个图元的包围信息并存储在数组中用于树构建。然后，使用 splitMethod 编码的算法选择来构建树。结果是一棵二叉树，每个内部节点持有指向其子节点的指针，每个叶节点持有一个或多个图元的引用。最后，这棵树被转换为更紧凑（因此更高效）的无指针表示用于渲染。

For each primitive to be stored in the BVH, an instance of the BVHPrimitive structure stores its complete bounding box and its index in the primitives array. A simple method makes the centroid of the bounding box available.

> 对于要存储在 BVH 中的每个图元，BVHPrimitive 结构的实例存储其完整包围盒及其在 primitives 数组中的索引。一个简单方法提供包围盒的质心。

An important side effect of the tree construction process is that a new array of Primitives is stored in orderedPrims; this array stores the primitives ordered so that the primitives in each leaf node occupy a contiguous range in the array. It is swapped with the original primitives array after tree construction.

> 树构建过程的一个重要副作用是在 orderedPrims 中存储了一个新的 Primitive 数组；该数组按顺序存储图元，使得每个叶节点中的图元占据数组中的一个连续范围。树构建完成后它与原始 primitives 数组交换。

Memory for the initial BVH is allocated using Allocators based on the C++ standard library's `pmr::monotonic_buffer_resource`, which efficiently allocates memory from larger buffers. This approach is not only more computationally efficient than using a general-purpose allocator, but also uses less memory in total. Because the `pmr::monotonic_buffer_resource` class cannot be used concurrently by multiple threads without mutual exclusion, each thread uses per-thread allocation via the ThreadLocal class.

> 初始 BVH 的内存使用基于 C++ 标准库的 `pmr::monotonic_buffer_resource` 的分配器进行分配，它从较大的缓冲区高效分配内存。这种方法不仅比使用通用分配器计算效率更高，而且总体内存使用更少。由于 `pmr::monotonic_buffer_resource` 类不能被多个线程并发使用而不需要互斥，每个线程通过 ThreadLocal 类使用逐线程分配。

Each BVHBuildNode represents a node of the BVH. All nodes store a Bounds3f that represents the bounds of all the children beneath the node. Each interior node stores pointers to its two children and records the coordinate axis along which primitives were partitioned for distribution to their two children; this information is used to improve the performance of the traversal algorithm. Leaf nodes record which primitives are stored in them via firstPrimOffset and nPrimitives. Leaf and interior nodes are distinguished by whether their child pointers are nullptr or not.

> 每个 BVHBuildNode 表示 BVH 的一个节点。所有节点存储一个 Bounds3f 表示该节点下方所有子节点的包围。每个内部节点存储指向其两个子节点的指针，并记录图元被划分以分配到两个子节点时所沿的坐标轴；此信息用于提高遍历算法的性能。叶节点通过 firstPrimOffset 和 nPrimitives 记录存储在其中的图元。叶节点和内部节点通过其子指针是否为 nullptr 来区分。

The InitInterior() method requires that the two child nodes already have been created, so that their pointers can be passed in. This makes it easy to compute the bounds of the interior node, since the children bounds are immediately available.

> InitInterior() 方法要求两个子节点已经创建，以便可以传入它们的指针。这使得计算内部节点的包围盒很容易，因为子节点的包围盒立即可用。

The `buildRecursive()` method takes a pointer `totalNodes` to track the total number of BVH nodes created (represented by `std::atomic` for thread safety), and `orderedPrimsOffset` that starts at 0 and keeps track of where the next free entry is in orderedPrims. If bvhPrimitives has only a single primitive, a leaf node is created. Otherwise, this method partitions its elements and reorders the array, then passes the two partitioned subsets to recursive calls.

> `buildRecursive()` 方法接受 `totalNodes` 指针来跟踪创建的 BVH 节点总数（用 `std::atomic` 表示以保证线程安全），以及从 0 开始跟踪 orderedPrims 中下一个空闲位置的 `orderedPrimsOffset`。如果 bvhPrimitives 只有一个图元，则创建叶节点。否则，该方法对其元素进行分区并重排数组，然后将两个分区后的子集传递给递归调用。

At leaf nodes, the primitives overlapping the leaf are appended to the orderedPrims array and a leaf node object is initialized. Because `orderedPrimsOffset` is a `std::atomic` variable and `fetch_add()` is an atomic operation, multiple threads can safely perform this operation concurrently without further synchronization.

> 在叶节点处，与叶节点重叠的图元被追加到 orderedPrims 数组中，并初始化叶节点对象。由于 `orderedPrimsOffset` 是 `std::atomic` 变量且 `fetch_add()` 是原子操作，多个线程可以安全地并发执行此操作而无需额外同步。

For interior nodes, we choose the coordinate axis with the largest extent of bounding box centroids for the primitives in bvhPrimitives to use in partitioning. The general goal is to select a partition that does not have too much overlap of the bounding boxes of the two resulting primitive sets. If all the centroid points are at the same position, recursion stops and a leaf node is created.

> 对于内部节点，我们选择 bvhPrimitives 中图元包围盒质心范围最大的坐标轴进行划分。总体目标是选择一个使两个结果图元集的包围盒没有太多重叠的分区。如果所有质心点在同一位置，递归停止并创建叶节点。

The two recursive calls access independent data, other than when they allocate space in the orderedPrims array, which is already thread safe. Therefore, when there are a reasonably large number of active primitives (more than 128×1024), those calls can be performed in parallel, which improves the performance of BVH construction.

> 两个递归调用访问独立的数据，除了在 orderedPrims 数组中分配空间时（这已经是线程安全的）。因此，当有相当多的活动图元（超过 128×1024）时，这些调用可以并行执行，从而提高 BVH 构建的性能。

**SplitMethod::Middle** computes the midpoint of the primitives' centroids along the splitting axis. The primitives are classified into the two sets using `std::partition()`. If the primitives all have large overlapping bounding boxes, this splitting method may fail to separate the primitives into two groups; in that case, execution falls through to EqualCounts.

> **SplitMethod::Middle** 计算沿分割轴的图元质心中点。图元使用 `std::partition()` 分为两组。如果所有图元都有大的重叠包围盒，此分割方法可能无法将图元分成两组；在这种情况下，执行回退到 EqualCounts。

**SplitMethod::EqualCounts** partitions the primitives into two equal-sized subsets using `std::nth_element()`, which runs in O(n) time. It ensures the first half are the n/2 with smallest centroid coordinate values along the chosen axis, and the second half are those with the largest.

> **SplitMethod::EqualCounts** 使用 `std::nth_element()`（运行时间为 O(n)）将图元划分为两个大小相等的子集。它确保前半部分是沿所选轴质心坐标值最小的 n/2 个，后半部分是最大的。

### 7.3.2 The Surface Area Heuristic

The two primitive partitioning approaches described so far can work well for some distributions of primitives, but they often choose partitions that perform poorly in practice. Most of the best current algorithms for building acceleration structures for ray tracing are based on the "surface area heuristic" (SAH), which provides a well-grounded cost model for answering questions like "which partition of primitives will lead to a better BVH?"

> 到目前为止描述的两种图元划分方法对某些图元分布可以工作得很好，但它们经常选择在实践中表现不佳的分区。当前构建光线追踪加速结构的大多数最佳算法都基于"表面积启发式"（SAH），它提供了一个有充分依据的代价模型来回答"哪种图元分区将产生更好的 BVH？"这类问题。

The SAH model estimates the computational expense of performing ray intersection tests, including the time spent traversing nodes and the time spent on ray–primitive intersection tests. At any point in building an adaptive acceleration structure, we could just create a leaf node for the current region and geometry, incurring a cost of ∑ᵢ t_isect(i). The other option is to split the region, incurring the cost:

c(A, B) = t_trav + p_A ∑ᵢ∈A t_isect(i) + p_B ∑ⱼ∈B t_isect(j)

where t_trav is the traversal cost, p_A and p_B are the probabilities that the ray passes through each child node, and A, B are the primitive sets in the two children.

> SAH 模型估计执行光线相交测试的计算开销，包括遍历节点的时间和光线-图元相交测试的时间。在构建自适应加速结构的任何阶段，我们可以直接为当前区域和几何体创建叶节点，代价为 ∑ᵢ t_isect(i)。另一个选择是分割区域，代价为：c(A, B) = t_trav + p_A ∑ᵢ∈A t_isect(i) + p_B ∑ⱼ∈B t_isect(j)，其中 t_trav 是遍历代价，p_A 和 p_B 是光线穿过每个子节点的概率，A 和 B 是两个子节点中的图元集。

The probabilities p_A and p_B can be computed using geometric probability. It can be shown that for a convex volume A contained in another convex volume B, the conditional probability that a uniformly distributed random ray passing through B will also pass through A is the ratio of their surface areas: p_A = SA(A)/SA(B).

> 概率 p_A 和 p_B 可以使用几何概率计算。可以证明，对于包含在另一个凸体 B 中的凸体 A，均匀分布的随机光线穿过 B 后也穿过 A 的条件概率是它们表面积的比值：p_A = SA(A)/SA(B)。

When splitMethod has the value SplitMethod::SAH, the SAH is used for building the BVH. Rather than exhaustively considering all possible partitions, the implementation divides the range along the axis into a small number of buckets (12) of equal extent. It then only considers partitions at bucket boundaries. This approach is more efficient than considering all partitions while usually still producing partitions that are nearly as effective.

> 当 splitMethod 的值为 SplitMethod::SAH 时，使用 SAH 构建 BVH。实现不是穷举所有可能的分区，而是将沿轴的范围划分为少量等宽的桶（12个），然后只考虑桶边界处的分区。这种方法比考虑所有分区更高效，同时通常仍能产生几乎一样有效的分区。

For each primitive, the bucket that its centroid lies in is determined and the bucket's bounds are updated. Then costs are computed via a forward scan and a backward scan over the buckets. The forward scan incrementally computes `countBelow * boundBelow.SurfaceArea()` for each candidate split; the backward scan adds `countAbove * boundAbove.SurfaceArea()`. This achieves O(n) cost computation.

> 对于每个图元，确定其质心所在的桶并更新该桶的包围。然后通过对桶的前向扫描和后向扫描计算代价。前向扫描增量地计算每个候选分割的 `countBelow * boundBelow.SurfaceArea()`；后向扫描添加 `countAbove * boundAbove.SurfaceArea()`。这实现了 O(n) 的代价计算。

We set the estimated intersection cost to 1 and the estimated traversal cost to 1/2. We have intentionally underestimated the performance ratio between traversal and intersection costs because with a lower traversal cost, the resulting BVHs would be deeper and require more nodes. For complex scenes, this additional memory use may be undesirable. Even for simpler scenes, visiting more nodes will generally incur cache misses.

> 我们将估计的相交代价设为 1，估计的遍历代价设为 1/2。我们有意低估了遍历与相交代价之间的性能比，因为较低的遍历代价会导致 BVH 更深并需要更多节点。对于复杂场景，这额外的内存使用可能不可取。即使对于简单场景，访问更多节点通常也会导致缓存未命中。

The final SAH cost for a split divides by the surface area of the overall bounding box to compute probabilities, then adds the estimated traversal cost. If the chosen split has a lower estimated cost than building a leaf node (or more than maxPrimsInNode primitives are present), `std::partition()` is used to reorder primitives at the selected bucket boundary. Otherwise, a leaf node is created.

> 分割的最终 SAH 代价除以总包围盒的表面积来计算概率，然后加上估计的遍历代价。如果所选分割的估计代价低于构建叶节点的代价（或存在超过 maxPrimsInNode 个图元），则使用 `std::partition()` 在选定的桶边界处重新排序图元。否则，创建叶节点。

### 7.3.3 Linear Bounding Volume Hierarchies

While building BVHs using the SAH gives very good results, that approach has two disadvantages: first, many passes are taken over the scene primitives. Second, top-down BVH construction is difficult to parallelize well. *Linear bounding volume hierarchies* (LBVHs) were developed to address these issues. With LBVHs, the tree is built with a small number of lightweight passes; tree construction time is linear in the number of primitives.

> 虽然使用 SAH 构建 BVH 效果很好，但该方法有两个缺点：首先，需要对场景图元进行多次遍历。其次，自顶向下的 BVH 构建难以很好地并行化。**线性包围体层次结构**（LBVH）被开发来解决这些问题。使用 LBVH，树通过少量轻量级遍历构建；树构建时间与图元数量成线性关系。

The key idea behind LBVHs is to turn BVH construction into a sorting problem. LBVHs are based on *Morton codes*, which map nearby points in n dimensions to nearby points along the 1D line. Morton codes are found by interleaving the bits of the integer coordinates in base 2. Points with coordinates that are close together in 2D/3D are generally close together along the Morton curve.

> LBVH 背后的关键思想是将 BVH 构建转化为排序问题。LBVH 基于 **Morton 码**，它将 n 维中相邻的点映射到一维线上的相邻点。Morton 码通过交错整数坐标的二进制位来得到。在二维/三维中坐标接近的点在 Morton 曲线上通常也接近。

A Morton-encoded value also encodes useful information about the position of the point it represents. Points with a number of matching high bits lie in a power-of-two sized and axis-aligned region of space determined by the matching bit values. This property is exploited for efficient partitioning.

> Morton 编码的值还编码了关于它所表示的点的位置的有用信息。具有若干匹配高位的点位于由匹配位值决定的二的幂大小的轴对齐空间区域中。这一性质被利用来进行高效划分。

In the implementation here, we build a *hierarchical linear bounding volume hierarchy* (HLBVH). Morton-curve-based clustering is used to first build trees for the lower levels of the hierarchy (referred to as "treelets"), and the top levels of the tree are then created using the surface area heuristic. The `buildHLBVH()` method implements this approach.

> 在此实现中，我们构建一个**分层线性包围体层次结构**（HLBVH）。基于 Morton 曲线的聚类首先用于构建层次结构低层的树（称为"树小枝"，treelet），然后使用表面积启发式创建树的顶层。`buildHLBVH()` 方法实现了此方法。

We use 10 bits for each of the x, y, and z dimensions, giving a total of 30 bits for the Morton code, fitting in a single 32-bit variable. Floating-point centroid offsets inside the bounding box are scaled by 1024 to get integer coordinates. `EncodeMorton3()` returns the 3D Morton code for the given integer values. Morton indices are computed in parallel using `ParallelFor`.

> 我们为 x、y、z 每个维度使用 10 位，Morton 码共 30 位，适合一个 32 位变量。包围盒内的浮点质心偏移按 1024 缩放以获得整数坐标。`EncodeMorton3()` 返回给定整数值的三维 Morton 码。Morton 索引使用 `ParallelFor` 并行计算。

The mortonPrims array is sorted by Morton index using a radix sort, which is noticeably faster than `std::sort()` for BVH construction. The implementation sorts 6 bits per pass over 5 passes to sort the 30 bits. Each pass counts how many values land in each of the 64 buckets, computes starting indices, and then distributes values to the output array.

> mortonPrims 数组通过基数排序按 Morton 索引排序，对于 BVH 构建这明显快于 `std::sort()`。实现每次排序 6 位，经过 5 次遍历排序完 30 位。每次遍历统计每个 64 个桶中有多少值落入，计算起始索引，然后将值分发到输出数组。

After sorting, primitive clusters with matching high 12 bits of their 30-bit Morton codes are identified in a linear pass. This corresponds to clustering primitives in a regular grid of 4096 total cells. Each cluster forms a treelet with preallocated BVHBuildNodes.

> 排序后，在一次线性遍历中识别具有匹配的 30 位 Morton 码高 12 位的图元聚类。这对应于在 4096 个网格单元的规则网格中聚类图元。每个聚类形成一个预分配了 BVHBuildNode 的树小枝。

LBVHs for all treelets are then created in parallel using `ParallelFor`. The `emitLBVH()` method takes primitives within a spatial region and successively partitions them with splitting planes that divide the current region in half along one axis. It processes bits from high to low; because the Morton codes are already sorted, the two spans of primitives are already on the correct sides of the partition plane. When all bits are exhausted or the primitive count is small enough, a leaf node is created. Binary search via `FindInterval()` efficiently finds the split point where a Morton code bit changes from 0 to 1.

> 然后使用 `ParallelFor` 并行创建所有树小枝的 LBVH。`emitLBVH()` 方法获取空间区域内的图元，并用将当前区域沿一个轴对半分割的分割平面逐步划分它们。它从高位到低位处理；由于 Morton 码已经排序，两个图元跨度已经在分割平面的正确一侧。当所有位用完或图元计数足够小时，创建叶节点。通过 `FindInterval()` 的二分搜索高效找到 Morton 码某位从 0 变为 1 的分割点。

Once all the LBVH treelets have been created, `buildUpperSAH()` creates a BVH of all the treelets using the SAH approach. Since there are generally tens or hundreds of them (and in any case, no more than 4096), this step takes very little time.

> 所有 LBVH 树小枝创建后，`buildUpperSAH()` 使用 SAH 方法创建所有树小枝的 BVH。由于通常有几十到几百个（最多不超过 4096 个），这一步花费很少时间。

### 7.3.4 Compact BVH for Traversal

Once the BVH is built, the last step is to convert it into a compact representation. The final BVH is stored in a linear array in memory. The nodes of the original tree are laid out in depth-first order, which means that the first child of each interior node is immediately after the node in memory. Only the offset to the second child of each interior node must be stored explicitly.

> BVH 构建完成后，最后一步是将其转换为紧凑表示。最终 BVH 存储在内存中的线性数组中。原始树的节点按深度优先顺序排列，这意味着每个内部节点的第一个子节点紧跟在该节点之后。只有到每个内部节点第二个子节点的偏移量需要显式存储。

The LinearBVHNode structure stores: the bounding box for each node; for leaf nodes, the offset and primitive count; for interior nodes, the offset to the second child and the partitioning axis. The structure is declared to require 32-byte alignment in memory, ensuring no LinearBVHNode straddles two cache lines, so at most a single cache miss is incurred when one is accessed.

> LinearBVHNode 结构存储：每个节点的包围盒；对于叶节点，存储偏移量和图元计数；对于内部节点，存储到第二个子节点的偏移量和划分轴。该结构声明需要 32 字节内存对齐，确保没有 LinearBVHNode 横跨两条缓存行，因此访问时最多产生一次缓存未命中。

The built tree is transformed to the LinearBVHNode representation by the `flattenBVH()` method, which performs a depth-first traversal and stores nodes in memory in linear order. The bvhPrimitives array memory is released first via `resize(0)` and `shrink_to_fit()`.

> 构建的树通过 `flattenBVH()` 方法转换为 LinearBVHNode 表示，该方法执行深度优先遍历并按线性顺序将节点存储在内存中。bvhPrimitives 数组的内存首先通过 `resize(0)` 和 `shrink_to_fit()` 释放。

### 7.3.5 Bounding and Intersection Tests

Given a built BVH, the `Bounds()` method simply returns the root node's bounds. The `Intersect()` method precomputes `invDir` (reciprocal of ray direction components) and `dirIsNeg` (sign of each direction component) for use in the traversal loop.

> 给定构建好的 BVH，`Bounds()` 方法简单返回根节点的包围。`Intersect()` 方法预计算 `invDir`（光线方向分量的倒数）和 `dirIsNeg`（每个方向分量的符号）用于遍历循环。

The BVH traversal code uses no recursive function calls. It maintains a `nodesToVisit[]` stack (statically allocated with 64 entries, sufficient for any practical tree depth) and a `currentNodeIndex`. At each iteration, the ray is tested against the current node's bounding box. If it misses, the next node is popped from the stack (or traversal ends). If it hits a leaf, all primitives in the leaf are tested; if it hits an interior node, the two children must be visited.

> BVH 遍历代码不使用递归函数调用。它维护一个 `nodesToVisit[]` 栈（静态分配 64 个条目，对任何实际树深度都足够）和一个 `currentNodeIndex`。每次迭代中，光线与当前节点的包围盒进行测试。如果未命中，从栈中弹出下一个节点（或遍历结束）。如果命中叶节点，测试叶中的所有图元；如果命中内部节点，两个子节点都必须被访问。

For leaf nodes, if an intersection is found, tMax is updated to the intersection's parametric distance, allowing efficient discard of remaining nodes that are farther away.

> 对于叶节点，如果找到交点，tMax 更新为交点的参数距离，从而可以高效地丢弃更远的剩余节点。

For interior nodes, an efficient front-to-back traversal is achieved by using the sign of the ray's direction vector for the coordinate axis along which primitives were partitioned. If the sign is negative, the second child is visited before the first child, since the primitives that went into the second child's subtree were on the upper side of the partition point. The offset for the node to be visited first is copied to `currentNodeIndex`, and the offset for the other node is added to the `nodesToVisit` stack. The first child is always immediately after the current node due to the depth-first layout.

> 对于内部节点，通过使用光线方向向量在图元被划分的坐标轴上的符号来实现高效的从前到后遍历。如果符号为负，则先访问第二个子节点再访问第一个子节点，因为进入第二个子节点子树的图元位于分割点的上方。要先访问的节点偏移量被复制到 `currentNodeIndex`，另一个节点的偏移量被添加到 `nodesToVisit` 栈中。由于深度优先布局，第一个子节点始终紧跟在当前节点之后。

The `BVHAggregate::IntersectP()` method is essentially the same as the regular intersection method, with the differences that `Primitive::IntersectP()` methods are called rather than `Intersect()`, and traversal stops immediately when any intersection is found.

> `BVHAggregate::IntersectP()` 方法与常规相交方法基本相同，不同之处在于调用 `Primitive::IntersectP()` 而非 `Intersect()`，且一旦找到任何交点就立即停止遍历。
