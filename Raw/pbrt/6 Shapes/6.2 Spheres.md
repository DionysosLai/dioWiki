## 6.2 Spheres

Spheres are a special case of a general type of surface called *quadrics* —surfaces described by quadratic polynomials in x, y, and z. They offer a good starting point for introducing ray intersection algorithms. In conjunction with a transformation matrix, pbrt 's Sphere shape can also take the form of an ellipsoid. pbrt supports two other basic types of quadrics: cylinders and disks. Other quadrics such as the cone, hyperboloid, and paraboloid are less useful for most rendering applications, and so are not included in the system.

> 球体是一类称为*二次曲面*的一般曲面的特例——即由 x、y、z 的二次多项式描述的曲面。它们是引入光线求交算法的良好起点。配合变换矩阵，pbrt 的 Sphere 形状还可以表现为椭球体。pbrt 支持另外两种基本二次曲面类型：圆柱和圆盘。其他二次曲面如锥面、双曲面和抛物面在大多数渲染应用中不太实用，因此未包含在系统中。

Many surfaces can be described in one of two main ways: in *implicit form* and in *parametric form*. An implicit function describes a 3D surface as f(x,y,z)=0. The set of all points that fulfill this condition defines the surface. For a unit sphere at the origin, the familiar implicit equation is x²+y²+z²−1=0.

> 许多曲面可以用两种主要方式之一来描述：*隐式形式*和*参数形式*。隐式函数将三维曲面描述为 f(x,y,z)=0。满足此条件的所有点的集合定义了该曲面。对于位于原点的单位球体，其熟悉的隐式方程为 x²+y²+z²−1=0。

Many surfaces can also be described parametrically using a function to map 2D points to 3D points on the surface. For example, a sphere of radius r can be described as a function of 2D spherical coordinates (θ, φ), where θ ranges from 0 to π and φ ranges from 0 to 2π.

> 许多曲面也可以用参数形式描述，使用一个函数将二维点映射到曲面上的三维点。例如，半径为 r 的球体可以描述为二维球坐标 (θ, φ) 的函数，其中 θ 的范围从 0 到 π，φ 的范围从 0 到 2π。

We can transform this function into a function over (u,v) and generalize it slightly to allow partial spheres that only sweep out limited θ and φ ranges. This form is particularly useful for texture mapping, where it can be directly used to map a texture defined over [0,1]² to the sphere.

> 我们可以将此函数转化为 (u,v) 上的函数，并略加推广以允许仅扫过有限 θ 和 φ 范围的部分球体。这种形式对于纹理映射特别有用，可以直接用于将定义在 [0,1]² 上的纹理映射到球体上。

As we describe the implementation of the sphere shape, we will make use of both the implicit and parametric descriptions of the shape, depending on which is a more natural way to approach the particular problem we are facing.

> 在描述球体形状的实现时，我们将同时使用形状的隐式和参数描述，取决于哪种方式更自然地适合我们面临的特定问题。

The Sphere class represents a sphere that is centered at the origin. As mentioned earlier, spheres in pbrt are defined in a coordinate system where the center of the sphere is at the origin. The sphere constructor is provided transformations that map between the sphere's object space and rendering space.

> Sphere 类表示以原点为中心的球体。如前所述，pbrt 中的球体定义在球心位于原点的坐标系中。球体的构造函数接收在球体物体空间和渲染空间之间映射的变换。

The radius of the sphere can have an arbitrary positive value, and the sphere's extent can be truncated in two different ways. First, minimum and maximum z values may be set; the parts of the sphere below and above these planes are cut off. Second, a maximum φ value can be set. The sphere sweeps out φ values from 0 to the given φMax. Finally, the constructor also takes a Boolean parameter, reverseOrientation, that indicates whether their surface normal directions should be reversed from the default.

> 球体的半径可以取任意正值，球体的范围可以通过两种方式截断。首先，可以设置最小和最大 z 值；球体在这些平面以下和以上的部分被截掉。其次，可以设置最大 φ 值。球体从 0 扫过到给定的 φMax。最后，构造函数还接收一个布尔参数 reverseOrientation，指示是否应将表面法线方向从默认方向反转。

### 6.2.1 Bounding

Computing an object-space bounding box for a sphere is straightforward. The implementation uses the z values provided by the user to tighten up the bound when less than an entire sphere is being rendered. However, it does not do the extra work to compute a tighter bounding box when φMax is less than 2π. The Sphere's NormalBounds() method does not consider any form of partial spheres but always returns the bounds for an entire sphere, which is all possible directions.

> 计算球体的物体空间包围盒很直接。实现使用用户提供的 z 值在渲染不完整球体时收紧边界。但是，当 φMax 小于 2π 时它不做额外工作来计算更紧的包围盒。Sphere 的 NormalBounds() 方法不考虑任何形式的部分球体，而是始终返回整个球体的界，即所有可能的方向。

### 6.2.2 Intersection Tests

The ray intersection test is broken into two stages. First, BasicIntersect() does the basic ray–sphere intersection test and returns a small structure, QuadricIntersection, if an intersection is found. A subsequent call to the InteractionFromIntersection() method transforms the QuadricIntersection into a full-blown SurfaceInteraction.

> 光线求交测试分为两个阶段。首先，BasicIntersect() 执行基本的光线-球体求交测试，如果找到交叉则返回一个小结构 QuadricIntersection。随后调用 InteractionFromIntersection() 方法将 QuadricIntersection 转化为完整的 SurfaceInteraction。

There are two motivations for separating the intersection test into two stages like this. One is that doing so allows the IntersectP() method to be implemented as a trivial wrapper around BasicIntersect(). The second is that pbrt's GPU rendering path is organized such that the closest intersection among all shapes is found before the full SurfaceInteraction is constructed; this decomposition fits that directly.

> 将求交测试分成两个阶段有两个动机。一是这样做可以让 IntersectP() 方法作为 BasicIntersect() 的简单包装实现。二是 pbrt 的 GPU 渲染路径的组织方式是先在所有形状中找到最近的交叉，然后再构造完整的 SurfaceInteraction；这种分解直接契合这一需求。

QuadricIntersection stores the parametric t along the ray where the intersection occurred, the object-space intersection point, and the sphere's φ value there.

> QuadricIntersection 存储交叉发生处沿光线的参数 t、物体空间交叉点以及该处球体的 φ 值。

The basic intersection test transforms the provided rendering-space ray to object space and intersects it with the complete sphere. If a partial sphere has been specified, some additional tests reject intersections with portions of the sphere that have been removed.

> 基本求交测试将提供的渲染空间光线变换到物体空间，并与完整球体求交。如果指定了部分球体，则一些额外测试会拒绝与已移除部分球体的交叉。

The transformed ray origin and direction are stored using Point3fi and Vector3fi classes rather than the usual Point3f and Vector3f. These classes represent those quantities as small intervals in each dimension that bound the floating-point round-off error introduced by applying the transformation. These error bounds will be useful for improving the geometric accuracy of the intersection computation.

> 变换后的光线原点和方向使用 Point3fi 和 Vector3fi 类存储，而非通常的 Point3f 和 Vector3f。这些类将这些量表示为每个维度上的小区间，界定了应用变换引入的浮点舍入误差。这些误差界对于提高求交计算的几何精度很有用。

If a sphere is centered at the origin with radius r, its implicit representation is x²+y²+z²−r²=0. By substituting the parametric representation of the ray into the implicit sphere equation, we get a general quadratic equation in t: at²+bt+c=0, where the coefficients a, b, c are derived from the ray origin and direction components and the sphere radius.

> 如果球体以原点为中心且半径为 r，其隐式表示为 x²+y²+z²−r²=0。将光线的参数表示代入隐式球体方程，我们得到关于 t 的一般二次方程：at²+bt+c=0，其中系数 a、b、c 由光线原点和方向分量以及球体半径推导而来。

The Interval class stores a small range of floating-point values to maintain bounds on floating-point rounding error. It is used for the quadratic coefficients to track numerical error throughout the computation.

> Interval 类存储一个浮点值的小范围以维护浮点舍入误差的界。它用于二次方程系数，以在整个计算过程中跟踪数值误差。

The discriminant is computed in a way that maintains numerical accuracy in tricky cases (deferred to §6.8.3). A variant of the traditional quadratic formula that gives more accuracy is used. Because t0 and t1 represent intervals, their lower bounds are used for ordering comparisons to avoid returning a farther hit when a closer one may exist.

> 判别式的计算方式在棘手情况下保持数值精度（推迟到 §6.8.3）。使用了传统二次方程求解公式的一个变体以获得更高精度。由于 t0 和 t1 表示区间，使用它们的下界进行排序比较，以避免在可能存在更近命中时返回更远的命中。

Given the parametric distance along the ray to the intersection with a full sphere, the intersection point pHit can be computed. Due to floating-point precision limitations, this computed point may lie a bit to one side of the actual sphere surface; the refinement step (§6.8.5) improves the accuracy of this value.

> 给定沿光线到与完整球体交叉的参数距离，可以计算交叉点 pHit。由于浮点精度限制，这个计算出的点可能略微偏离实际球体表面；细化步骤（§6.8.5）提高了该值的精度。

It is next necessary to handle partial spheres with clipped z or φ ranges. The hit point can now be tested against the specified minima and maxima. One subtlety is that it is important to skip the z tests if the range includes the entire sphere, since the computed pHit.z value may be slightly out of range due to floating-point round-off. If the intersection is not valid, the routine tries again with t1.

> 接下来需要处理 z 或 φ 范围被裁剪的部分球体。现在可以将命中点与指定的最小值和最大值进行测试。一个微妙之处是，如果范围包含整个球体，则应跳过 z 测试，因为计算出的 pHit.z 值可能因浮点舍入而略微超出范围。如果交叉无效，例程会用 t1 再试一次。

At this point in the routine, it is certain that the ray hits the sphere. A QuadricIntersection is returned. Recall from §6.1.4 that even though tShapeHit was computed in object space, it is also the correct value in rendering space.

> 此时，例程已确定光线命中了球体。返回一个 QuadricIntersection。回忆 §6.1.4，尽管 tShapeHit 是在物体空间中计算的，它在渲染空间中也是正确的值。

A QuadricIntersection can be upgraded to a SurfaceInteraction with a call to InteractionFromIntersection(). The method first computes u and v values, then finds the parametric partial derivatives of position (∂p/∂u, ∂p/∂v) and surface normal (∂n/∂u, ∂n/∂v).

> QuadricIntersection 可以通过调用 InteractionFromIntersection() 升级为 SurfaceInteraction。该方法首先计算 u 和 v 值，然后找到位置的参数偏导数（∂p/∂u、∂p/∂v）和表面法线的参数偏导数（∂n/∂u、∂n/∂v）。

Computing the partial derivatives of a point on the sphere is a short exercise in algebra. For dpdu, the x-component is: ∂px/∂u = −φMax · pHit.y. Similarly, dpdv is derived from the θ derivatives.

> 计算球体上某点的偏导数是一个简短的代数练习。对于 dpdu，x 分量为：∂px/∂u = −φMax · pHit.y。类似地，dpdv 由 θ 的导数推导。

It is also useful to determine how the normal changes as we move along the surface in the u and v directions. The differential changes in normal ∂n/∂u and ∂n/∂v are given by the *Weingarten equations* from differential geometry, involving the coefficients of the first fundamental form (E, F, G) and the second fundamental form (e, f, g).

> 确定法线在沿 u 和 v 方向移动时如何变化也很有用。法线的微分变化 ∂n/∂u 和 ∂n/∂v 由微分几何中的 *Weingarten 方程*给出，涉及第一基本形式的系数（E、F、G）和第二基本形式的系数（e、f、g）。

The two fundamental forms capture elementary metric properties of a surface, including notions of distance, angle, and curvature. To find e, f, and g, it is necessary to compute the second-order partial derivatives d²p/du², d²p/dudv, and d²p/dv².

> 两个基本形式捕获曲面的基本度量属性，包括距离、角度和曲率的概念。为了求 e、f 和 g，需要计算二阶偏导数 d²p/du²、d²p/dudv 和 d²p/dv²。

We use DifferenceOfProducts() to compute EG−F² for better numerical accuracy. Note also that we must be careful to avoid dividing by 0 if that expression is zero-valued so that dndu and dndv do not take on NaN values.

> 我们使用 DifferenceOfProducts() 来计算 EG−F² 以获得更好的数值精度。还需注意避免在该表达式为零时除以 0，以使 dndu 和 dndv 不会变为 NaN 值。

Having computed the surface parameterization and all the relevant partial derivatives, a SurfaceInteraction can be returned. Three things to note: (1) the intersection point is provided as a Point3fi with error bounds; (2) the SurfaceInteraction is initialized with object-space geometric quantities and then transformed to rendering space; (3) the flipNormal parameter indicates whether the surface normal should be flipped, which occurs either when ReverseOrientation is enabled or when the transform swaps handedness (but not both).

> 计算完表面参数化和所有相关偏导数后，可以返回 SurfaceInteraction。需要注意三点：(1) 交叉点以带误差界的 Point3fi 提供；(2) SurfaceInteraction 以物体空间几何量初始化，然后变换到渲染空间；(3) flipNormal 参数指示是否应翻转表面法线，这在启用 ReverseOrientation 或变换交换手性时发生（但不是两者同时发生时）。

### 6.2.3 Surface Area

To compute the surface area of quadrics, we use a standard formula from integral calculus for surfaces of revolution. For the sphere (a surface of revolution of a circular arc), the surface area is: Area = φMax · r · (zMax − zMin). For the full sphere, this gives the standard formula 4πr².

> 为计算二次曲面的表面积，我们使用积分学中旋转曲面的标准公式。对于球体（圆弧的旋转曲面），表面积为：Area = φMax · r · (zMax − zMin)。对于完整球体，这给出标准公式 4πr²。

### 6.2.4 Sampling

Uniformly sampling a point on the sphere's area is easy: Sphere::Sample() generates a point on the unit sphere using SampleUniformSphere() and then scales the point by the sphere's radius. The final sample is returned in rendering space with a PDF equal to one over the surface area.

> 对球体面积均匀采样很简单：Sphere::Sample() 使用 SampleUniformSphere() 在单位球上生成一个点，然后按球体半径缩放该点。最终样本以渲染空间返回，PDF 等于表面积的倒数。

For the sphere sampling method that is given a point being illuminated, we can do much better than sampling over the sphere's entire area. A better approach is not to sample points on the sphere that are definitely not visible (such as those on the back side). The sampling routine instead uniformly samples directions over the solid angle subtended by the sphere from the reference point and then computes the point on the sphere corresponding to the sampled direction.

> 对于给定被照亮点的球体采样方法，我们可以做得比在整个球体表面上采样好得多。更好的方法是不采样那些肯定不可见的球体上的点（如背面的点）。采样例程改为在球体从参考点所张的立体角内均匀采样方向，然后计算与采样方向对应的球体上的点。

For points that lie inside the sphere, the entire sphere should be sampled, since the whole sphere is visible from inside it. The reference point is computed using OffsetRayOrigin() to ensure that if it came from a ray intersecting the sphere, the point tested does not lie on the wrong side due to rounding error.

> 对于位于球体内部的点，应采样整个球体，因为从内部可以看到整个球体。参考点使用 OffsetRayOrigin() 计算，以确保如果它来自与球体相交的光线，被测试的点不会因舍入误差而位于错误一侧。

If the reference point is outside the sphere, the sphere subtends an angle θMax where sin(θMax) = r/d, with r the radius and d the distance to the center. The sampling method computes cos(θMax) and then uniformly samples directions inside this cone using the uniform cone sampling approach (derived in §A.5.4), sampling a θ offset from the center vector and uniformly sampling a rotation angle φ.

> 如果参考点在球体外部，球体张角 θMax，其中 sin(θMax) = r/d，r 为半径，d 为到中心的距离。采样方法计算 cos(θMax)，然后使用均匀锥采样方法（在 §A.5.4 中推导）在此锥内均匀采样方向，采样从中心向量偏移的 θ 以及均匀采样旋转角 φ。

For very small angles, cos(θMax) is close to one. Computing sin²θ by subtracting this value from 1 gives a value close to 0 with very little accuracy. Therefore, in this case, we use single-term Taylor expansions near 0, which gives much better accuracy.

> 对于非常小的角度，cos(θMax) 接近 1。通过从 1 中减去此值来计算 sin²θ 会得到一个精度很低的接近 0 的值。因此，在这种情况下，我们使用在 0 附近的单项 Taylor 展开，这能给出好得多的精度。

Given a sample angle θ with respect to the sampling coordinate system, we directly compute the corresponding point on the sphere. The angle α between the vector from the reference point to the sampled point and the vector from the center to the reference point is found using the law of sines. We avoid expensive inverse trigonometric functions by computing sin(α) and cos(α) directly using trigonometric identities.

> 给定相对于采样坐标系的样本角 θ，我们直接计算球体上的对应点。从参考点到采样点的向量与从中心到参考点的向量之间的角度 α 使用正弦定理求得。我们通过使用三角恒等式直接计算 sin(α) 和 cos(α)，避免了昂贵的反三角函数。

The angle α and φ give the spherical coordinates for the sampled direction with respect to a coordinate system centered around the vector from the reference point to the sphere center. A Frame instance transforms the direction from that coordinate system to rendering space. The surface normal on the sphere is the negation of that vector, and the point on the sphere is found by scaling by the radius and translating by the sphere's center point.

> 角度 α 和 φ 给出了相对于以从参考点到球心的向量为中心的坐标系的采样方向的球坐标。Frame 实例将方向从该坐标系变换到渲染空间。球体上的表面法线是该向量的取反，球体上的点通过按半径缩放并按球心平移得到。

The PDF for uniform sampling in a cone is 1/(2π(1−cos θMax)). The PDF method that computes the PDF for sampling a direction toward a sphere from a reference point also differs depending on which of the two sampling strategies would be used. Here it is also worth considering numerical accuracy when the sphere subtends a small solid angle: in that case, we switch to the one-term Taylor approximation of 1−cos θ ≈ sin²θ/2, which is more accurate near zero.

> 锥内均匀采样的 PDF 为 1/(2π(1−cos θMax))。计算从参考点向球体采样方向的 PDF 的方法也根据使用的两种采样策略而不同。这里也值得考虑球体张小立体角时的数值精度：在这种情况下，我们切换到 1−cos θ ≈ sin²θ/2 的单项 Taylor 近似，这在接近零时更精确。
