## 6.7 Curves

While triangles or bilinear patches can be used to represent thin shapes for modeling fine geometry like hair, fur, or fields of grass, it is worthwhile to have a specialized Shape in order to more efficiently render these sorts of objects, since many individual instances of them are often present. The Curve shape, introduced in this section, represents thin geometry modeled with cubic Bézier curves, which are defined by four control points. The Bézier spline passes through the first and last control points. Points along it are given by the polynomial.

> 虽然三角形或双线性面片可用于表示薄形状以建模毛发、皮毛或草地等精细几何体，但为了更高效地渲染这类对象（因为通常存在大量个体实例），值得拥有一个专用的 Shape。本节介绍的 Curve 形状用三次贝塞尔曲线表示薄几何体，由四个控制点定义。贝塞尔样条通过第一个和最后一个控制点，沿曲线的点由多项式给出。

(See Figure 6.29.) Curves specified using another basis (e.g., Hermite splines or b-splines) must therefore be converted to the Bézier basis to be used with this Shape.

> （见图 6.29。）使用其他基（例如 Hermite 样条或 B 样条）指定的曲线必须转换为贝塞尔基才能用于此 Shape。

Figure 6.29: A cubic Bézier curve is defined by four control points. The curve passes through the first and last control points at u=0 and u=1, respectively.

> 图 6.29：三次贝塞尔曲线由四个控制点定义。曲线分别在 u=0 和 u=1 处通过第一个和最后一个控制点。

The Curve shape is defined by a 1D Bézier curve along with a width that is linearly interpolated from starting and ending widths along its extent. Together, these define a flat 2D surface (Figure 6.30). It is possible to directly intersect rays with this representation without tessellating it, which in turn makes it possible to efficiently render smooth curves without using too much storage.

> Curve 形状由一条一维贝塞尔曲线及沿其范围从起始宽度到结束宽度线性插值的宽度定义。这些一起定义了一个平坦的二维表面（图 6.30）。可以直接与此表示进行光线求交而无需细分它，从而能够在不使用过多存储的情况下高效渲染光滑曲线。

Figure 6.30: Basic Geometry of the Curve Shape. A 1D Bézier curve is offset by half of the specified width in both the directions orthogonal to the curve at each point along it. The resulting area represents the curve's surface.

> 图 6.30：Curve 形状的基本几何。一维贝塞尔曲线在沿其每个点的两个正交方向上各偏移指定宽度的一半。所得区域代表曲线的表面。

Figure 6.31: Furry Bunny. Bunny model with over one million Curve shapes used to model fur. Here, we have used unrealistically long curves to better show off the Curve's capabilities, giving an unrealistically shaggy bunny. *(Underlying bunny mesh courtesy of the Stanford Computer Graphics Laboratory.)*

> 图 6.31：毛茸茸的兔子。兔子模型使用超过一百万个 Curve 形状来建模皮毛。这里我们使用了不切实际的长曲线来更好地展示 Curve 的能力，产生了一只不切实际的蓬松兔子。*（底层兔子网格由斯坦福计算机图形学实验室提供。）*

There are three types of curves that the Curve shape can represent, shown in Figure 6.32.

> Curve 形状可以表示三种类型的曲线，如图 6.32 所示。

- *Flat:* Curves with this representation are always oriented to face the ray being intersected with them; they are useful for modeling fine swept cylindrical shapes like hair or fur.
- *Cylinder:* For curves that span a few pixels on the screen (like spaghetti seen from not too far away), the Curve shape can compute a shading normal that makes the curve appear to actually be a cylinder.
- *Ribbon:* This variant is useful for modeling shapes that do not actually have a cylindrical cross section (such as a blade of grass).

> - *Flat（平面型）：* 使用此表示的曲线始终朝向与之相交的光线；它们适用于建模毛发或皮毛等精细的扫掠圆柱形状。
> - *Cylinder（圆柱型）：* 对于在屏幕上跨越几个像素的曲线（如从不太远处看到的意大利面），Curve 形状可以计算着色法线，使曲线看起来确实是一个圆柱体。
> - *Ribbon（带状型）：* 此变体适用于建模实际上不具有圆柱截面的形状（如草叶）。

Figure 6.32: The Three Types of Curves That the Curve Shape Can Represent. On the top is a flat curve that is always oriented to be perpendicular to a ray approaching it. The middle is a variant of this curve where the shading normal is set so that the curve appears to be cylindrical. On the bottom is a ribbon, which has a fixed orientation at its starting and ending points; intermediate orientations are smoothly interpolated between them.

> 图 6.32：Curve 形状可以表示的三种曲线类型。顶部是始终朝向垂直于接近它的光线的平面曲线。中间是此曲线的变体，其中着色法线被设置为使曲线看起来呈圆柱状。底部是带状曲线，在其起点和终点具有固定方向；中间方向在它们之间平滑插值。

The CurveType enumerator records which of them a given Curve instance models.

> CurveType 枚举器记录给定 Curve 实例建模的是哪种类型。

The flat and cylinder curve variants are intended to be used as convenient approximations of deformed cylinders. It should be noted that intersections found with respect to them do not correspond to a physically realizable 3D shape, which can potentially lead to minor inconsistencies when taking a scene with true cylinders as a reference.

> 平面型和圆柱型曲线变体旨在用作变形圆柱体的便捷近似。应该注意，相对于它们找到的交点不对应于物理上可实现的三维形状，当以具有真实圆柱体的场景作为参考时，这可能导致轻微的不一致性。

Given a curve specified in a pbrt scene description file, it can be worthwhile to split it into a few segments, each covering part of the parametric range of the curve. (One reason for doing so is that axis-aligned bounding boxes do not tightly bound wiggly curves, but subdividing Bézier curves makes them less wiggly—the *variation diminishing property* of polynomials.) Therefore, the Curve constructor takes a parametric range of values, as well as a pointer to a CurveCommon structure, which stores the control points and other information about the curve that is shared across curve segments. In this way, the memory footprint for individual curve segments is reduced, which makes it easier to keep many of them in memory.

> 对于在 pbrt 场景描述文件中指定的曲线，将其拆分为几个段可能是值得的，每个段覆盖曲线参数范围的一部分。（这样做的一个原因是轴对齐包围盒不能紧密包围弯曲的曲线，但细分贝塞尔曲线使它们变得不那么弯曲——多项式的*变差递减性质*。）因此，Curve 构造函数接受一个参数值范围以及指向 CurveCommon 结构的指针，该结构存储控制点和其他在曲线段之间共享的曲线信息。这样，单个曲线段的内存占用减少了，使得在内存中保留大量曲线段更加容易。

The CurveCommon constructor initializes member variables with values passed into it for the control points, the curve width, etc. The control points provided to it should be in the curve's object space.

> CurveCommon 构造函数使用传入的控制点、曲线宽度等值初始化成员变量。提供给它的控制点应在曲线的物体空间中。

For Ribbon curves, CurveCommon stores a surface normal to orient the curve at each endpoint. The constructor precomputes the angle between the two normal vectors and one over the sine of this angle; these values will be useful when computing the orientation of the curve at arbitrary points along its extent.

> 对于 Ribbon 曲线，CurveCommon 在每个端点存储一个表面法线来定向曲线。构造函数预计算两个法向量之间的角度和该角度正弦的倒数；这些值在计算曲线沿其范围任意点的方向时将很有用。

### 6.7.1 Bounding Curves

### 6.7.1 曲线包围

The object-space bound of a curve can be found by first bounding the spline along the center of the curve and then expanding that bound by half the maximum width the curve takes on over its extent. The Bounds() method then transforms that bound to rendering space before returning it.

> 曲线的物体空间包围可以通过首先沿曲线中心包围样条，然后将该包围扩展曲线在其范围上取得的最大宽度的一半来找到。Bounds() 方法然后在返回之前将该包围变换到渲染空间。

The Curve shape cannot be used as an area light, as it does not provide implementations of the required sampling methods. It does provide a NormalBounds() method that returns a conservative bound.

> Curve 形状不能用作面光源，因为它不提供所需采样方法的实现。它确实提供了一个 NormalBounds() 方法，返回一个保守的包围。

### 6.7.2 Intersection Tests

### 6.7.2 求交测试

Both of the intersection methods required by the Shape interface are implemented via another Curve method, IntersectRay(). Rather than returning an optional ShapeIntersection, it takes a pointer to one.

> Shape 接口所需的两个求交方法都通过另一个 Curve 方法 IntersectRay() 实现。它不是返回一个可选的 ShapeIntersection，而是接受一个指向它的指针。

IntersectP() passes nullptr to IntersectRay(), which indicates that it can return immediately if an intersection is found.

> IntersectP() 向 IntersectRay() 传递 nullptr，这表示一旦找到交点就可以立即返回。

The Curve intersection algorithm is based on discarding curve segments as soon as it can be determined that the ray definitely does not intersect them and otherwise recursively splitting the curve in half to create two smaller segments that are then tested. Eventually, the curve is linearly approximated for an efficient intersection test. That process starts after some initial preparation and early culling tests in IntersectRay().

> Curve 求交算法基于以下策略：一旦能够确定光线确实不与曲线段相交就丢弃该段，否则递归地将曲线一分为二创建两个更小的段然后进行测试。最终，曲线被线性近似以进行高效的求交测试。该过程在 IntersectRay() 中经过一些初始准备和早期剔除测试后开始。

The CurveCommon class stores the control points for the full curve, but a Curve instance generally needs the four control points that represent the Bézier curve for its extent. The CubicBezierControlPoints() utility function performs this computation.

> CurveCommon 类存储完整曲线的控制点，但 Curve 实例通常需要表示其范围内贝塞尔曲线的四个控制点。CubicBezierControlPoints() 实用函数执行此计算。

Like the ray–triangle intersection algorithm from Section 6.5.3, the ray–curve intersection test is based on transforming the curve to a coordinate system with the ray's origin at the origin of the coordinate system and the ray's direction aligned to be along the +z axis. Performing this transformation at the start greatly reduces the number of operations that must be performed for intersection tests.

> 与第 6.5.3 节的光线-三角形求交算法类似，光线-曲线求交测试基于将曲线变换到一个坐标系中，其中光线的原点位于坐标系的原点，光线的方向对齐到 +z 轴方向。在开始时执行此变换大大减少了求交测试必须执行的操作数量。

For the Curve shape, we will need an explicit representation of the transformation, so the LookAt() function is used to generate it here. The origin is the ray's origin and the "look at" point is a point offset from the origin along the ray's direction. The "up" direction is set to be perpendicular to both the ray's direction and the vector from the first to the last control point. Doing so helps orient the curve to be roughly parallel to the x axis in the ray coordinate system, which in turn leads to tighter bounds in y (see Figure 6.33). This improvement in the fit of the bounds often makes it possible to terminate the recursive intersection tests earlier than would be possible otherwise.

> 对于 Curve 形状，我们需要变换的显式表示，因此这里使用 LookAt() 函数来生成它。原点是光线的原点，"看向"点是从原点沿光线方向偏移的点。"上"方向设置为同时垂直于光线方向和从第一个到最后一个控制点的向量。这样做有助于将曲线在光线坐标系中大致平行于 x 轴定向，从而在 y 方向上产生更紧密的包围（见图 6.33）。包围拟合的这种改进通常使得能够比其他方式更早地终止递归求交测试。

Figure 6.33: 2D Bounding Boxes of a Bézier Curve. (a) Bounding box computed using the curve's control points as given. (b) The effect of rotating the curve so that the vector from its first to last control point is aligned with the x axis before computing bounds. The resulting bounding box is a much tighter fit.

> 图 6.33：贝塞尔曲线的二维包围盒。(a) 使用给定的曲线控制点计算的包围盒。(b) 在计算包围之前旋转曲线使得从第一个到最后一个控制点的向量与 x 轴对齐的效果。所得包围盒拟合得更紧密。

If the ray and the vector between the first and last control points are parallel, dx will be degenerate. In that case we find an arbitrary "up" vector direction so that intersection tests can proceed in this unusual case.

> 如果光线和第一个与最后一个控制点之间的向量平行，dx 将退化。在这种情况下，我们找到一个任意的"上"向量方向，以便在这种不寻常的情况下求交测试可以继续进行。

Along the lines of the implementation in Curve::Bounds(), a conservative bounding box for a curve segment can be found by taking the bounds of the curve's control points and expanding by half of the maximum width of the curve over the range being considered.

> 沿着 Curve::Bounds() 实现的思路，可以通过取曲线控制点的包围并扩展所考虑范围内曲线最大宽度的一半来找到曲线段的保守包围盒。

Figure 6.34: Ray–Curve Bounds Test. In the ray coordinate system, the ray's origin is at (0,0,0) and its direction is aligned with the +z axis. Therefore, if the 2D point (0,0) is outside the bounding box of the curve segment, then it is impossible that the ray intersects the curve.

> 图 6.34：光线-曲线包围测试。在光线坐标系中，光线的原点在 (0,0,0)，其方向与 +z 轴对齐。因此，如果二维点 (0,0) 在曲线段的包围盒之外，那么光线不可能与曲线相交。

Because the ray's origin is at (0,0,0) and its direction is aligned with the +z axis in the intersection space, its bounding box only includes the origin in x and y (Figure 6.34); its z extent is given by the range that its parametric extent covers. Before proceeding with the recursive intersection testing algorithm, the ray's bounding box is tested for intersection with the curve's bounding box. The method can return immediately if they do not intersect.

> 因为在求交空间中光线的原点在 (0,0,0) 且其方向与 +z 轴对齐，其包围盒在 x 和 y 方向上只包含原点（图 6.34）；其 z 范围由其参数范围所覆盖的范围给出。在继续递归求交测试算法之前，光线的包围盒与曲线的包围盒进行求交测试。如果它们不相交，方法可以立即返回。

The maximum number of times to subdivide the curve is computed so that the maximum distance from the eventual linearized curve at the finest refinement level is bounded to be less than a small fixed distance. We will not go into the details of this computation. With the culling tests passed and that value in hand, the recursive intersection tests begin.

> 曲线的最大细分次数被计算出来，使得在最细细化级别上最终线性化曲线的最大距离被限制为小于一个固定的小距离。我们不深入此计算的细节。通过剔除测试并获得该值后，递归求交测试开始。

The RecursiveIntersect() method then tests whether the given ray intersects the given curve segment over the given parametric range. It assumes that the ray has already been tested against the curve's bounding box and found to intersect it.

> RecursiveIntersect() 方法然后测试给定光线是否在给定参数范围内与给定曲线段相交。它假设光线已经与曲线的包围盒进行了测试并发现与之相交。

If the maximum depth has not been reached, a call to SubdivideCubicBezier() gives the control points for the two Bézier curves that result in splitting the Bézier curve given by cp in half. The last control point of the first curve is the same as the first control point of the second, so 7 values are returned for the total of 8 control points. The u array is then initialized so that it holds the parametric range of the two curves before each one is processed in turn.

> 如果未达到最大深度，调用 SubdivideCubicBezier() 给出将 cp 给出的贝塞尔曲线一分为二所得两条贝塞尔曲线的控制点。第一条曲线的最后一个控制点与第二条曲线的第一个控制点相同，因此总共 8 个控制点返回 7 个值。然后初始化 u 数组，使其在依次处理每条曲线之前保存两条曲线的参数范围。

The bounding box test in the <<Check ray against curve segment's bounding box>> fragment is essentially the same as the one in <<Test ray against bound of projected control points>> except that it takes values from the u array when computing the curve's maximum width over the range and it uses control points from cpSplit. Therefore, it is not included here.

> <<检查光线与曲线段包围盒>>片段中的包围盒测试与<<测试光线与投影控制点的包围>>中的基本相同，只是在计算范围内曲线的最大宽度时从 u 数组取值，并使用 cpSplit 中的控制点。因此不在此处包含。

If the ray does intersect the bounding box, the corresponding segment is given to a recursive call of RecursiveIntersect(). If an intersection is found and the ray is a shadow ray, si will be nullptr and an intersection can immediately be reported. For non-shadow rays, even if an intersection has been found, it may not be the closest intersection, so the other segment still must be considered.

> 如果光线确实与包围盒相交，对应的段被传给 RecursiveIntersect() 的递归调用。如果找到交点且光线是阴影光线，si 将为 nullptr，可以立即报告交点。对于非阴影光线，即使找到了交点，它可能不是最近的交点，所以另一个段仍必须被考虑。

The intersection test is made more efficient by using a linear approximation of the curve; the variation diminishing property allows us to make this approximation without introducing too much error.

> 通过使用曲线的线性近似使求交测试更加高效；变差递减性质允许我们进行这种近似而不引入过多误差。

Figure 6.35: Curve Segment Boundaries. The intersection test for a segment of a larger curve computes edge functions for the lines that are perpendicular to the segment endpoints (dashed lines). If a potential intersection point (solid dot) is on the other side of the edge than the segment, it is rejected; another curve segment (if present on that side) should account for this intersection instead.

> 图 6.35：曲线段边界。较大曲线的一个段的求交测试为垂直于段端点的线（虚线）计算边缘函数。如果潜在交点（实心点）在边缘的另一侧而不是段的这一侧，则被拒绝；另一个曲线段（如果存在于那一侧）应该负责处理此交点。

It is important that the intersection test only accepts intersections that are on the Curve's surface for the segment currently under consideration. Therefore, the first step of the intersection test is to compute edge functions for lines perpendicular to the curve starting point and ending point and to classify the potential intersection point against them (Figure 6.35).

> 重要的是，求交测试只接受当前考虑的段在 Curve 表面上的交点。因此，求交测试的第一步是为垂直于曲线起点和终点的线计算边缘函数，并针对它们对潜在交点进行分类（图 6.35）。

Projecting the curve control points into the ray coordinate system makes this test more efficient for two reasons. First, because the ray's direction is oriented with the +z axis, the problem is reduced to a 2D test in x and y. Second, because the ray origin is at the origin of the coordinate system, the point we need to classify is (0,0), which simplifies evaluating the edge function, just like the ray–triangle intersection test.

> 将曲线控制点投影到光线坐标系中使此测试更加高效，原因有二。首先，因为光线的方向与 +z 轴对齐，问题被简化为 x 和 y 上的二维测试。其次，因为光线原点在坐标系的原点，我们需要分类的点是 (0,0)，这简化了边缘函数的评估，就像光线-三角形求交测试一样。

Edge functions were introduced for ray–triangle intersection tests in Equation (6.5). To define the edge function here, we need any two points on the line perpendicular to the curve going through the starting point. The first control point is a fine choice for the first. For the second, we will compute the vector perpendicular to the curve's tangent and add that offset to the control point.

> 边缘函数在方程 (6.5) 中为光线-三角形求交测试引入。要在这里定义边缘函数，我们需要通过起点的垂直于曲线的线上的任意两个点。第一个控制点是第一个点的好选择。对于第二个点，我们将计算垂直于曲线切线的向量并将该偏移添加到控制点。

Differentiation of Equation (6.16) shows that the tangent to the curve at the first control point is p₁-p₀. The scaling factor does not matter here. Computing the vector perpendicular to the tangent is easy in 2D: it is just necessary to swap the x and y coordinates and negate one of them. (To see why this works, consider the dot product. Because the cosine of the angle between the two vectors is zero, they must be perpendicular.) Thus, the second point on the edge is computed from this perpendicular.

> 方程 (6.16) 的微分表明曲线在第一个控制点处的切线是 p₁-p₀。缩放因子在这里不重要。计算垂直于切线的向量在二维中很容易：只需交换 x 和 y 坐标并取反其中一个。（要理解为什么这样做有效，考虑点积。因为两个向量之间角度的余弦为零，它们必须是垂直的。）因此，边缘上的第二个点由此垂直向量计算得出。

Substituting these two points into the definition of the edge function, Equation (6.5), and simplifying gives the edge function expression. Finally, substituting (0,0) gives the final expression to test.

> 将这两个点代入边缘函数的定义，方程 (6.5)，并简化得到边缘函数表达式。最后，代入 (0,0) 给出要测试的最终表达式。

The <<Test sample point against tangent perpendicular at curve end>> fragment, not included here, does the corresponding test at the end of the curve.

> <<测试样本点与曲线末端切线垂直>>片段（此处未包含）在曲线末端执行相应的测试。

The next part of the test is to determine the w value along the curve segment where the point is closest to the curve. This will be the intersection point, if it is no farther than the curve's width away from the center at that point. Determining this distance for a cubic Bézier curve requires a significant amount of computation, so instead the implementation here approximates the curve with a linear segment to compute this value.

> 测试的下一部分是确定曲线段上点最接近曲线的 w 值。如果该点与该处中心的距离不超过曲线的宽度，这将是交点。为三次贝塞尔曲线确定此距离需要大量计算，因此这里的实现用线性段近似曲线来计算此值。

Figure 6.36: Approximation of a Cubic Bézier Curve with a Linear Segment. For this part of the ray–curve intersection test, we approximate the Bézier with a linear segment (dashed line) passing through its starting and ending points. (In practice, after being subdivided, the curve will be already nearly linear, so the error is less than this figure suggests.)

> 图 6.36：用线性段近似三次贝塞尔曲线。对于光线-曲线求交测试的这一部分，我们用通过其起点和终点的线性段（虚线）近似贝塞尔曲线。（在实践中，细分后曲线将已经几乎是线性的，所以误差比此图暗示的要小。）

We linearly approximate the Bézier curve with a line segment from its starting point to its endpoint that is parameterized by w. In this case, the position is cp[0] at w=0 and cp[3] at w=1 (Figure 6.36). Our task is to compute the value of w along the line corresponding to the point on the line that is closest to the point (0,0). The key insight to apply is that at the closest w, the vector from the corresponding point on the line to (0,0) will be perpendicular to the line (Figure 6.37 (a)).

> 我们用从起点到终点的线性段线性近似贝塞尔曲线，该线段由 w 参数化。在这种情况下，位置在 w=0 处为 cp[0]，在 w=1 处为 cp[3]（图 6.36）。我们的任务是计算沿线对应于最接近点 (0,0) 的线上点的 w 值。要应用的关键洞察是，在最近的 w 处，从线上对应点到 (0,0) 的向量将垂直于该线（图 6.37 (a)）。

Figure 6.37: (a) Given an infinite line and a point, the vector from the point to the closest point on the line is then perpendicular to the line. (b) Because this vector is perpendicular, we can compute the distance from the first point of the line to the point of closest approach.

> 图 6.37：(a) 给定一条无限直线和一个点，从该点到线上最近点的向量垂直于该线。(b) 因为此向量是垂直的，我们可以计算从线的第一个点到最近接近点的距离。

The computation of the value of w is in turn slightly simplified from the fact that p=(0,0) in the intersection coordinate system.

> w 值的计算由于在求交坐标系中 p=(0,0) 而稍微简化。

The parametric u coordinate of the (presumed) closest point on the Bézier curve to the candidate intersection point is computed by linearly interpolating along the range of the segment. Given this value, the width of the curve at that point can be computed.

> 贝塞尔曲线上（假定的）最接近候选交点的参数 u 坐标通过沿段范围线性插值计算。给定此值，可以计算该点处曲线的宽度。

For Ribbon curves, the curve is not always oriented to face the ray. Rather, its orientation is interpolated between two surface normals given at each endpoint. Here, spherical linear interpolation is used to interpolate the normal at u. The curve's width is then scaled by the cosine of the angle between the normalized ray direction and the ribbon's orientation so that it corresponds to the visible width of the curve from the given direction.

> 对于 Ribbon 曲线，曲线并不总是朝向光线。相反，其方向在每个端点给出的两个表面法线之间插值。这里，使用球面线性插值来插值 u 处的法线。然后曲线的宽度按归一化光线方向和带状曲线方向之间角度的余弦缩放，使其对应于从给定方向看到的曲线的可见宽度。

To finally classify the potential intersection as a hit or miss, the Bézier curve must still be evaluated at w. The derivative of the curve at this point will be useful shortly, so it is recorded now.

> 要最终将潜在交点分类为命中或未命中，贝塞尔曲线仍必须在 w 处求值。该点处曲线的导数很快将会有用，所以现在记录下来。

We would like to test whether the distance from (0,0) to this point on the curve pc is less than half the curve's width. Because p=(0,0), we can equivalently test whether the distance from pc to the origin is less than half the width or whether the squared distance is less than one quarter the width squared. If this test passes, the last thing to check is if the intersection point is in the ray's parametric range.

> 我们想要测试从 (0,0) 到曲线上此点 pc 的距离是否小于曲线宽度的一半。因为 p=(0,0)，我们可以等价地测试从 pc 到原点的距离是否小于宽度的一半，或者距离的平方是否小于宽度平方的四分之一。如果此测试通过，最后要检查的是交点是否在光线的参数范围内。

For non-shadow rays, the ShapeIntersection for the intersection can finally be initialized. Doing so requires computing the ray t value for the intersection as well as its SurfaceInteraction.

> 对于非阴影光线，求交的 ShapeIntersection 最终可以被初始化。这样做需要计算交点的光线 t 值及其 SurfaceInteraction。

After the tHit value has been computed, it is compared against the tHit of a previously found ray–curve intersection, if there is one. This check ensures that the closest intersection is returned.

> 在计算出 tHit 值后，将其与先前找到的光线-曲线交点的 tHit 进行比较（如果有的话）。此检查确保返回最近的交点。

A variety of additional quantities need to be computed in order to be able to initialize the intersection's SurfaceInteraction.

> 需要计算各种额外的量才能初始化交点的 SurfaceInteraction。

We have gotten this far without computing the v coordinate of the intersection point, which is now needed. The curve's v coordinate ranges from 0 to 1, taking on the value 1/2 at the center of the curve; here, we classify the intersection point with respect to an edge function going through the point on the curve pc and a point along its derivative to determine which side of the center the intersection point is on and in turn how to compute v.

> 我们到目前为止还没有计算交点的 v 坐标，现在需要它了。曲线的 v 坐标范围从 0 到 1，在曲线中心取值 1/2；这里，我们使用通过曲线上的点 pc 和沿其导数的点的边缘函数对交点进行分类，以确定交点在中心的哪一侧，从而确定如何计算 v。

The partial derivative ∂p/∂u comes directly from the derivative of the underlying Bézier curve. The second partial derivative, ∂p/∂v, is computed in different ways based on the type of the curve. For ribbons, we have ∂p/∂u and the surface normal, and so ∂p/∂v must be the vector such that the cross product gives the normal and has length equal to the curve's width.

> 偏导数 ∂p/∂u 直接来自底层贝塞尔曲线的导数。第二个偏导数 ∂p/∂v 根据曲线类型以不同方式计算。对于带状曲线，我们有 ∂p/∂u 和表面法线，因此 ∂p/∂v 必须是使叉积给出法线且长度等于曲线宽度的向量。

For flat and cylinder curves, we transform ∂p/∂u to the intersection coordinate system. For flat curves, we know that ∂p/∂v lies in the x-y plane, is perpendicular to ∂p/∂u, and has length equal to hitWidth. We can find the 2D perpendicular vector using the same approach as was used earlier for the perpendicular curve segment boundary edges.

> 对于平面型和圆柱型曲线，我们将 ∂p/∂u 变换到求交坐标系。对于平面曲线，我们知道 ∂p/∂v 在 x-y 平面内，垂直于 ∂p/∂u，且长度等于 hitWidth。我们可以使用与之前垂直曲线段边界边相同的方法找到二维垂直向量。

The ∂p/∂v vector for cylinder curves is rotated around the dpduPlane axis so that its appearance resembles a cylindrical cross-section.

> 圆柱型曲线的 ∂p/∂v 向量围绕 dpduPlane 轴旋转，使其外观类似于圆柱截面。
