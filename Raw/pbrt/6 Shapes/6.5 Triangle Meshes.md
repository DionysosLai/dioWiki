## 6.5 Triangle Meshes

The triangle is one of the most commonly used shapes in computer graphics; complex scenes may be modeled using millions of triangles to achieve great detail.

> 三角形是计算机图形学中最常用的形状之一；复杂场景可以使用数百万个三角形来建模以实现丰富的细节。

A more memory-efficient representation than storing three vertex positions per triangle is to store entire triangle meshes with an array of vertex positions, where each triangle just stores three index offsets. By the Euler–Poincaré formula for closed meshes, there are approximately twice as many faces as vertices. Since each face references three vertices, every vertex is referenced on average six times. The amortized storage is 18 bytes per triangle (12 bytes for three 32-bit indices plus 6 bytes for half a vertex), versus 36 bytes when storing positions directly. Savings are even greater with per-vertex normals or texture coordinates.

> 比每个三角形存储三个顶点位置更节省内存的方式是存储整个三角形网格的顶点位置数组，每个三角形只存储三个索引偏移。根据封闭网格的 Euler–Poincaré 公式，面数约为顶点数的两倍。由于每个面引用三个顶点，每个顶点平均被引用六次。摊销存储为每个三角形 18 字节（三个 32 位索引 12 字节加半个顶点 6 字节），而直接存储位置需 36 字节。有逐顶点法线或纹理坐标时节省更多。

### 6.5.1 Mesh Representation and Storage

pbrt uses the TriangleMesh class to store shared mesh information including vertex positions, vertex indices, per-vertex normals n, tangent vectors s, and texture coordinates uv. Mesh data is exposed via public member variables. The vertexIndices pointer points to 3 × nTriangles values, and per-vertex pointers (if not nullptr) point to nVertices values.

> pbrt 使用 TriangleMesh 类存储共享的网格信息，包括顶点位置、顶点索引、逐顶点法线 n、切向量 s 和纹理坐标 uv。网格数据通过公有成员变量公开。vertexIndices 指针指向 3 × nTriangles 个值，逐顶点指针（非 nullptr 时）指向 nVertices 个值。

Although the constructor takes std::vector parameters, TriangleMesh stores plain pointers so that different meshes can share identical arrays in memory. The BufferCache class handles deduplication: its LookupOrAdd() method returns a pointer to a unique copy of each buffer. This can save significant memory—over 400 MB in the landscape scene.

> 虽然构造函数接受 std::vector 参数，TriangleMesh 存储原始指针以便不同网格共享内存中相同的数组。BufferCache 类处理去重：其 LookupOrAdd() 方法返回指向每个缓冲区唯一副本的指针。这可以显著节省内存——在景观场景中超过 400 MB。

BufferCache allows concurrent multi-threaded access. To avoid mutex contention, the cache is split into 64 independent shards, each with its own std::shared_mutex. Lookups use lock_shared() for concurrent reads; insertions acquire an exclusive lock. A double-check pattern handles the rare case where another thread inserts the same buffer first.

> BufferCache 允许多线程并发访问。为避免互斥锁竞争，缓存被分为 64 个独立分片，每个分片有自己的 std::shared_mutex。查找使用 lock_shared() 进行并发读取；插入获取排他锁。双重检查模式处理另一个线程先插入相同缓冲区的罕见情况。

Unlike other shapes that keep geometry in object space, triangle meshes are transformed into rendering space at construction time, saving per-ray transformation costs during rendering. The TriangleMesh also records reverseOrientation and transformSwapsHandedness once for the entire mesh rather than per triangle.

> 与其他形状将几何保持在物体空间不同，三角形网格在构造时变换到渲染空间，节省了渲染期间每条光线的变换开销。TriangleMesh 还为整个网格记录一次 reverseOrientation 和 transformSwapsHandedness，而非每个三角形都记录。

### 6.5.2 Triangle Class

The Triangle class implements the Shape interface for a single triangle. Each Triangle uses only 8 bytes: two ints (meshIndex and triIndex) indexing into a global vector of TriangleMesh pointers. If Triangle inherited from an abstract base class with virtual functions, the vtable pointer alone would double its size on 64-bit architectures.

> Triangle 类为单个三角形实现 Shape 接口。每个 Triangle 仅使用 8 字节：两个 int（meshIndex 和 triIndex）索引到 TriangleMesh 指针的全局向量。如果 Triangle 从带虚函数的抽象基类继承，仅虚表指针就会在 64 位架构上使其大小翻倍。

Vertex lookup requires indirection: find the mesh pointer, look up the three vertex indices, then read positions from the mesh's p array. The bounding box encompasses the three vertices directly in rendering space. Area is computed as half the parallelogram area formed by two edge vectors (via cross product length).

> 顶点查找需要间接操作：找到网格指针，查找三个顶点索引，然后从网格的 p 数组读取位置。包围盒直接在渲染空间中包围三个顶点。面积计算为两条边向量构成的平行四边形面积的一半（通过叉积长度）。

NormalBounds() returns the geometric normal direction, but must handle two subtleties: if per-vertex normals exist, the geometric normal is flipped to match their orientation (via FaceForward); otherwise, the normal is flipped if reverseOrientation XOR transformSwapsHandedness is true.

> NormalBounds() 返回几何法线方向，但必须处理两个细节：如果存在逐顶点法线，几何法线通过 FaceForward 翻转以匹配其朝向；否则，当 reverseOrientation 异或 transformSwapsHandedness 为 true 时翻转法线。

SolidAngle() computes the solid angle subtended by the triangle from a reference point using SphericalTriangleArea().

> SolidAngle() 使用 SphericalTriangleArea() 计算三角形从参考点所张的立体角。

### 6.5.3 Ray–Triangle Intersection

pbrt provides a stand-alone IntersectTriangle() function that takes a ray and three vertices directly, without requiring Triangle or TriangleMesh instances.

> pbrt 提供了一个独立的 IntersectTriangle() 函数，直接接受光线和三个顶点，无需 Triangle 或 TriangleMesh 实例。

The algorithm first transforms the ray into a coordinate system where the origin is at the ray's origin and the direction is along +z. This is done in three steps: (1) translate vertices by subtracting the ray origin, (2) permute axes so that the largest absolute component of the ray direction maps to z, and (3) apply a shear to align the ray direction with +z. These coefficients depend only on the ray and could be precomputed.

> 算法首先将光线变换到一个坐标系中，使原点在光线原点处、方向沿 +z 轴。分三步完成：(1) 将顶点减去光线原点进行平移，(2) 置换轴使光线方向绝对值最大的分量映射到 z，(3) 应用剪切使光线方向对齐 +z。这些系数仅依赖光线，可以预计算。

In this coordinate system, intersection reduces to a 2D test: whether (0,0) lies inside the triangle projected onto the xy plane. This is evaluated using signed edge functions e(p_i, p_j) = x_i·y_j − y_i·x_j, computed via DifferenceOfProducts for precision. If any edge function is exactly zero, double-precision recomputation is used (Section 6.8.4 explains why).

> 在此坐标系中，求交简化为二维测试：(0,0) 是否在三角形投影到 xy 平面内。使用有符号边函数 e(p_i, p_j) = x_i·y_j − y_i·x_j 求值，通过 DifferenceOfProducts 提高精度。如果任何边函数恰好为零，则使用双精度重新计算（6.8.4 节解释原因）。

If edge function signs differ, the point is outside the triangle. If their sum (the determinant) is zero, the ray approaches edge-on. Otherwise, the barycentric coordinates b0, b1, b2 are computed as e_i / det, and the intersection t is found by applying barycentric interpolation to the sheared z coordinates of the transformed vertices.

> 如果边函数符号不同，该点在三角形外部。如果它们的和（行列式）为零，光线从侧面接近。否则，重心坐标 b0、b1、b2 计算为 e_i / det，交点 t 通过对变换后顶点的剪切 z 坐标进行重心插值得到。

A scaled-t optimization avoids the division when the final t is clearly out of range. After conservative error bound checks (Section 6.8.7) to avoid false intersections with negative t, a TriangleIntersection with barycentrics and t is returned.

> 缩放 t 的优化在最终 t 明显超出范围时避免除法。在保守误差边界检查（6.8.7 节）避免负 t 的假交点后，返回包含重心坐标和 t 的 TriangleIntersection。

InteractionFromIntersection() is a static method that computes the SurfaceInteraction from barycentric coordinates. This design supports both CPU (Triangle class) and GPU (hardware ray intersection API) paths.

> InteractionFromIntersection() 是一个从重心坐标计算 SurfaceInteraction 的静态方法。此设计同时支持 CPU（Triangle 类）和 GPU（硬件光线求交 API）路径。

Partial derivatives ∂p/∂u and ∂p/∂v are computed by inverting a 2×2 matrix of uv differences. Degenerate cases (identical uvs, or dpdu × dpdv = 0) fall back to building a coordinate system from the geometric normal. The intersection point and uv are found by barycentric interpolation rather than the ray equation, yielding better accuracy.

> 偏导数 ∂p/∂u 和 ∂p/∂v 通过对 uv 差分的 2×2 矩阵求逆计算。退化情况（相同的 uv，或 dpdu × dpdv = 0）回退到从几何法线构建坐标系。交点和 uv 通过重心插值而非光线方程求得，精度更高。

The geometric normal is initialized from edge vectors (not dpdu × dpdv) to avoid dependence on potentially bad uv parameterizations. If per-vertex normals or tangents are provided, shading geometry is set up: the shading normal ns is interpolated from vertex normals, shading tangent ss from vertex tangents, and the bitangent ts = ns × ss. The tangent ss is then re-orthogonalized as ss = ts × ns. Normal partial derivatives ∂n/∂u and ∂n/∂v are computed by a similar 2×2 matrix inversion using vertex normal differences.

> 几何法线从边向量初始化（而非 dpdu × dpdv），以避免依赖可能不良的 uv 参数化。如果提供了逐顶点法线或切线，则设置着色几何：着色法线 ns 从顶点法线插值，着色切线 ss 从顶点切线插值，副切线 ts = ns × ss。然后切线 ss 被重新正交化为 ss = ts × ns。法线偏导数 ∂n/∂u 和 ∂n/∂v 通过使用顶点法线差分的类似 2×2 矩阵求逆计算。

### 6.5.4 Sampling

The uniform area sampling method maps a random sample u to barycentric coordinates via SampleUniformTriangle(), which uses an area-preserving mapping from the unit square to the unit triangle. Rather than reflecting samples across the diagonal (which would cause distant square samples to map nearby on the triangle), it translates points along the diagonal by a varying amount, yielding a constant Jacobian determinant.

> 均匀面积采样方法通过 SampleUniformTriangle() 将随机样本 u 映射到重心坐标，使用从单位正方形到单位三角形的保面积映射。它不是将样本沿对角线反射（那会导致正方形上远处的样本在三角形上映射到附近），而是沿对角线以变化的量平移点，产生常数雅可比行列式。

The surface normal for the sampled point accounts for per-vertex normals (via FaceForward) and the reverseOrientation/transformSwapsHandedness flags. The PDF is simply 1/Area.

> 采样点的表面法线考虑逐顶点法线（通过 FaceForward）和 reverseOrientation/transformSwapsHandedness 标志。PDF 简单地为 1/Area。

For sampling with respect to a reference point, solid angle sampling significantly reduces variance compared to area sampling. With area sampling, the 1/r² factor can have unbounded variation, especially for large triangles close to the reference point. With solid angle sampling, the estimator's non-constant factors are bounded between 0 and 1, reducing MSE by orders of magnitude in some scenes.

> 对于相对参考点的采样，立体角采样相比面积采样显著降低方差。使用面积采样时，1/r² 因子可能有无界变化，特别是对于靠近参考点的大三角形。使用立体角采样时，估计器的非常数因子限定在 0 到 1 之间，在某些场景中将均方误差降低数个数量级。

Numerically unstable cases (very small or very large solid angles) fall back to uniform area sampling. The spherical triangle sampling is implemented by SampleSphericalTriangle(), which works in two stages: (1) uniformly sample a sub-triangle area A' between 0 and the total spherical area A using u[0], finding a vertex c' along the arc from a to c; (2) sample along the arc from c' to b using u[1] with a density that produces uniform distribution inside the spherical triangle.

> 数值不稳定的情况（非常小或非常大的立体角）回退到均匀面积采样。球面三角形采样由 SampleSphericalTriangle() 实现，分两个阶段：(1) 使用 u[0] 在 0 和总球面面积 A 之间均匀采样子三角形面积 A'，在从 a 到 c 的弧上找到顶点 c'；(2) 使用 u[1] 沿从 c' 到 b 的弧采样，密度使球面三角形内产生均匀分布。

The implementation avoids subtracting π from the angle sum (which loses precision for small triangles) by working with A+π throughout. Finding c' requires computing cos(b') via the spherical law of cosines. The arc parameterization uses Gram-Schmidt orthogonalization. Barycentric coordinates for the sampled direction are recovered using part of the Möller–Trumbore ray–triangle intersection algorithm.

> 实现通过始终使用 A+π 来避免从角度和中减去 π（对小三角形会损失精度）。找到 c' 需要通过球面余弦定律计算 cos(b')。弧参数化使用 Gram-Schmidt 正交化。采样方向的重心坐标通过 Möller–Trumbore 光线-三角形求交算法的一部分恢复。

pbrt also approximates the cosθ factor at the reference point using warp product sampling. The cosθ function, when mapped back to the sampling domain through the inverse of the spherical triangle sampling algorithm, varies smoothly. It is approximated by a bilinear function with values at the four corners of the [0,1]² domain set to |cosθ| at the corresponding triangle vertices. The uniform sample is warped through SampleBilinear before being passed to the spherical triangle sampler. The overall PDF is the product of BilinearPDF and the spherical triangle PDF (1/solidAngle).

> pbrt 还通过 warp product 采样近似参考点处的 cosθ 因子。cosθ 函数通过球面三角形采样算法的逆映射回采样域后变化平滑。它被近似为双线性函数，[0,1]² 域四个角的值设为对应三角形顶点处的 |cosθ|。均匀样本先通过 SampleBilinear 变形，再传给球面三角形采样器。总 PDF 是 BilinearPDF 和球面三角形 PDF (1/solidAngle) 的乘积。

The PDF() method mirrors the sampling logic: it uses InvertSphericalTriangleSample() to recover the sample u from a given direction wi, then multiplies 1/solidAngle by BilinearPDF(u, w) when the warp is active. InvertSphericalTriangleSample() reverses both stages: it finds c' from the intersection of great circles through (b,w) and (c,a), computes the sub-triangle area ratio for u[0], and inverts the arc sampling equation for u[1].

> PDF() 方法镜像采样逻辑：使用 InvertSphericalTriangleSample() 从给定方向 wi 恢复样本 u，当 warp 激活时将 1/solidAngle 乘以 BilinearPDF(u, w)。InvertSphericalTriangleSample() 反转两个阶段：通过 (b,w) 和 (c,a) 的大圆交线找到 c'，计算子三角形面积比得到 u[0]，反转弧采样方程得到 u[1]。
