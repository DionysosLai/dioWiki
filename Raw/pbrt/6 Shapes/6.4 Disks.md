## 6.4 Disks

The disk is an interesting quadric since it has a particularly straightforward intersection routine that avoids solving the quadratic equation. In pbrt, a Disk is a circular disk of radius r at height h along the z axis.

> 圆盘是一个有趣的二次曲面，因为它有一个特别直接的交点求解过程，避免了求解二次方程。在 pbrt 中，Disk 是一个半径为 r、位于 z 轴高度 h 处的圆盘。

To describe partial disks, the user may specify a maximum φ value beyond which the disk is cut off. The disk can also be generalized to an annulus by specifying an inner radius. In parametric form, it is described by the standard parametric equations.

> 为了描述部分圆盘，用户可以指定一个最大 φ 值，超过该值后圆盘被截断。通过指定内半径，圆盘还可以推广为环形。在参数形式中，它由标准参数方程描述。

The Disk constructor directly initializes its various member variables from the values passed to it. We have omitted it here because it is trivial.

> Disk 构造函数直接从传入的值初始化其各种成员变量。我们在此省略它，因为它很简单。

### 6.4.1 Area and Bounding

Disks have easily computed surface area, since they are just portions of an annulus: Area = φ_max/2 × (r² - r_inner²).

> 圆盘的表面积很容易计算，因为它们只是环形的一部分：Area = φ_max/2 × (r² - r_inner²)。

The bounding method is also quite straightforward; it computes a bounding box centered at the height of the disk along z, with extent of radius in both the x and y directions.

> 包围方法也相当直接；它计算一个以圆盘沿 z 轴高度为中心的包围盒，在 x 和 y 方向上的范围为半径。

A disk has a single surface normal.

> 圆盘只有一个表面法线。

### 6.4.2 Intersection Tests

The Disk intersection test methods follow the same form as the earlier quadrics. We omit Intersect(), as it is exactly the same as Sphere::Intersect() and Cylinder::Intersect(), with calls to BasicIntersect() and then InteractionFromIntersection().

> Disk 交点测试方法遵循与之前二次曲面相同的形式。我们省略 Intersect()，因为它与 Sphere::Intersect() 和 Cylinder::Intersect() 完全相同，调用 BasicIntersect() 然后调用 InteractionFromIntersection()。

The basic intersection test for a ray with a disk is easy. The intersection of the ray with the plane that the disk lies in is found and then the intersection point is checked to see if it lies inside the disk.

> 光线与圆盘的基本交点测试很简单。首先找到光线与圆盘所在平面的交点，然后检查交点是否位于圆盘内部。

The first step is to compute the parametric t value where the ray intersects the plane that the disk lies in. We want to find t such that the z component of the ray's position is equal to the height of the disk. The intersection method computes a t value and checks to see if it is inside the valid range. If not, the routine can report that there is no intersection.

> 第一步是计算光线与圆盘所在平面相交处的参数 t 值。我们要找到 t 使得光线位置的 z 分量等于圆盘的高度。交点方法计算一个 t 值并检查它是否在有效范围内。如果不是，则例程报告没有交点。

If the ray is parallel to the disk's plane (i.e., the z component of its direction is zero), no intersection is reported. The case where a ray is both parallel to the disk's plane and lies within the plane is somewhat ambiguous, but it is most reasonable to define intersecting a disk edge-on as "no intersection." This case must be handled explicitly so that not-a-number floating-point values are not generated by the following code.

> 如果光线平行于圆盘平面（即其方向的 z 分量为零），则不报告交点。光线既平行于圆盘平面又位于平面内的情况有些模糊，但将侧面相交定义为"无交点"是最合理的。必须显式处理这种情况，以避免后续代码生成非数字（NaN）浮点值。

Now the intersection method can compute the point pHit where the ray intersects the plane. Once the plane intersection is known, an invalid intersection is returned if the distance from the hit to the center of the disk is more than Disk::radius or less than Disk::innerRadius. This check can be optimized by computing the squared distance to the center, taking advantage of the fact that the x and y coordinates of the center point are zero, and the z coordinate of pHit is equal to height.

> 现在交点方法可以计算光线与平面相交的点 pHit。一旦知道平面交点，如果从命中点到圆盘中心的距离大于 Disk::radius 或小于 Disk::innerRadius，则返回无效交点。这个检查可以通过计算到中心的平方距离来优化，利用中心点的 x 和 y 坐标为零，且 pHit 的 z 坐标等于高度这一事实。

If the distance check passes, a final test makes sure that the φ value of the hit point is between zero and φ_max. Because a ray can only intersect a disk once, there is no need to consider a second intersection if this test fails, as was the case with the two earlier quadrics.

> 如果距离检查通过，最后一个测试确保命中点的 φ 值在零和 φ_max 之间。因为光线只能与圆盘相交一次，如果这个测试失败，就不需要像前两个二次曲面那样考虑第二个交点。

Finding the SurfaceInteraction corresponding to a disk intersection follows the same process of inverting the parametric representation we have seen before.

> 找到与圆盘交点对应的 SurfaceInteraction 遵循我们之前见过的反转参数表示的相同过程。

The parameter u is first scaled to reflect the partial disk specified by φ_max, and v is computed by inverting the parametric equation. The equations for the partial derivatives at the hit point can be derived with a process similar to that used for the previous quadrics. Because the normal of a disk is the same everywhere, the partial derivatives dn/du and dn/dv are both trivially zero.

> 参数 u 首先被缩放以反映由 φ_max 指定的部分圆盘，v 通过反转参数方程来计算。命中点处偏导数的方程可以用与前面二次曲面类似的过程推导。因为圆盘的法线处处相同，偏导数 dn/du 和 dn/dv 都简单地为零。

As usual, the implementation of IntersectP() is straightforward.

> 和往常一样，IntersectP() 的实现很直接。

### 6.4.3 Sampling

The Disk area sampling method uses a utility routine, SampleUniformDiskConcentric(), that uniformly samples a unit disk. (It is defined in Section A.5.1.) The point that it returns is then scaled by the radius and offset in z so that it lies on the disk of a given radius and height. Note that our implementation here does not account for partial disks due to Disk::innerRadius being nonzero or Disk::phiMax being less than 2π. Fixing this bug is left for an exercise at the end of the chapter.

> Disk 面积采样方法使用一个工具函数 SampleUniformDiskConcentric()，它均匀采样一个单位圆盘。（它在 A.5.1 节中定义。）返回的点然后按半径缩放并在 z 方向偏移，使其位于给定半径和高度的圆盘上。注意我们这里的实现没有处理由于 Disk::innerRadius 非零或 Disk::phiMax 小于 2π 导致的部分圆盘情况。修复这个 bug 留作本章末尾的练习。

The same computation as in the Intersect() method gives the parametric (u,v) for the sampled point.

> 与 Intersect() 方法中相同的计算给出采样点的参数 (u,v)。

We do not provide a specialized solid angle sampling method for disks, but follow the same approach that we did for cylinders, sampling uniformly by area and then computing the probability density to be with respect to solid angle.

> 我们没有为圆盘提供专门的立体角采样方法，而是遵循与圆柱体相同的方法，按面积均匀采样，然后计算关于立体角的概率密度。
