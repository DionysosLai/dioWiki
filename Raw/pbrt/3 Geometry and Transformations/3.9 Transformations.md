## 3.9 Transformations

In general, a *transformation* is a mapping from points to points and from vectors to vectors. The transformation may be an arbitrary procedure. However, we will consider a subset of all possible transformations in this chapter. In particular, they will be:

> 一般来说，*变换*是从点到点和从向量到向量的映射。变换可以是任意过程。然而，在本章中我们将考虑所有可能变换的一个子集。特别地，它们将是：

- *Linear:* If is an arbitrary linear transformation and is an arbitrary scalar, then and. These two properties can greatly simplify reasoning about transformations.
- *Continuous:* Roughly speaking, maps the neighborhoods around and to neighborhoods around and.
- *One-to-one and invertible:* For each, maps to a single unique. Furthermore, there exists an inverse transform that maps back to.

> - *线性的：*两个基本性质大大简化了关于变换的推理。
> - *连续的：*粗略地说，将邻域映射到邻域。
> - *一一对应且可逆的：*对于每个输入，映射到唯一的输出。此外，存在一个逆变换将其映射回原来。

We will often want to take a point, vector, or normal defined with respect to one coordinate frame and find its coordinate values with respect to another frame. Using basic properties of linear algebra, a matrix can be shown to express the linear transformation of a point or vector from one frame to another. There are two different (and incompatible!) ways that a matrix can be interpreted:

> 我们经常需要将相对于一个坐标框架定义的点、向量或法线，求出其相对于另一个框架的坐标值。利用线性代数的基本性质，可以证明矩阵能够表达点或向量从一个框架到另一个框架的线性变换。矩阵可以以两种不同的（且不兼容的！）方式来解释：

- *Transformation within the frame:* Given a point, the matrix could express how to compute a *new* point in the same frame that represents the transformation of the original point.
- *Transformation from one frame to another:* A matrix can express the coordinates of a point or vector in a new frame in terms of the coordinates in the original frame.

> - *框架内的变换：*给定一个点，矩阵可以表达如何计算同一框架中代表原始点变换的*新*点。
> - *从一个框架到另一个框架的变换：*矩阵可以用原始框架中的坐标来表达点或向量在新框架中的坐标。

Most uses of transformations in pbrt are for transforming points from one frame to another.

> pbrt 中变换的大多数用途是将点从一个框架变换到另一个。

In general, transformations make it possible to work in the most convenient coordinate space. For example, we can write routines that define a virtual camera, assuming that the camera is located at the origin, looks down the axis, and has the axis pointing up and the axis pointing right. To place the camera at any point in the scene looking in any direction, we construct a transformation that maps points in the scene's coordinate system to the camera's coordinate system.

> 一般来说，变换使得可以在最方便的坐标空间中工作。例如，我们可以编写定义虚拟相机的例程，假设相机位于原点，朝 z 轴方向看，y 轴朝上，x 轴朝右。要将相机放置在场景中任意位置朝任意方向看，我们构造一个将场景坐标系中的点映射到相机坐标系的变换。

### 3.9.1 Homogeneous Coordinates

Given a frame, there is ambiguity between the representation of a point and a vector with the same coordinates. Using homogeneous representations, the fourth coordinate (the *weight*) distinguishes them: it is 1 for points and 0 for vectors.

> 给定一个框架，具有相同坐标的点和向量的表示之间存在歧义。使用齐次表示，第四个坐标（*权重*）区分了它们：点的权重为 1，向量的为 0。

We will not use homogeneous coordinates explicitly in our code; there is no HomogeneousPoint class in pbrt. However, the various transformation routines will implicitly convert points, vectors, and normals to homogeneous form, transform them, and then convert back.

> 我们不会在代码中显式使用齐次坐标；pbrt 中没有 HomogeneousPoint 类。然而，各种变换例程将隐式地将点、向量和法线转换为齐次形式，进行变换，然后转换回来。

### 3.9.2 Transform Class Definition

The Transform class represents a transformation. Its implementation is in the files util/transform.h and util/transform.cpp. The transformation matrix is represented by the elements of the matrix m, which is a SquareMatrix<4> object stored in *row-major* form. For convenience, the Transform also stores the inverse of m in its mInv member variable.

> Transform 类表示一个变换。其实现在 util/transform.h 和 util/transform.cpp 文件中。变换矩阵由矩阵 m 的元素表示，它是一个以*行主序*存储的 SquareMatrix<4> 对象。为方便起见，Transform 还在其 mInv 成员变量中存储 m 的逆矩阵。

This representation is relatively memory hungry: a Transform requires 128 bytes of storage. Therefore, Shapes in pbrt store a pointer to a Transform and the scene specification code uses an InternCache of Transforms to ensure that all shapes sharing the same transformation point to a single instance in memory.

> 这种表示相对耗内存：一个 Transform 需要 128 字节的存储空间。因此，pbrt 中的 Shape 存储指向 Transform 的指针，场景规范代码使用 Transform 的 InternCache 来确保共享相同变换的所有形状指向内存中的单个实例。

### 3.9.3 Basic Operations

When a new Transform is created, it defaults to the *identity transformation*. A Transform can also be created from a given matrix, in which case the matrix must be explicitly inverted. If the matrix is degenerate and cannot be inverted, mInv is initialized with floating-point not-a-number values, which poison computations that involve them.

> 当创建新的 Transform 时，它默认为*单位变换*。Transform 也可以从给定的矩阵创建，在这种情况下矩阵必须被显式求逆。如果矩阵是退化的且无法求逆，mInv 将被初始化为浮点非数字值，这些值会"毒化"涉及它们的计算。

The most commonly used constructor takes a reference to the transformation matrix along with an explicitly provided inverse. This avoids the expense and potential loss of numeric accuracy from computing a general matrix inverse.

> 最常用的构造函数接受变换矩阵的引用以及显式提供的逆矩阵。这避免了计算通用矩阵逆的开销和潜在的数值精度损失。

The Transform representing the inverse can be returned by just swapping the roles of mInv and m.

> 表示逆变换的 Transform 可以通过简单地交换 mInv 和 m 的角色来返回。

### 3.9.4 Translations

One of the simplest transformations is the *translation transformation*. When applied to a point, it translates the coordinates by offsets in x, y, and z. Translation only affects points, leaving vectors unchanged.

> 最简单的变换之一是*平移变换*。当应用于点时，它将坐标按 x、y 和 z 的偏移量平移。平移只影响点，向量保持不变。

### 3.9.5 Scaling

Another basic transformation is the *scale transformation*. It has the effect of multiplying components by scale factors in x, y, and z. We can differentiate between *uniform scaling*, where all three scale factors have the same value, and *nonuniform scaling*, where they may have different values.

> 另一个基本变换是*缩放变换*。它的效果是将分量按 x、y 和 z 的缩放因子相乘。我们可以区分*均匀缩放*（所有三个缩放因子相同）和*非均匀缩放*（它们可能有不同的值）。

### 3.9.6 x, y, and z Axis Rotations

Another useful type of transformation is the *rotation transformation*. Rotations have the property that the inverse equals the transpose, since the rotation matrix is *orthogonal*. For a left-handed coordinate system, the matrix for clockwise rotation around the x axis is well-defined, and similar matrices exist for y and z rotations.

> 另一种有用的变换类型是*旋转变换*。旋转具有逆等于转置的性质，因为旋转矩阵是*正交*的。对于左手坐标系，围绕 x 轴顺时针旋转的矩阵是明确定义的，y 和 z 旋转也有类似的矩阵。

### 3.9.7 Rotation around an Arbitrary Axis

We also provide a routine to compute the transformation that represents rotation around an arbitrary axis. A more elegant derivation can be constructed with vector algebra by computing the vector along the axis, constructing basis vectors in the perpendicular plane, and rotating those.

> 我们还提供了计算围绕任意轴旋转变换的例程。可以用向量代数构建更优雅的推导：计算沿轴的向量，在垂直平面中构建基向量，然后旋转它们。

### 3.9.8 Rotating One Vector to Another

It is sometimes useful to find the transformation that performs a rotation that aligns one unit vector with another. The approach used is based on finding a pair of reflection transformations using the *Householder matrix*, which reflects the given vector to its negation while leaving all vectors orthogonal to it unchanged. The product of two such reflections gives the desired rotation.

> 有时需要找到将一个单位向量对齐到另一个的旋转变换。使用的方法基于利用 *Householder 矩阵*找到一对反射变换，该矩阵将给定向量反射为其相反方向，同时保持所有正交向量不变。两个这样的反射的乘积给出了所需的旋转。

### 3.9.9 The Look-at Transformation

The *look-at transformation* is particularly useful for placing a camera in the scene. The caller specifies the desired position of the camera, a point the camera is looking at, and an "up" vector that orients the camera along the viewing direction. All of these values are typically given in world-space coordinates; this gives a transformation from world space to camera space.

> *观察变换*（look-at 变换）对于在场景中放置相机特别有用。调用者指定相机的期望位置、相机注视的点以及沿观察方向定向相机的"上"向量。所有这些值通常以世界空间坐标给出；这给出了从世界空间到相机空间的变换。

In order to find the entries of the look-at transformation matrix, we use the principle that the columns of a transformation matrix give the effect of the transformation on the basis of a coordinate system.

> 为了找到观察变换矩阵的元素，我们使用这样的原理：变换矩阵的列给出了变换对坐标系基的效果。
