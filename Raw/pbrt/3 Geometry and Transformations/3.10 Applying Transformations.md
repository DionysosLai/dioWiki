## 3.10 Applying Transformations

We can now define routines that perform the appropriate matrix multiplications to transform points and vectors. We will overload the function application operator to describe these transformations; this lets us write code like:

> 我们现在可以定义执行适当矩阵乘法来变换点和向量的例程。我们将重载函数调用运算符来描述这些变换；这允许我们编写如下代码：

```
Point3f p = ...; Transform T = ...; Point3f pNew = T(p);
```

### 3.10.1 Points

The point transformation routine takes a point and implicitly represents it as the homogeneous column vector. It then transforms the point by premultiplying this vector with the transformation matrix. Finally, it divides by w to convert back to a nonhomogeneous point representation. For efficiency, this method skips the division by the homogeneous weight when w == 1, which is common for most of the transformations that will be used in pbrt—only the projective transformations defined in Chapter 5 will require this division.

> 点变换例程接受一个点，并隐式地将其表示为齐次列向量。然后通过将该向量左乘变换矩阵来变换该点。最后除以 w 以转换回非齐次点表示。为了效率，当 w == 1 时该方法跳过除以齐次权重的操作，这对于 pbrt 中使用的大多数变换来说很常见——只有第 5 章中定义的投影变换需要此除法。

The Transform class also provides a corresponding `ApplyInverse()` method for each type it transforms. The one for `Point3` applies its inverse transformation to the given point. Calling this method is more succinct and generally more efficient than calling `Transform::Inverse()` and then calling its `operator()`.

> Transform 类还为其变换的每种类型提供了相应的 `ApplyInverse()` 方法。用于 `Point3` 的版本将其逆变换应用于给定点。调用此方法比先调用 `Transform::Inverse()` 再调用其 `operator()` 更简洁且通常更高效。

All subsequent types that can be transformed also have an `ApplyInverse()` method, though we will not include them in the book text.

> 所有后续可变换的类型也都有 `ApplyInverse()` 方法，但我们不会在书中列出它们。

### 3.10.2 Vectors

The transformations of vectors can be computed in a similar fashion. However, the multiplication of the matrix and the column vector is simplified since the implicit homogeneous coordinate is zero.

> 向量的变换可以用类似的方式计算。不过，由于隐式齐次坐标为零，矩阵与列向量的乘法得到了简化。

### 3.10.3 Normals

Figure 3.29: Transforming Surface Normals. (a) Original circle, with the normal at a point indicated by an arrow. (b) When scaling the circle to be half as tall in the y direction, simply treating the normal as a direction and scaling it in the same manner gives a normal that is no longer perpendicular to the surface. (c) A properly transformed normal.

> 图 3.29：变换表面法线。(a) 原始圆，箭头表示某点处的法线。(b) 当将圆在 y 方向缩放到一半高度时，简单地将法线当作方向并以相同方式缩放会得到不再垂直于表面的法线。(c) 正确变换后的法线。

Normals do not transform in the same way that vectors do, as shown in Figure 3.29. Although tangent vectors at a surface transform in the straightforward way, normals require special treatment. Because the normal vector and any tangent vector on the surface are orthogonal by construction, we know that n · t = 0.

> 如图 3.29 所示，法线的变换方式与向量不同。虽然表面上的切线向量以直接方式变换，但法线需要特殊处理。由于法线向量与表面上任何切线向量在构造上是正交的，我们知道 n · t = 0。

When we transform a point on the surface by some matrix M, the new tangent vector at the transformed point is M·t. The transformed normal should be equal to S·n for some 4×4 matrix S. To maintain the orthogonality requirement, we must have (S·n)ᵀ(M·t) = 0, which gives nᵀSᵀMt = 0. This condition holds if SᵀM = I, the identity matrix. Therefore, S = (M⁻¹)ᵀ, and we see that normals must be transformed by the inverse transpose of the transformation matrix. This detail is one of the reasons why Transforms maintain their inverses.

> 当我们用某个矩阵 M 变换表面上的一个点时，变换后点处的新切线向量为 M·t。变换后的法线应等于 S·n（其中 S 为某个 4×4 矩阵）。为维持正交性要求，我们必须有 (S·n)ᵀ(M·t) = 0，这给出 nᵀSᵀMt = 0。当 SᵀM = I（单位矩阵）时此条件成立。因此 S = (M⁻¹)ᵀ，我们看到法线必须通过变换矩阵的逆转置来变换。这一细节是 Transform 维护其逆矩阵的原因之一。

Note that this method does not explicitly compute the transpose of the inverse when transforming normals. It just indexes into the inverse matrix in a different order (compare to the code for transforming `Vector3f`s).

> 注意，该方法在变换法线时并未显式计算逆矩阵的转置。它只是以不同的顺序索引逆矩阵（与变换 `Vector3f` 的代码比较即可看出）。

### 3.10.4 Rays

Transforming rays is conceptually straightforward: it is a matter of transforming the constituent origin and direction and copying the other data members. (pbrt also provides a similar method for transforming `RayDifferential`s.)

> 变换光线在概念上很简单：只需变换其组成的原点和方向，并复制其他数据成员。（pbrt 还为变换 `RayDifferential` 提供了类似方法。）

The approach used in pbrt to manage floating-point round-off error introduces some subtleties that require a small adjustment to the transformed ray origin. The "Offset ray origin to edge of error bounds and compute tMax" fragment handles these details; it is defined in Section 6.8.6, where round-off error and pbrt's mechanisms for dealing with it are discussed.

> pbrt 中用于管理浮点舍入误差的方法引入了一些细微之处，需要对变换后的光线原点进行微小调整。"将光线原点偏移到误差边界边缘并计算 tMax"代码片段处理了这些细节；它在第 6.8.6 节中定义，该节讨论了舍入误差及 pbrt 的处理机制。

### 3.10.6 Composition of Transformations

Having defined how the matrices representing individual types of transformations are constructed, we can now consider an aggregate transformation resulting from a series of individual transformations. We will finally see the real value of representing transformations with matrices.

> 在定义了各种类型变换矩阵的构造方式之后，我们现在可以考虑由一系列单独变换组合而成的聚合变换。我们将最终看到用矩阵表示变换的真正价值。

Consider a series of transformations A, B, C. We would like to compute a new transformation T such that applying T gives the same result as applying each of A, B, and C in reverse order; that is, T(p) = A(B(C(p))). Such a transformation can be computed by multiplying the matrices of the transformations A, B, and C together. In pbrt, we can write:

> 考虑一系列变换 A、B、C。我们希望计算一个新变换 T，使得应用 T 与按逆序依次应用 A、B、C 产生相同结果；即 T(p) = A(B(C(p)))。这样的变换可以通过将变换 A、B、C 的矩阵相乘来计算。在 pbrt 中，我们可以写：

```
Transform T = A * B * C;
```

Then we can apply T to `Point3f`s p as usual, `Point3f pp = T(p)`, instead of applying each transformation in turn: `Point3f pp = A(B(C(p)))`.

> 然后我们可以像往常一样将 T 应用于 `Point3f` p：`Point3f pp = T(p)`，而不是逐一应用每个变换：`Point3f pp = A(B(C(p)))`。

We overload the C++ `*` operator in the Transform class to compute the new transformation that results from postmultiplying a transformation with another transformation t2. In matrix multiplication, the (i,j)th element of the resulting matrix is the inner product of the ith row of the first matrix with the jth column of the second.

> 我们在 Transform 类中重载 C++ `*` 运算符，以计算一个变换右乘另一个变换 t2 所得到的新变换。在矩阵乘法中，结果矩阵的第 (i,j) 个元素是第一个矩阵第 i 行与第二个矩阵第 j 列的内积。

The inverse of the resulting transformation is equal to the product of `t2.mInv * mInv`. This is a result of the matrix identity (AB)⁻¹ = B⁻¹A⁻¹.

> 所得变换的逆等于 `t2.mInv * mInv` 的乘积。这是矩阵恒等式 (AB)⁻¹ = B⁻¹A⁻¹ 的结果。

### 3.10.7 Transformations and Coordinate System Handedness

Certain types of transformations change a left-handed coordinate system into a right-handed one, or vice versa. Some routines will need to know if the handedness of the source coordinate system is different from that of the destination. In particular, routines that want to ensure that a surface normal always points "outside" of a surface might need to flip the normal's direction after transformation if the handedness changes.

> 某些类型的变换会将左手坐标系变为右手坐标系，反之亦然。一些例程需要知道源坐标系的手性是否与目标坐标系不同。特别是，那些希望确保表面法线始终指向表面"外部"的例程，如果手性发生变化，可能需要在变换后翻转法线方向。

Fortunately, it is easy to tell if handedness is changed by a transformation: it happens only when the determinant of the transformation's upper-left 3×3 submatrix is negative.

> 幸运的是，很容易判断变换是否改变了手性：仅当变换的左上角 3×3 子矩阵的行列式为负时才会发生。

### 3.10.8 Vector Frames

It is sometimes useful to define a rotation that aligns three orthonormal vectors in a coordinate system with the x, y, and z axes. Applying such a transformation to direction vectors in that coordinate system can simplify subsequent computations. For example, in pbrt, BSDF evaluation is performed in a coordinate system where the surface normal is aligned with the z axis. Among other things, this makes it possible to efficiently evaluate trigonometric functions using functions like the `CosTheta()` function that was introduced in Section 3.8.3.

> 有时定义一个旋转来将坐标系中三个正交归一化向量与 x、y、z 轴对齐是很有用的。将这样的变换应用于该坐标系中的方向向量可以简化后续计算。例如，在 pbrt 中，BSDF 求值在表面法线与 z 轴对齐的坐标系中执行。除其他好处外，这使得利用第 3.8.3 节中介绍的 `CosTheta()` 等函数高效求值三角函数成为可能。

The Frame class efficiently represents and performs such transformations, avoiding the full generality (and hence, complexity) of the Transform class. It only needs to store a 3×3 matrix, and storing the inverse is unnecessary since it is just the matrix's transpose, given orthonormal basis vectors.

> Frame 类高效地表示和执行此类变换，避免了 Transform 类的全部通用性（以及随之而来的复杂性）。它只需存储一个 3×3 矩阵，并且无需存储逆矩阵，因为对于正交归一化基向量，逆矩阵就是其转置。

Given three orthonormal vectors x, y, and z, the matrix M that transforms vectors into their space is formed with these vectors as rows. The Frame stores this matrix using three `Vector3f`s.

> 给定三个正交归一化向量 x、y、z，将向量变换到其空间的矩阵 M 以这些向量为行构成。Frame 使用三个 `Vector3f` 存储该矩阵。

The three basis vectors can be specified explicitly; in debug builds, `DCHECK()`s in the constructor ensure that the provided vectors are orthonormal.

> 三个基向量可以显式指定；在调试构建中，构造函数中的 `DCHECK()` 确保提供的向量是正交归一化的。

Frame also provides convenience methods that construct a frame from just two of the basis vectors, using the cross product to compute the third.

> Frame 还提供了便利方法，仅从两个基向量构造 frame，使用叉积计算第三个。

Only the z axis vector can be provided as well, in which case the others are set arbitrarily using the `CoordinateSystem()` function.

> 也可以仅提供 z 轴向量，此时其他向量通过 `CoordinateSystem()` 函数任意设定。

A variety of other functions, not included here, allow specifying a frame using a normal vector and specifying it via just the x or y basis vector.

> 此处未列出的其他多种函数允许使用法线向量指定 frame，或仅通过 x 或 y 基向量来指定。

Transforming a vector into the frame's coordinate space is done using the matrix M. Because `Vector3f`s were used to store its rows, the matrix-vector product can be expressed as three dot products.

> 将向量变换到 frame 的坐标空间使用矩阵 M 完成。由于使用 `Vector3f` 存储其行，矩阵-向量乘积可以表示为三个点积。

A `ToLocal()` method is also provided for normal vectors. In this case, we do not need to compute the inverse transpose of M for the transformation of normals (recall the discussion of transforming normals in Section 3.10.3). Because M is an orthonormal matrix (its rows and columns are mutually orthogonal and unit length), its inverse is equal to its transpose, so it is its own inverse transpose already.

> 还为法线向量提供了 `ToLocal()` 方法。在这种情况下，我们不需要计算 M 的逆转置来变换法线（回忆第 3.10.3 节中关于法线变换的讨论）。因为 M 是正交矩阵（其行和列相互正交且为单位长度），其逆等于其转置，所以它本身就是自己的逆转置。

The method that transforms vectors out of the frame's local space transposes M to find its inverse before multiplying by the vector. In this case, the resulting computation can be expressed as the sum of three scaled versions of the matrix columns. As before, surface normals transform as regular vectors. (That method is not included here.)

> 将向量从 frame 的局部空间变换出来的方法先转置 M 以求其逆，然后乘以向量。在这种情况下，计算结果可以表示为矩阵列的三个缩放版本之和。与之前一样，表面法线像普通向量一样变换。（该方法未在此处列出。）

For convenience, there is a Transform constructor that takes a Frame. Its simple implementation is not included here.

> 为方便起见，有一个接受 Frame 的 Transform 构造函数。其简单实现未在此处列出。
