## 3.6 Rays

A *ray* is a semi-infinite line specified by its origin and direction; see Figure [3.8](#fig:rayexample). pbrt represents [Ray](#Ray) s using a [Point3f](https://pbr-book.org/4ed/Geometry_and_Transformations/Points.html#Point3f) for the origin and a [Vector3f](https://pbr-book.org/4ed/Geometry_and_Transformations/Vectors.html#Vector3f) for the direction; there is no need for non- Float -based rays in pbrt. See the files [ray.h](https://github.com/mmp/pbrt-v4/tree/master/src/pbrt/ray.h) and [ray.cpp](https://github.com/mmp/pbrt-v4/tree/master/src/pbrt/ray.cpp) in the pbrt source code distribution for the implementation of the Ray class.

> *光线*是由其原点和方向指定的半无限直线；见图 3.8。pbrt 使用 Point3f 表示原点，Vector3f 表示方向来表示 Ray；pbrt 中不需要非 Float 类型的光线。有关 Ray 类的实现，请参见 pbrt 源代码分发中的 ray.h 和 ray.cpp 文件。

[![](https://pbr-book.org/4ed/Geometry_and_Transformations/pha03f08.svg)](https://pbr-book.org/4ed/Geometry_and_Transformations/pha03f08.svg)

Figure 3.8: A ray is a semi-infinite line defined by its origin and its direction vector.

> 图 3.8：光线是由其原点和方向向量定义的半无限直线。

Because we will be referring to these variables often throughout the code, the origin and direction members of a [Ray](#Ray) are succinctly named o and d. Note that we again make the data publicly available for convenience.

> 因为我们将在整个代码中经常引用这些变量，Ray 的原点和方向成员被简洁地命名为 o 和 d。注意我们再次将数据设为公共可访问以便使用。

The *parametric form* of a ray expresses it as a function of a scalar value, giving the set of points that the ray passes through:

> 光线的*参数形式*将其表示为标量值的函数，给出光线经过的点集：

([3.4](#eq:ray))

The Ray class overloads the function application operator for rays in order to match the notation in Equation ([3.4](#eq:ray)).

> Ray 类重载了光线的函数调用运算符，以匹配公式 (3.4) 中的符号表示。

Given this method, when we need to find the point at a particular position along a ray, we can write code like:

> 有了这个方法，当我们需要找到光线上某个特定位置的点时，可以编写如下代码：

Ray r(Point3f(0, 0, 0), Vector3f(1, 2, 3)); Point3f p = r(1.7);

Each ray also has a time value associated with it. In scenes with animated objects, the rendering system constructs a representation of the scene at the appropriate time for each ray.

> 每条光线还有一个关联的时间值。在包含动画物体的场景中，渲染系统为每条光线在适当的时间构建场景表示。

Each ray also records the medium at its origin. The [Medium](https://pbr-book.org/4ed/Volume_Scattering/Media.html#Medium) class, which will be introduced in Section [11.4](https://pbr-book.org/4ed/Volume_Scattering/Media.html#sec:media), encapsulates the (potentially spatially varying) properties of participating media such as a foggy atmosphere, smoke, or scattering liquids like milk. Associating this information with rays makes it possible for other parts of the system to account correctly for the effect of rays passing from one medium to another.

> 每条光线还记录其原点处的介质。Medium 类将在第 11.4 节介绍，它封装了参与介质（如雾气大气、烟雾或像牛奶这样的散射液体）的（可能空间变化的）属性。将此信息与光线关联使得系统的其他部分能够正确考虑光线从一种介质传播到另一种介质的效果。

Constructing [Ray](#Ray) s is straightforward. The default constructor relies on the [Point3f](https://pbr-book.org/4ed/Geometry_and_Transformations/Points.html#Point3f) and [Vector3f](https://pbr-book.org/4ed/Geometry_and_Transformations/Vectors.html#Vector3f) constructors to set the origin and direction to. Alternately, a particular point and direction can be provided. If an origin and direction are provided, the constructor allows values to be given for the ray's time and medium.

> 构造 Ray 很简单。默认构造函数依赖 Point3f 和 Vector3f 构造函数将原点和方向设置为零。或者，可以提供特定的点和方向。如果提供了原点和方向，构造函数允许为光线的时间和介质提供值。

### 3.6.1 Ray Differentials

To be able to perform better antialiasing with the texture functions defined in Chapter [10](https://pbr-book.org/4ed/Textures_and_Materials.html#chap:texture), pbrt makes use of the [RayDifferential](#RayDifferential) class, which is a subclass of [Ray](#Ray) that contains additional information about two auxiliary rays. These extra rays represent camera rays offset by one sample in the and direction from the main ray on the film plane. By determining the area that these three rays project to on an object being shaded, a [Texture](https://pbr-book.org/4ed/Textures_and_Materials/Texture_Interface_and_Basic_Textures.html#Texture) can estimate an area to average over for proper antialiasing (Section [10.1](https://pbr-book.org/4ed/Textures_and_Materials/Texture_Sampling_and_Antialiasing.html#sec:texture-anti-aliasing)).

> 为了能够使用第 10 章定义的纹理函数执行更好的抗锯齿，pbrt 使用了 RayDifferential 类，它是 Ray 的子类，包含关于两条辅助光线的额外信息。这些额外的光线代表在胶片平面上从主光线沿 x 和 y 方向各偏移一个采样的相机光线。通过确定这三条光线投影到被着色物体上的面积，Texture 可以估计出用于正确抗锯齿的平均面积（第 10.1 节）。

Because [RayDifferential](#RayDifferential) inherits from [Ray](#Ray), geometric interfaces in the system can be written to take const Ray & parameters, so that either a Ray or RayDifferential can be passed to them. Only the routines that need to account for antialiasing and texturing require RayDifferential parameters.

> 因为 RayDifferential 继承自 Ray，系统中的几何接口可以编写为接受 const Ray & 参数，这样 Ray 或 RayDifferential 都可以传递给它们。只有需要处理抗锯齿和纹理的例程才需要 RayDifferential 参数。

In some cases, differential rays may not be available. Routines that take [RayDifferential](#RayDifferential) parameters should check the hasDifferentials member variable before accessing the differential rays' origins or directions.

> 在某些情况下，差分光线可能不可用。接受 RayDifferential 参数的例程应在访问差分光线的原点或方向之前检查 hasDifferentials 成员变量。

[Camera](https://pbr-book.org/4ed/Cameras_and_Film/Camera_Interface.html#Camera) implementations in pbrt compute differentials for rays leaving the camera under the assumption that camera rays are spaced one pixel apart. Integrators usually generate multiple camera rays per pixel, in which case the actual distance between samples is lower and the differentials should be updated accordingly; if this factor is not accounted for, then textures in images will generally be too blurry. The ScaleDifferentials() method below takes care of this, given an estimated sample spacing of s.

> pbrt 中的 Camera 实现在假设相机光线间隔一个像素的前提下计算离开相机的光线的差分。积分器通常每个像素生成多条相机光线，在这种情况下采样之间的实际距离更小，差分应相应更新；如果不考虑这一因素，图像中的纹理通常会过于模糊。下面的 ScaleDifferentials() 方法在给定估计的采样间距 s 的情况下处理此问题。
