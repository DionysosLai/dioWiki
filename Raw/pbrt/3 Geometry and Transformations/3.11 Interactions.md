## 3.11 Interactions

The last abstractions in this chapter, SurfaceInteraction and MediumInteraction, respectively represent local information at points on surfaces and in participating media. For example, the ray–shape intersection routines in Chapter 6 return information about the local differential geometry at intersection points in a SurfaceInteraction. Later, the texturing code in Chapter 10 computes material properties using values from the SurfaceInteraction. The closely related MediumInteraction class is used to represent points where light interacts with participating media like smoke or clouds. The implementations of all of these classes are in the files interaction.h and interaction.cpp.

> 本章最后的抽象——SurfaceInteraction 和 MediumInteraction——分别表示表面上点和参与介质中点的局部信息。例如，第 6 章中的光线-形体求交例程在 SurfaceInteraction 中返回交点处的局部微分几何信息。之后，第 10 章的纹理代码使用 SurfaceInteraction 中的值来计算材质属性。密切相关的 MediumInteraction 类用于表示光与烟雾或云等参与介质发生交互的点。所有这些类的实现位于 interaction.h 和 interaction.cpp 文件中。

Both SurfaceInteraction and MediumInteraction inherit from a generic Interaction class that provides common member variables and methods, which allows parts of the system for which the differences between surface and medium interactions do not matter to be implemented purely in terms of Interactions.

> SurfaceInteraction 和 MediumInteraction 都继承自一个通用的 Interaction 类，该类提供公共的成员变量和方法，这使得系统中不关心表面交互和介质交互之间差异的部分可以纯粹基于 Interaction 来实现。

A variety of Interaction constructors are available; depending on what sort of interaction is being constructed and what sort of information about it is relevant, corresponding sets of parameters are accepted. The most general constructor takes a `Point3fi` for the position, a `Normal3f` for the surface normal, `Point2f` for UV coordinates, a `Vector3f` for the outgoing direction, and a `Float` for time.

> 有多种 Interaction 构造函数可用；根据正在构造的交互类型及其相关信息，接受相应的参数集。最通用的构造函数接受 `Point3fi` 表示位置、`Normal3f` 表示表面法线、`Point2f` 表示 UV 坐标、`Vector3f` 表示出射方向以及 `Float` 表示时间。

All interactions have a point associated with them. This point is stored using the `Point3fi` class, which uses an `Interval` to represent each coordinate value. Storing a small interval of floating-point values rather than a single `Float` makes it possible to represent bounds on the numeric error in the intersection point, as occurs when the point was computed by a ray intersection calculation. This information will be useful for avoiding incorrect self-intersections for rays leaving surfaces, as will be discussed in Section 6.8.6.

> 所有交互都有一个关联的点。该点使用 `Point3fi` 类存储，它使用 `Interval` 来表示每个坐标值。存储一个浮点值的小区间而非单个 `Float`，使得表示交点中数值误差的边界成为可能（例如当该点由光线求交计算得出时）。此信息对于避免从表面出发的光线的不正确自相交很有用，这将在第 6.8.6 节中讨论。

Interaction provides a convenience method `p()` that returns a regular `Point3f` for the interaction point for the parts of the system that do not need to account for any error in it (e.g., the texture evaluation routines).

> Interaction 提供了一个便利方法 `p()`，为系统中不需要考虑误差的部分（例如纹理求值例程）返回交互点的常规 `Point3f`。

All interactions also have a time associated with them. Among other uses, this value is necessary for setting the time of a spawned ray leaving the interaction.

> 所有交互还有一个关联的时间。除其他用途外，该值用于设置从交互点出发的派生光线的时间。

For interactions that lie along a ray (either from a ray–shape intersection or from a ray passing through participating media), the negative ray direction is stored in the `wo` member variable, which corresponds to ω_o, the notation we use for the outgoing direction when computing lighting at points. For other types of interaction points where the notion of an outgoing direction does not apply (e.g., those found by randomly sampling points on the surface of shapes), `wo` has the value (0, 0, 0).

> 对于沿光线的交互（无论来自光线-形体求交还是光线穿过参与介质），负光线方向存储在 `wo` 成员变量中，对应于 ω_o，即我们在计算点上光照时用于表示出射方向的符号。对于出射方向概念不适用的其他类型的交互点（例如，通过在形体表面随机采样点获得的），`wo` 的值为 (0, 0, 0)。

For interactions on surfaces, `n` stores the surface normal at the point and `uv` stores its parametric coordinates. It is fair to ask, why are these values stored in the base Interaction class rather than in SurfaceInteraction? The reason is that there are some parts of the system that *mostly* do not care about the distinction between surface and medium interactions—for example, some of the routines that sample points on light sources given a point to be illuminated. Those make use of these values if they are available and ignore them if they are set to zero. By accepting the small dissonance of having them in the wrong place here, the implementations of those methods and the code that calls them is made that much simpler.

> 对于表面上的交互，`n` 存储该点的表面法线，`uv` 存储其参数坐标。有人可能会问，为什么这些值存储在基类 Interaction 中而不是在 SurfaceInteraction 中？原因是系统中有些部分*基本上*不关心表面交互和介质交互的区别——例如，某些根据待照明点在光源上采样点的例程。这些例程在值可用时使用它们，设为零时忽略它们。通过接受将它们放在"错误"位置这一小小的不协调，那些方法的实现及调用它们的代码变得简单得多。

It is possible to check if a pointer or reference to an Interaction is one of the two subclasses. A nonzero surface normal is used as a distinguisher for a surface interaction.

> 可以检查指向 Interaction 的指针或引用属于两个子类中的哪一个。非零表面法线用作表面交互的标识。

Methods are provided to cast to the subclass types as well. This is a good place for a runtime check to ensure that the requested conversion is valid.

> 还提供了转换为子类类型的方法。这是进行运行时检查以确保请求的转换有效的好位置。

Interactions can also represent either an interface between two types of participating media using an instance of the `MediumInterface` class, which is defined in Section 11.4, or the properties of the scattering medium at their point using a `Medium`. Here as well, the Interaction abstraction leaks: surfaces can represent interfaces between media, and at a point inside a medium, there is no interface but there is the current medium. Both of these values are stored in Interaction for the same reasons of expediency that `n` and `uv` were.

> 交互还可以表示两种参与介质之间的界面（使用 `MediumInterface` 类的实例，定义于第 11.4 节），或者使用 `Medium` 表示其点处散射介质的属性。在这里，Interaction 的抽象同样存在泄漏：表面可以表示介质之间的界面，而在介质内部的点处没有界面但有当前介质。这两个值存储在 Interaction 中，理由与存储 `n` 和 `uv` 一样是为了便利。

### 3.11.1 Surface Interaction

As described earlier, the geometry of a particular point on a surface (often a position found by intersecting a ray against the surface) is represented by a SurfaceInteraction. Having this abstraction lets most of the system work with points on surfaces without needing to consider the particular type of geometric shape the points lie on.

> 如前所述，表面上特定点的几何信息（通常是光线与表面求交找到的位置）由 SurfaceInteraction 表示。有了这一抽象，系统的大部分可以处理表面上的点而无需考虑这些点所在的具体几何形状类型。

In addition to the point p, the surface normal n, and (u,v) coordinates from the Interaction base class, the SurfaceInteraction also stores the parametric partial derivatives of the point ∂p/∂u and ∂p/∂v, and the partial derivatives of the surface normal ∂n/∂u and ∂n/∂v. See Figure 3.30 for a depiction of these values.

> 除了来自 Interaction 基类的点 p、表面法线 n 和 (u,v) 坐标外，SurfaceInteraction 还存储点的参数偏导数 ∂p/∂u 和 ∂p/∂v，以及表面法线的偏导数 ∂n/∂u 和 ∂n/∂v。这些值的图示见图 3.30。

Figure 3.30: The Local Differential Geometry around a Point. The parametric partial derivatives of the surface, ∂p/∂u and ∂p/∂v, lie in the tangent plane but are not necessarily orthogonal. The surface normal n is given by the cross product of ∂p/∂u and ∂p/∂v. The vectors ∂n/∂u and ∂n/∂v record the differential change in surface normal as we move u and v along the surface.

> 图 3.30：点周围的局部微分几何。表面的参数偏导数 ∂p/∂u 和 ∂p/∂v 位于切平面中但不一定正交。表面法线 n 由 ∂p/∂u 和 ∂p/∂v 的叉积给出。向量 ∂n/∂u 和 ∂n/∂v 记录了沿表面移动 u 和 v 时表面法线的微分变化。

This representation implicitly assumes that shapes have a parametric description—that for some range of (u,v) values, points on the surface are given by some function f(u,v). Although this is not true for all shapes, all of the shapes that pbrt supports do have at least a local parametric description, so we will stick with the parametric representation since this assumption is helpful elsewhere (e.g., for antialiasing of textures in Chapter 10).

> 这种表示隐式地假设形状具有参数描述——对于某个 (u,v) 值范围，表面上的点由某个函数 f(u,v) 给出。虽然这对所有形状并非都成立，但 pbrt 支持的所有形状至少有局部参数描述，因此我们将坚持使用参数表示，因为这一假设在其他地方很有用（例如，第 10 章中纹理的抗锯齿）。

The SurfaceInteraction constructor takes parameters that set all of these values. It computes the normal as the cross product of the partial derivatives.

> SurfaceInteraction 构造函数接受设置所有这些值的参数。它将法线计算为偏导数的叉积。

SurfaceInteraction stores a second instance of a surface normal and the various partial derivatives to represent possibly perturbed values of these quantities—as can be generated by bump mapping or interpolated per-vertex normals with meshes. Some parts of the system use this shading geometry, while others need to work with the original quantities.

> SurfaceInteraction 存储了表面法线和各种偏导数的第二个实例，以表示这些量可能被扰动后的值——如凹凸贴图或网格的逐顶点法线插值所生成的。系统的某些部分使用此着色几何，而其他部分需要使用原始量。

The shading geometry values are initialized in the constructor to match the original surface geometry. If shading geometry is present, it generally is not computed until some time after the SurfaceInteraction constructor runs. The `SetShadingGeometry()` method updates the shading geometry.

> 着色几何值在构造函数中初始化为与原始表面几何匹配。如果存在着色几何，通常在 SurfaceInteraction 构造函数运行之后的某个时间才计算。`SetShadingGeometry()` 方法更新着色几何。

The surface normal has special meaning to pbrt, which assumes that, for closed shapes, the normal is oriented such that it points to the outside of the shape. For geometry used as an area light source, light is by default emitted from only the side of the surface that the normal points toward; the other side is black. Because normals have this special meaning, pbrt provides a mechanism for the user to reverse the orientation of the normal, flipping it to point in the opposite direction. A `ReverseOrientation` directive in a pbrt input file flips the normal to point in the opposite, non-default direction. Therefore, it is necessary to check if the given Shape has the corresponding flag set and, if so, switch the normal's direction here.

> 表面法线对 pbrt 有特殊含义，pbrt 假设对于封闭形状，法线朝向指向形状的外部。对于用作面光源的几何体，默认情况下仅从法线指向的一侧发射光；另一侧是黑色的。由于法线具有这种特殊含义，pbrt 提供了一种机制让用户反转法线的朝向，将其翻转到相反方向。pbrt 输入文件中的 `ReverseOrientation` 指令将法线翻转到相反的非默认方向。因此，有必要检查给定的 Shape 是否设置了相应的标志，如果是，则在此处切换法线的方向。

However, one other factor plays into the orientation of the normal and must be accounted for here as well. If a shape's transformation matrix has switched the handedness of the object coordinate system from pbrt's default left-handed coordinate system to a right-handed one, we need to switch the orientation of the normal as well. To see why this is so, consider a scale matrix S(-1, 1, 1). We would naturally expect this scale to switch the direction of the normal, although because we have computed the normal by the cross product of ∂p/∂u and ∂p/∂v, the flip is already accounted for.

> 然而，还有一个因素影响法线的朝向，也必须在此处考虑。如果形状的变换矩阵将物体坐标系的手性从 pbrt 默认的左手坐标系切换为右手坐标系，我们也需要切换法线的朝向。要理解原因，考虑缩放矩阵 S(-1, 1, 1)。我们自然期望此缩放会切换法线的方向，尽管由于我们通过 ∂p/∂u 和 ∂p/∂v 的叉积计算法线，翻转已经被考虑在内。

Therefore, it is also necessary to flip the normal's direction if the transformation switches the handedness of the coordinate system, since the flip will not be accounted for by the computation of the normal's direction using the cross product. A flag passed by the caller indicates whether this flip is necessary.

> 因此，如果变换切换了坐标系的手性，也有必要翻转法线的方向，因为使用叉积计算法线方向时不会考虑到这个翻转。调用者传递的标志指示是否需要此翻转。

pbrt also provides the capability to associate an integer index with each face of a polygon mesh. This information is used for certain texture mapping operations.

> pbrt 还提供了将整数索引与多边形网格的每个面关联的功能。此信息用于某些纹理映射操作。

When a shading coordinate frame is computed, the SurfaceInteraction is updated via its `SetShadingGeometry()` method. After performing the same cross product (and possibly flipping the orientation of the normal) to compute an initial shading normal, the implementation then flips either the shading normal or the true geometric normal if needed so that the two normals lie in the same hemisphere. Depending on the context, either the geometric normal or the shading normal may more authoritatively point toward the correct "outside" of the surface, so the caller passes a Boolean value that determines which should be flipped if needed.

> 当计算着色坐标系时，通过其 `SetShadingGeometry()` 方法更新 SurfaceInteraction。在执行相同的叉积（并可能翻转法线朝向）以计算初始着色法线后，实现会在需要时翻转着色法线或真实几何法线，使两个法线位于同一半球。根据上下文，几何法线或着色法线可能更权威地指向表面正确的"外部"，因此调用者传递一个布尔值来决定在需要时应翻转哪个。

### 3.11.2 Medium Interaction

As described earlier, the MediumInteraction class is used to represent an interaction at a point in a scattering medium like smoke or clouds.

> 如前所述，MediumInteraction 类用于表示在烟雾或云等散射介质中某点处的交互。

In contrast to SurfaceInteraction, it adds little to the base Interaction class. The only addition is a `PhaseFunction`, which describes how the particles in the medium scatter light. Phase functions and the PhaseFunction class are introduced in Section 11.3.

> 与 SurfaceInteraction 相比，它对基类 Interaction 的补充很少。唯一的新增是 `PhaseFunction`，它描述介质中的粒子如何散射光线。相位函数和 PhaseFunction 类在第 11.3 节中介绍。
