## 3.7 Bounding Boxes

Many parts of the system operate on axis-aligned regions of space. For example, multi-threading in pbrt is implemented by subdividing the image into 2D rectangular tiles that can be processed independently, and the bounding volume hierarchy in Section [7.3](https://pbr-book.org/4ed/Primitives_and_Intersection_Acceleration/Bounding_Volume_Hierarchies.html#sec:bvh) uses 3D boxes to bound geometric primitives in the scene. The [Bounds2](#Bounds2) and [Bounds3](#Bounds3) template classes are used to represent the extent of these sorts of regions. Both are parameterized by a type T that is used to represent the coordinates of their extents. As with the earlier vector math types, we will focus here on the 3D variant, [Bounds3](#Bounds3), since [Bounds2](#Bounds2) is effectively a subset of it.

> 系统的许多部分在轴对齐的空间区域上操作。例如，pbrt 中的多线程通过将图像细分为可独立处理的二维矩形块来实现，第 7.3 节中的包围体层次结构使用三维盒子来包围场景中的几何图元。Bounds2 和 Bounds3 模板类用于表示这类区域的范围。两者都以用于表示其范围坐标的类型 T 为参数。与之前的向量数学类型一样，我们将在这里重点关注三维变体 Bounds3，因为 Bounds2 实际上是它的子集。

We use the same shorthand as before to define names for commonly used bounding types.

> 我们使用与之前相同的简写来定义常用包围类型的名称。

using Bounds2f = [Bounds2](#Bounds2) <Float>; using Bounds2i = [Bounds2](#Bounds2) <int>; using Bounds3f = [Bounds3](#Bounds3) <Float>; using Bounds3i = [Bounds3](#Bounds3) <int>;

There are a few possible representations for these sorts of bounding boxes; pbrt uses *axis-aligned bounding boxes* (AABBs), where the box edges are mutually perpendicular and aligned with the coordinate system axes. Another possible choice is *oriented bounding boxes* (OBBs), where the box edges on different sides are still perpendicular to each other but not necessarily coordinate-system aligned. A 3D AABB can be described by one of its vertices and three lengths, each representing the distance spanned along the,, and coordinate axes. Alternatively, two opposite vertices of the box can describe it. We chose the two-point representation for pbrt 's [Bounds2](#Bounds2) and [Bounds3](#Bounds3) classes; they store the positions of the vertex with minimum coordinate values and of the one with maximum coordinate values.

> 这类包围盒有几种可能的表示方式；pbrt 使用*轴对齐包围盒*（AABB），其中盒子边缘相互垂直且与坐标系轴对齐。另一种可能的选择是*有向包围盒*（OBB），其中不同边上的边缘仍然彼此垂直但不一定与坐标系对齐。三维 AABB 可以由其一个顶点和三个长度来描述，每个长度表示沿 x、y 和 z 坐标轴跨越的距离。或者，盒子的两个对角顶点可以描述它。我们为 pbrt 的 Bounds2 和 Bounds3 类选择了两点表示法；它们存储具有最小坐标值的顶点和具有最大坐标值的顶点的位置。

[![](https://pbr-book.org/4ed/Geometry_and_Transformations/pha03f09.svg)](https://pbr-book.org/4ed/Geometry_and_Transformations/pha03f09.svg)

Figure 3.9: An Axis-Aligned Bounding Box. The [Bounds2](#Bounds2) and [Bounds3](#Bounds3) classes store only the coordinates of the minimum and maximum points of the box; the other box corners are implicit in this representation.

> 图 3.9：轴对齐包围盒。Bounds2 和 Bounds3 类仅存储盒子最小点和最大点的坐标；其他盒角在此表示中是隐含的。

The default constructors create an empty box by setting the extent to an invalid configuration, which violates the invariant that pMin.x <= pMax.x (and similarly for the other dimensions). By initializing two corner points with the largest and smallest representable number, any operations involving an empty box (e.g., Union()) will yield the correct result.

> 默认构造函数通过将范围设置为无效配置来创建一个空盒子，这违反了 pMin.x <= pMax.x 的不变量（其他维度类似）。通过用可表示的最大和最小数初始化两个角点，涉及空盒子的任何操作（如 Union()）都将产生正确的结果。

Given a bounding box and a point, the Union() function returns a new bounding box that encompasses that point as well as the original bounds.

> 给定一个包围盒和一个点，Union() 函数返回一个同时包含该点和原始边界的新包围盒。

It is similarly possible to construct a new box that bounds the space encompassed by two other bounding boxes.

> 类似地，可以构建一个新的盒子来约束两个其他包围盒所包含的空间。

The intersection of two bounding boxes can be found by computing the maximum of their two respective minimum coordinates and the minimum of their maximum coordinates.

> 两个包围盒的交集可以通过计算它们各自最小坐标的最大值和最大坐标的最小值来找到。

[![](https://pbr-book.org/4ed/Geometry_and_Transformations/pha03f10.svg)](https://pbr-book.org/4ed/Geometry_and_Transformations/pha03f10.svg)

Figure 3.10: Intersection of Two Bounding Boxes.

> 图 3.10：两个包围盒的交集。

We can also determine if two bounding boxes overlap by seeing if their extents overlap in all of,, and:

> 我们还可以通过查看它们的范围是否在 x、y 和 z 所有维度上重叠来确定两个包围盒是否重叠。

Three 1D containment tests determine if a given point is inside a bounding box. The InsideExclusive() variant does not consider points on the upper boundary to be inside the bounds. It is mostly useful with integer-typed bounds.

> 三个一维包含测试确定给定点是否在包围盒内部。InsideExclusive() 变体不将上边界上的点视为在边界内。它主要用于整数类型的边界。

DistanceSquared() returns the squared distance from a point to a bounding box or zero if the point is inside it.

> DistanceSquared() 返回从点到包围盒的平方距离，如果点在盒子内部则返回零。

[![](https://pbr-book.org/4ed/Geometry_and_Transformations/pha03f11.svg)](https://pbr-book.org/4ed/Geometry_and_Transformations/pha03f11.svg)

Figure 3.11: Computing the Squared Distance from a Point to an Axis-Aligned Bounding Box.

> 图 3.11：计算从点到轴对齐包围盒的平方距离。

The Expand() function pads the bounding box by a constant factor in all dimensions.

> Expand() 函数在所有维度上将包围盒按常数因子填充。

Diagonal() returns the vector along the box diagonal from the minimum point to the maximum point.

> Diagonal() 返回从最小点到最大点沿盒子对角线的向量。

Methods for computing the surface area of the six faces of the box and the volume inside of it are also useful.

> 计算盒子六个面的表面积和内部体积的方法也很有用。

The [Bounds3::MaxDimension()](#Bounds3::MaxDimension) method returns the index of which of the three axes is longest. This is useful, for example, when deciding which axis to subdivide when building some of the ray-intersection acceleration structures.

> Bounds3::MaxDimension() 方法返回三个轴中最长的轴的索引。例如，在构建某些光线求交加速结构时决定沿哪个轴细分时，这很有用。

[Lerp()](https://pbr-book.org/4ed/Monte_Carlo_Integration/Sampling_Using_the_Inversion_Method.html#Lerp) linearly interpolates between the corners of the box by the given amount in each dimension.

> Lerp() 在每个维度上按给定量在盒子角点之间进行线性插值。

Offset() is effectively the inverse of Lerp(). It returns the continuous position of a point relative to the corners of the box, where a point at the minimum corner has offset, a point at the maximum corner has offset, and so forth.

> Offset() 实际上是 Lerp() 的逆操作。它返回点相对于盒子角点的连续位置，其中最小角点处的点偏移为 (0,0,0)，最大角点处的点偏移为 (1,1,1)，依此类推。

[Bounds3](#Bounds3) also provides a method that returns the center and radius of a sphere that bounds the bounding box. In general, this may give a far looser fit than a sphere that bounded the original contents of the [Bounds3](#Bounds3) directly, although for some geometric operations it is easier to work with a sphere than a box, in which case the worse fit may be an acceptable trade-off.

> Bounds3 还提供了一个返回包围该包围盒的球体的中心和半径的方法。一般来说，这可能比直接包围 Bounds3 原始内容的球体松弛得多，不过对于某些几何操作来说，使用球体比使用盒子更容易，在这种情况下较差的拟合可能是可接受的权衡。

Straightforward methods test for empty and degenerate bounding boxes. Note that "empty" means that a bounding box has zero volume but does not necessarily imply that it has zero surface area.

> 简单的方法测试空的和退化的包围盒。注意"空"意味着包围盒具有零体积，但不一定意味着它具有零表面积。

Finally, for integer bounds, there is an iterator class that fulfills the requirements of a C++ forward iterator. Having this definition makes it possible to write code using range-based for loops to iterate over integer coordinates in a bounding box:

> 最后，对于整数边界，有一个满足 C++ 前向迭代器要求的迭代器类。有了这个定义，就可以使用基于范围的 for 循环来遍历包围盒中的整数坐标：

Bounds2i b =...; for (Point2i p: b) { // … }

As implemented, the iteration goes up to but does not visit points equal to the maximum extent in each dimension.

> 按照实现，迭代向上进行但不访问每个维度中等于最大范围的点。
