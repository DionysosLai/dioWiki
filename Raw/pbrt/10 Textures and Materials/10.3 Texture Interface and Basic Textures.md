## 10.3 Texture Interface and Basic Textures

Given a variety of ways to generate 2D and 3D texture coordinates, we will now define the general interfaces for texture functions. As mentioned earlier, pbrt supports two types of Texture s: scalar Float -valued, and spectral-valued.

> 有了生成 2D 和 3D 纹理坐标的多种方法，我们现在将定义纹理函数的通用接口。如前所述，pbrt 支持两种类型的 Texture：标量 Float 值和光谱值。

For the first, there is FloatTexture, which is defined in [base/texture.h](https://github.com/mmp/pbrt-v4/tree/master/src/pbrt/base/texture.h). There are currently 14 implementations of this interface in pbrt, which leads to a lengthy list of types for the [TaggedPointer](https://pbr-book.org/4ed/Utilities/Containers_and_Memory_Management.html#TaggedPointer) template class. Therefore, we have gathered them into a fragment, <<FloatTextures>>, that is not included here.

### 10.3.1 Constant Texture

> ### 10.3.1 常量纹理

The constant textures return the same value no matter where they are evaluated. Because they represent constant functions, they can be accurately reconstructed with any sampling rate and therefore need no antialiasing. Although these two textures are trivial, they are actually quite useful. By providing these classes, all parameters to all [Material](https://pbr-book.org/4ed/Textures_and_Materials/Material_Interface_and_Implementations.html#Material) s can be represented as Texture s, whether they are spatially varying or not. For example, a red diffuse object will have a [SpectrumConstantTexture](#SpectrumConstantTexture) that always returns red as the diffuse color of the material. This way, the material system always evaluates a texture to get the surface properties at a point, avoiding the need for separate textured and nontextured versions of materials. Such an approach would grow increasingly unwieldy as the number of material parameters increased.

> 常量纹理无论在哪里评估都返回相同的值。因为它们表示常量函数，可以用任何采样率精确重建，因此不需要反走样。虽然这两个纹理很简单，但实际上非常有用。通过提供这些类，所有 Material 的所有参数都可以表示为 Texture，无论它们是否空间变化。例如，一个红色漫反射对象将有一个 SpectrumConstantTexture，始终返回红色作为材质的漫反射颜色。这样，材质系统总是评估纹理来获取某点的表面属性，避免了需要材质的纹理和非纹理两个版本。随着材质参数数量的增加，这种做法会变得越来越笨拙。

FloatConstantTexture, like all the following texture implementations, is defined in the files [texture.h](https://github.com/mmp/pbrt-v4/tree/master/src/pbrt/texture.h) and [texture.cpp](https://github.com/mmp/pbrt-v4/tree/master/src/pbrt/texture.cpp).

### 10.3.2 Scale Texture

We have defined the texture interface in a way that makes it easy to use the output of one texture function when computing another. This is useful since it lets us define generic texture operations using any of the other texture types. The [FloatScaledTexture](#FloatScaledTexture) takes two Float -valued textures and returns the product of their values.

<<FloatScaledTexture Definition>>=

class FloatScaledTexture { public: << [FloatScaledTexture Public Methods](#fragment-FloatScaledTexturePublicMethods-0) >>

FloatScaledTexture([FloatTexture](#FloatTexture) [tex](#FloatScaledTexture::tex), [FloatTexture](#FloatTexture) [scale](#FloatScaledTexture::scale)): [tex](#FloatScaledTexture::tex) ([tex](#FloatScaledTexture::tex)), [scale](#FloatScaledTexture::scale) ([scale](#FloatScaledTexture::scale)) {} static [FloatTexture](#FloatTexture) Create(const [Transform](https://pbr-book.org/4ed/Geometry_and_Transformations/Transformations.html#Transform) &renderFromTexture, const TextureParameterDictionary &parameters, const FileLoc \*loc, Allocator alloc); Float [Evaluate](#FloatTexture::Evaluate) ([TextureEvalContext](https://pbr-book.org/4ed/Textures_and_Materials/Texture_Coordinate_Generation.html#TextureEvalContext) ctx) const { Float sc = [scale](#FloatScaledTexture::scale).[Evaluate](#FloatTexture::Evaluate) (ctx); if (sc == 0) return 0; return [tex](#FloatScaledTexture::tex).[Evaluate](#FloatTexture::Evaluate) (ctx) \* sc; } std::string ToString() const;

private: [FloatTexture](#FloatTexture) tex, scale; };

[FloatScaledTexture](#FloatScaledTexture) ignores antialiasing, leaving it to its two subtextures to antialias themselves but not making an effort to antialias their product. While it is easy to show that the product of two band-limited functions is also band limited, the maximum frequency present in the product may be greater than that of either of the two terms individually. Thus, even if the scale and value textures are perfectly antialiased, the result might not be. Fortunately, the most common use of this texture is to scale another texture by a constant, in which case the other texture’s antialiasing is sufficient.

One thing to note in the implementation of its Evaluate() method is that it skips evaluating the tex texture if the scale texture returns 0. It is worthwhile to avoid incurring the cost of this computation if it is unnecessary.

### 10.3.3 Mix Textures

The mix textures are more general variations of the scale textures. They blend between two textures of the same type based on a scalar blending factor. Note that a constant texture could be used for the blending factor to achieve a uniform blend, or a more complex Texture could be used to blend in a spatially nonuniform way. Figure [10.12](#fig:mix-texture-cup) shows the use of the [SpectrumMixTexture](#SpectrumMixTexture) where an image is used to blend between two constant RGB colors.

<<FloatMixTexture Definition>>=

class FloatMixTexture { public: << [FloatMixTexture Public Methods](#fragment-FloatMixTexturePublicMethods-0) >>

FloatMixTexture([FloatTexture](#FloatTexture) [tex1](#FloatMixTexture::tex1), [FloatTexture](#FloatTexture) [tex2](#FloatMixTexture::tex2), [FloatTexture](#FloatTexture) [amount](#FloatMixTexture::amount)): [tex1](#FloatMixTexture::tex1) ([tex1](#FloatMixTexture::tex1)), [tex2](#FloatMixTexture::tex2) ([tex2](#FloatMixTexture::tex2)), [amount](#FloatMixTexture::amount) ([amount](#FloatMixTexture::amount)) {} Float [Evaluate](#FloatTexture::Evaluate) ([TextureEvalContext](https://pbr-book.org/4ed/Textures_and_Materials/Texture_Coordinate_Generation.html#TextureEvalContext) ctx) const { Float amt = [amount](#FloatMixTexture::amount).[Evaluate](#FloatTexture::Evaluate) (ctx); Float t1 = 0, t2 = 0; if (amt!= 1) t1 = [tex1](#FloatMixTexture::tex1).[Evaluate](#FloatTexture::Evaluate) (ctx); if (amt!= 0) t2 = [tex2](#FloatMixTexture::tex2).[Evaluate](#FloatTexture::Evaluate) (ctx); return (1 - amt) \* t1 + amt \* t2; } static FloatMixTexture \*Create(const [Transform](https://pbr-book.org/4ed/Geometry_and_Transformations/Transformations.html#Transform) &renderFromTexture, const TextureParameterDictionary &parameters, const FileLoc \*loc, Allocator alloc); std::string ToString() const;

private: [FloatTexture](#FloatTexture) tex1, tex2; [FloatTexture](#FloatTexture) amount; };

To evaluate the mixture, the three textures are evaluated and the floating-point value is used to linearly interpolate between the two. When the blend amount amt is zero, the first texture’s value is returned, and when it is one the second one’s value is returned. The Evaluate() method here makes sure not to evaluate textures unnecessarily if the blending amount implies that only one of their values is necessary. (Section [15.1.1](https://pbr-book.org/4ed/Wavefront_Rendering_on_GPUs/Mapping_Path_Tracing_to_the_GPU.html#sec:basic-gpu-model) has further discussion about why the logic for that is written just as it is here, rather than with, for example, cascaded if tests that each directly return the appropriate value.) We will generally assume that amt will be between zero and one, but this behavior is not enforced, so extrapolation is possible as well.

As with the scale textures, antialiasing is ignored, so the introduction of aliasing here is a possibility.

<<FloatMixTexture Public Methods>>=

Float [Evaluate](#FloatTexture::Evaluate) ([TextureEvalContext](https://pbr-book.org/4ed/Textures_and_Materials/Texture_Coordinate_Generation.html#TextureEvalContext) ctx) const { Float amt = [amount](#FloatMixTexture::amount).[Evaluate](#FloatTexture::Evaluate) (ctx); Float t1 = 0, t2 = 0; if (amt!= 1) t1 = [tex1](#FloatMixTexture::tex1).[Evaluate](#FloatTexture::Evaluate) (ctx); if (amt!= 0) t2 = [tex2](#FloatMixTexture::tex2).[Evaluate](#FloatTexture::Evaluate) (ctx); return (1 - amt) \* t1 + amt \* t2; }

We will not include the implementation of SpectrumMixTexture here, as it parallels that of FloatMixTexture.

The FloatDirectionMixTexture and SpectrumDirectionMixTexture use the dot product of the surface normal with a specified direction to compute such a weight. As they are very similar, we will only discuss SpectrumDirectionMixTexture here.
